    Inicio rápido: Personalización del jugador: aspecto heredado

    Este documento contiene información sobre el aspecto de Brightcove Player 1.x (grafito), no sobre el aspecto más actual. Esta transición a la piel más nueva tuvo lugar la segunda semana de diciembre de 2015. Para ver la versión más reciente del documento, consulte este documento.

    En este tema, se proporcionan pasos prácticos que, cuando se siguen, personaliza el aspecto del Reproductor de Brightcove. Para personalizar el aspecto del reproductor de Brightcove, debe utilizar Hojas de estilos en cascada (CSS). Este documento pasará por el proceso de cambiar la apariencia y la ubicación del botón de reproducción que inicialmente superpone el vídeo.

    Resumen

    Este documento tendrá varias secciones, ya que el uso de CSS con el incrustado en página y las implementaciones iframe del reproductor son bastante diferentes.

    El contenido de este documento utiliza las herramientas de desarrollo de Chrome, pero todos los navegadores modernos tienen una funcionalidad equivalente. Para obtener una breve introducción al uso de herramientas de desarrollo en diferentes navegadores, consulte el documento Conceptos básicos de depuración .

    Nota: En Video Cloud Studio se proporciona un estilo básico del reproductor. Para obtener más información, consulte Cambiar el estilo de los reproductores.

    Determinar el selector de jugadores

    Los selectores CSS le permiten seleccionar y manipular elementos HTML en una página. Los selectores CSS se utilizan para «encontrar» (o seleccionar) elementos HTML en función de su id, clase, tipo, atributo y más. Lo primero que debe hacer al intentar alterar el aspecto de un elemento en una página HTML es determinar el selector que se utilizará para dirigir el elemento. Los siguientes pasos muestran cómo encontrar el selector para el reproductor y, a continuación, cambiar el tamaño del reproductor.

    1. En Video Cloud Studio, abra el módulo Media y seleccione el vídeo que desea publicar.
    2. Copie el código de publicación en página .
    3. Con el editor HTML, cree una nueva página HTML.
    4. Pegue el código de publicación en el cuerpo de la página HTML.
    5. Examine la página recién creada. De forma predeterminada, el tamaño del reproductor tendrá dimensiones predeterminadas de 300 x 150 píxeles.
    6. Abre las herramientas de desarrollo de tu navegador y profundiza en el HTML para encontrar la div etiqueta que rodea al jugador. Verá que la div etiqueta tiene una clase de video-js. El div que contiene el jugador siempre tendrá una clase de video-js . video-js es el nombre de la clase que necesitaremos al diseñar el jugador.
    7. Compruebe la captura de pantalla a continuación para ver la clase resaltada en rojo. En verde se destaca la etiqueta de vídeo, que es Brightcove Player.

      Nota: Dependiendo del sistema operativo y del navegador que se utilice, puede mostrarse la versión de reserva Flash del reproductor. Si su reproductor está utilizando la versión de reserva Flash del reproductor, no verá ninguna video etiqueta. En su lugar, verá el div entorno de una objectetiqueta, como se resalta a continuación.

    8. Ahora que conoce la clase correcta para usar para un selector CSS, podemos usar ese selector de clase para cambiar el tamaño del reproductor. El style bloque de abajo debe colocarse justo encima del final de la head sección de su página HTML. En terminología CSS, este es un ejemplo del uso de un selector de clases. 
              <style>
          .video-js {
            height: 344px;
            width: 610px;
          }
        </style>
      </head>
    9. Guarde y navegue por su página HTML y verá que el reproductor tiene un nuevo tamaño.

    Selector de botón Reproducir

    Otro enfoque se va a utilizar para encontrar el selector para el botón de reproducción. Los siguientes pasos muestran cómo encontrar el selector correcto para el botón de reproducción y, a continuación, cambiar su apariencia.

    1. Examine su página HTML.
    2. Haga clic derecho en el botón de reproducción que superpone al jugador y seleccione Inspeccionar elemento.
    3. Tenga en cuenta que la div etiqueta que está resaltada representa el botón de reproducción y tiene una clase de vjs-big-play-button.
    4. Altere sus estilos para incluir CSS que cambie el color y la opacidad del botón de reproducción (líneas 17-18), así como centrar el botón dentro del reproductor (líneas 19-23). 
            <style>
          .video-js {
            height: 344px;
            width: 610px;
          }
          .vjs-big-play-button {
            background-color: red;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
        </style>
    5. Examine su página y verá que no se han utilizado todos los estilos. El botón no es rojo.
    6. Mira a la derecha de tus herramientas de desarrollador y verás un resumen de tus estilos. Puede mirar hacia abajo y ver que cambiar el fondo a rojo está anulado, representado por el tachado.
      Nota: En Safari, el estilo puede no ser tachado a pesar de que está siendo anulado.

      Lo que ha sucedido es que se realizan numerosos cambios en el color de fondo. El estilo que establece se ha anulado en la parte de «cascada» de CSS. Puede cambiar esto utilizando la funcionalidad de CSS llamada especificidad. La especificidad determina qué regla CSS es aplicada por los navegadores y es un tipo de ponderación que influye en cómo se muestran las reglas de hoja de estilos en cascada (CSS). Usando la especificidad, puede hacer que su estilo sea más importante que otros, forzándolo a ser utilizado. Hay una breve introducción a la especificidad de CSS en el documento Personalizar apariencia del jugador - Aspecto heredado , y numerosas referencias en cualquier búsqueda.

    7. Tenga en cuenta que la div etiqueta rodea al jugador tiene numerosas clases asociadas con él.
    8. En este caso, usando la combinación de video-js y « vjs-mouse aumentará» sus estilos para tener prioridad. Altere sus estilos para utilizar el nuevo selector de combinación. Tenga en cuenta que no hay espacio entre las dos primeras clases, pero hay una antes de la última clase (detalles de por qué se pueden encontrar en las reglas de especificidad). 
                .video-js.vjs-mouse .vjs-big-play-button {
            background-color: #ff0000;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
    9. Examine su página y vea que ahora se aplican todos los estilos.
    10. Vuelva a mirar a la derecha de sus herramientas de desarrollador y verá un resumen de sus estilos. Ahora verá que el cambio de color de fondo no se anula.

    Incrustar en página

    Cuando se trabaja con selectores y estilos anteriores, los resultados se utilizaron con la implementación incrustada en página del reproductor, como se muestra aquí:

          <!doctype html>
      <html>
      
      <head>
          <meta charset="UTF-8">
          <title>Embed in page tester</title>
         <style>
          .video-js {
            height: 344px;
            width: 610px;
          }
          .video-js.vjs-mouse .vjs-big-play-button {
            background-color: red;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
         </style>
      </head>
      <body>
      
        <video
          data-video-id="4443311217001"
          data-account="1507807800001"
          data-player="30a9c9e0-295b-4102-a7af-a3f86bba91ab"
          data-embed="default"
          class="video-js" controls></video>
        <script src="//players.brightcove.net/1507807800001/30a9c9e0-295b-4102-a7af-a3f86bba91ab_default/index.min.js"></script>
      
      
      </body>
      </html>

    Esta es una forma muy común de usar CSS con una página HTML. Verá que el style bloque está en la página con la video etiqueta.

    Nota: Dependiendo de su experiencia CSS, es posible que se pregunte por qué el código anterior no asignó un id atributo a la video etiqueta. A continuación, podría usar un selector como.#myPlayer .vjs-big-play-button De hecho, podrías haberlo hecho y habría funcionado. El problema es que el CSS creado con selectores de id no funcionará cuando se usa la implementación del reproductor iframe, por lo que solo se usaron los nombres de clase existentes. Al no usar un id atributo, los estilos serán directamente aplicables a las implementaciones de iframe player.

    iframe

    No es sorprendente que la implementación del iframe coloque Brightcove Player en un iframe que luego coloque en una página HTML. Puede dar estilo al iframe real, cambiar su tamaño, por ejemplo, utilizando estilos en la página que contiene el iframe, pero los estilos en la página del iframe NO caen en cascada en el iframe mismo. El iframe literalmente extrae una página HTML separada, por lo que los estilos deseados para el contenido del iframe deben estar asociados con la página HTML en el iframe.

    Cuando se utiliza la implementación iframe del Reproductor de Brightcove, los pasos para aplicar CSS son los siguientes:

    • Colocar estilos en un archivo accesible a Internet
    • Vincular la hoja de estilo al reproductor (hecho mediante Studio o la API de administración de jugadores)
    Nota: Los siguientes pasos suponen que trabajará desde el HTML que completó en los pasos anteriores y se muestra en el fragmento de código anterior. Si desea mantenerlo como ejemplo de trabajo, simplemente copie el código en otro archivo e inicie los siguientes pasos en el nuevo archivo.
    1. En Studio, publique un vídeo y copie el código para una implementación iframe del reproductor.
    2. En su página HTML, reemplace el código incrustado en la página por el código iframe.
    3. Cambie el selector de .video-js clase a un selector de elementos utilizando iframe como elemento. Dejar width y lo height mismo, como se muestra aquí: 
        iframe {
      height: 344px;
      width: 610px;
    }
    4. Compruebe que su página HTML tiene un aspecto similar al siguiente: 
            <!doctype html>
      <html>
      <head>
        <meta charset="utf-8">
        <title>iframe tester</title>
      
         <style>
          iframe {
            height: 344px;
            width: 610px;
          }
          .video-js.vjs-mouse .vjs-big-play-button {
            background-color: red;
            opacity: .3;
            top: 0px;
            left: 0px;
            right: 0px;
            bottom: 0px;
            margin: auto;
          }
         </style>
      <body>
      
        <iframe src='//players.brightcove.net/1507807800001/30a9c9e0-295b-4102-a7af-a3f86bba91ab_default/index.html?videoId=4443311217001'
          allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
      
      </body>
      </html>
    5. Navegue por la página y verá que el reproductor iframe tiene el tamaño correcto. Como se mencionó anteriormente, los estilos del botón NO funcionarán.
    6. Cree un archivo denominado customize-qs.css y guárdelo en una ubicación accesible a Internet.
    7. Corta los estilos para el botón y péguelos en el archivo y guárdelo. Cortar sólo los estilos del botón, no se pueden incluir las style etiquetas. El archivo debe contener solo este contenido: 
            .video-js.vjs-mouse .vjs-big-play-button {
        background-color: red;
        opacity: .3;
        top: 0px;
        left: 0px;
        right: 0px;
        bottom: 0px;
        margin: auto;
      }
    8. Compruebe que su página HTML tiene un aspecto similar al siguiente: 
            <!doctype html>
      <html>
      <head>
        <meta charset="utf-8">
        <title>iframe tester</title>
      
        <style>
          iframe {
            height: 344px;
            width: 610px;
          }
        </style>
      <body>
      
        <iframe src='//players.brightcove.net/1507807800001/30a9c9e0-295b-4102-a7af-a3f86bba91ab_default/index.html?videoId=4443311217001'
          allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
      
      </body>
      </html>
    9. En Studio, vaya al módulo Reproductores y edite el reproductor que está utilizando para este inicio rápido.
    10. Localice la sección PLUGINS y haga clic en el botón Editar .
    11. Haga clic en la entrada CSS e ingrese la ruta a su archivo CSS, como se muestra aquí:
    12. Haga clic en Guardar para guardar los cambios en la sección PLUGINS .
      El reproductor de vista previa en la parte superior de la página se actualizará para mostrar los estilos CSS aplicados al botón.
    13. En la parte superior de la página, haz clic en Publicar para publicar el reproductor.
    14. Vuelva a su navegador y actualice la página. Una vez que los cambios publicados estén activos, verá sus estilos aplicados al reproductor iframe. Los cambios publicados pueden pasar varios minutos antes de que se vean los cambios publicados.

    Explorar estilos

    En esta sección se utiliza un JSBin para permitirle explorar estilos y hacer más personalizaciones en el botón de juego grande que lo que se muestra arriba. Una vez que obtenga un aspecto que le guste, simplemente puede copiar el CSS generado y usarlo con cualquiera de las implementaciones de reproductor como se muestra arriba.
