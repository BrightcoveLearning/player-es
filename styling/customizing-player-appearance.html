    Personalización de la apariencia del reproductor

    Este tema es una introducción sobre la personalización de la apariencia de Brightcove Player. El contenido utiliza herramientas de desarrollo de Chrome, pero todos los navegadores modernos tienen una funcionalidad equivalente. Para obtener una breve introducción al uso de herramientas de desarrollo en diferentes exploradores, consulte el documento Conceptos básicos de depuración .
    Este documento personaliza el reproductor usando CSS. Hay un conjunto limitado de opciones de personalización disponibles en Studio, que no requiere conocimiento de CSS para implementar. Consulte el documento Jugadores de estilos para obtener más información.

    Crea tu propia piel

    Si no desea utilizar el aspecto predeterminado y crear el suyo propio, esto es posible mediante la API de administración de Player. La Guía de configuración del reproductor proporciona la información necesaria.

    Reproductor

    Nota: Este documento asume algunos conocimientos de CSS y el uso de herramientas de desarrollo de un navegador para ver las clases del elemento HTML y examinar CSS aplicable. Si no tiene el conocimiento de base previo requisito, sería mejor que trabaje a través del paso a paso: Personalización del reproductor.

    Puede alterar la apariencia del jugador, pero primero debe tener una forma de abordarlo. Para ver esto, busque una página HTML con el código Avanzado (incrustar en página) insertado. En la Elements sección de las Herramientas de desarrollo localiza la <video-js> etiqueta y verás que tiene una clase con el valor video-js asignado, entre muchos otros.

    Una vez que lo sepas, puedes usar estilos para alterar el propio jugador. Por ejemplo, para incluir un borde alrededor del reproductor puede usar este estilo:

    <style>
  .video-js {
    border-style: double;
    border-width: thick;
    border-color: red;
  }
</style>

    reproductor iframe

    Si está utilizando la implementación Estándar (iframe) del jugador, la situación es diferente. Todavía verás al jugador con la video-js clase, pero estará, por supuesto, dentro del iframe, y dentro de la video-js etiqueta.

    Los estilos que cree seguirán funcionando con el reproductor en el iframe, pero será necesario crear un archivo CSS y asociarlo con el reproductor utilizando Studio. Para ello, vaya al módulo Jugadores , luego haga clic en el reproductor con el que desea asociar el CSS, luego en la sección Plugins > Stylesheets agregue la ruta que su archivo CSS.

    Si desea personalizar el iframe en sí, puede hacerlo utilizando un selector de elementos. A continuación, utilice una etiqueta para modificar como desee.<style> En el ejemplo siguiente se añade un borde alrededor del jugador.

  <meta charset="UTF-8">
  <title>Untitled Document</title>
  <style>
    .video-js {
      border-style: double;
      border-width: thick;
      border-color: red;
    }
  </style>
</head>

    Botón Reproducir

    Si desea modificar la apariencia del botón de reproducción, primero necesita saber cómo abordarlo. En Chrome, haga clic derecho en el botón. De las selecciones que aparecen elija Inspect. Cuando seleccionas Inspect , Chrome Herramientas de desarrollo abrirá.

    En la Elements sección de Herramientas de desarrollo, verá el código HTML que corresponde al elemento button.

    Desde el código resaltado arriba, se puede ver la clase asignada al botón es vjs-big-play-button. El color del botón de reproducción (para que coincida con el tigre) y la opacidad se cambiarán con estos estilos:

    .vjs-big-play-button {
  background-color: #B37D5B;
  opacity: .6;
}

    Usted pensaría que este enfoque funcionaría, pero no lo hace. Al trabajar con CSS, debe ser consciente de la especificidad de la propiedad. Esto se refiere a cómo el navegador determina qué valores de propiedad se aplican a un elemento en función de su relevancia.

    Una forma de aplicar los cambios de botón es agregar otra clase en el selector. Por ejemplo, puede usar la .video-js clase para aumentar la especificidad del selector. Para obtener más detalles, consulte la sección de especificidad CSS

    A continuación, haga referencia al botón de juego grande del jugador en el CSS de la siguiente manera:

    .video-js .vjs-big-play-button {
  background-color: #B37D5B;
  opacity: .6;
}

    Aquí está el resultado de cambiar el botón de gris oscuro a naranja:

    Ocultar botón de reproducción

    Puede ocultar completamente el botón de reproducción utilizando el siguiente estilo:

    Cambiar texto de cursor

    Si desea cambiar el texto que aparece al pasar el cursor sobre el botón de reproducción grande, que es Reproducir vídeode forma predeterminada, a algo más específico de sus vídeos, puede hacerlo. Agregue el siguiente JavaScript utilizando la verbiage deseada como argumento para el controlText método.

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  myPlayer.getChild('bigPlayButton').controlText('Play Audio');
});

    Controla visibilidad

    Puede controlar si desea que la barra de control, y los controles contenidos en ella, sean visibles o no. Si desea ocultar la barra de control desde el inicio de la creación del reproductor, puede usar este estilo:

    .video-js .vjs-control-bar {
  display: none;
}

    Si desea hacer esto dinámicamente basado en algún evento, puede usar los controlBar.hide()/show() métodos. El siguiente fragmento de código muestra el uso de los métodos (esto supone que tiene id un valor de myPlayer en el video):

    <video-js id="myPlayerID"
	data-video-id="5781068653001"
	data-account="1507807800001"
	data-player="default"
	data-embed="default"
	data-application-id
	class="video-js"
	controls></video-js>
<script src="https://players.brightcove.net/1507807800001/default_default/index.min.js"></script>

<p><button onclick="hideBtn()">hide controls</button></p>
<p><button onclick="showBtn()">show controls</button></p>

<script>
  var myPlayer;
	videojs.getPlayer('myPlayerID').ready(function() {
		myPlayer = this;
	});

	function hideBtn() {
		myPlayer.controlBar.hide();
	}

	function showBtn() {
		myPlayer.controlBar.show();
	}
</script>

    Los controles nunca se ocultan

    También puede hacerlo para que los controles nunca se oculten. Para hacer esto use el transform.none estilo. El selector es muy largo ya que seis clases se utilizan para la especificidad de las transformaciones.

    .video-js.not-hover.vjs-has-started.vjs-paused.vjs-user-active .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible),
.video-js.not-hover.vjs-has-started.vjs-paused.vjs-user-inactive .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible),
.video-js.not-hover.vjs-has-started.vjs-playing.vjs-user-active .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible),
.video-js.not-hover.vjs-has-started.vjs-playing.vjs-user-inactive .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible),
.video-js.vjs-has-started.vjs-paused.vjs-ad-playing.vjs-user-inactive .vjs-control-bar,
.video-js.vjs-has-started.vjs-playing.vjs-user-inactive .vjs-control-bar:not(.vjs-focus-within):not(.vjs-control-bar-visible){
  transform: none;
  -webkit-transform: none;
  -ms-transform: none;
}
    Aunque se considera un poco un truco, puede simplemente repetir la clase de la barra de control ocho veces para construir su especificidad para que coincida con el selector anterior: 
    .vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar.vjs-control-bar {
  transform: none;
  -webkit-transform: none;
  -ms-transform: none;
}
    Usar este CSS para evitar que la barra de control desaparezca es muy útil al hacer clic derecho en los elementos de la barra de control para inspeccionarlos. Si no lo hace, al hacer clic con el botón derecho, la barra de control desaparecerá. También tenga en cuenta que el primer botón derecho muestra el menú de información del Brightcove Player y el segundo botón derecho mostrará el menú donde podrá elegir inspeccionar.

    Actualizar mediante el objeto options

    Para cambiar el diseño del control de volumen, puede pasar el options objeto al método bc () de la siguiente manera:

    1. Retrasar la creación del reproductor eliminando los siguientes atributos de la video etiqueta:
      • data-account
      • data-player
      • data-video-id
    2. Defina el options objeto de modo que el volumePanel sea vertical y no esté en línea con la barra de control. 
      var options = {
  controlBar: {
    volumePanel: {
      inline: false,
      vertical: true
    }
  }
};
    3. Añada los atributos del jugador que se eliminaron inicialmente.
    4. Cree el reproductor llamando al método bc () con el options objeto. 
      bc("myPlayerID", options);
    5. Para obtener más información, consulte el ejemplo del reproductor Vertical Volume Control .

    Fuentes

    El título y la descripción aparecen cuando el reproductor se carga por primera vez y cuando el usuario pasa el cursor sobre el reproductor. Puede cambiar el estilo de las fuentes del título y la descripción, ya sea juntas o por separado. El HTML del dock aparece como se muestra en la siguiente captura de pantalla. Tenga en cuenta que hay un <div> elemento con una clase de vjs-dock-text que contiene otros dos <div> elementos que contienen el título y descripción.

    Puede afectar tanto al título como a la descripción estableciendo un estilo en la vjs-dock-text clase. Tenga en cuenta que los estilos establecidos en el .vjs-dock-title y .vjs-dock-description , por supuesto, no serán completamente anulados por el estilo establecido en un nivel superior. Algunos estilos se heredan, como se muestra en el siguiente estilo y su resultado en el texto:

    .video-js .vjs-dock-text {
  color: yellow;
  font-size: .7em;
}

    Puede dar estilo directamente al título y la descripción individuales. A continuación se muestran los estilos establecidos en el .vjs-dock-title y .vjs-dock-description selectores de clase, con el texto resultante mostrando los estilos:

    .video-js .vjs-dock-title {
  color: red;
  font-size: 1.5em;
}
.video-js .vjs-dock-description {
  color: yellow;
  font-size: 1.0em;
}

    Manipulación de iconos de la barra

    El orden de los iconos de la barra de controles determinado por el orden de <div>los HTML s en la barra de control principal <div>. Aquí se ve la barra de control <div> con todos sus hijos.

    Generalmente, este es el enfoque a tomar cuando se coloca en el icono en la barra de control:

    • Construya dinámicamente el elemento que desea insertar en la barra de control.
    • Obtener una referencia al elemento de la barra de control.
    • Obtenga una referencia al elemento que desea insertar su nuevo elemento delante de en la barra de control.
    • Utilice el insertBefore() método JavaScript para colocar el icono.

    La sintaxis de insertBefore() es:

    parent.insertBefore(newElement,insertBeforeElement)

    Esto significa que en el código detallado a continuación verá JavaScript para obtener una referencia a la barra de control (padre), una nueva <div> con un icono para insertar en la barra de control y, por último, una referencia al elemento desde el que se insertará el nuevo icono de.

    Específicamente, se utiliza el siguiente código para insertar un icono de descarga de vídeo delante del icono del volumen:

      • El <div> que contendrá el icono para el icono de descarga.
      • La etiqueta de anclaje que hace que el icono se pueda hacer clic.
      • La imagen del icono en sí.
    • Líneas 297-2980: Asignar e id y clases CSS al nuevo elemento.
    • Línea 300: Asigne una fuente a la imagen.
    • Línea 301: Asigne un href al vínculo.
    • Línea 302: Añada la imagen al enlace.
    • Línea 303: Añada el vínculo al creado dinámicamente <div>.
    • Línea 306: Obtener una referencia a la barra de control para usarla insertBefore().
    • Línea 308: Obtener una referencia al elemento con el que se insertará el nuevo icono delante de, para usar con insertBefore().
    • Línea 310: Inserte el elemento creado dinámicamente.
    <!-- styles for page -->
  /* Style the download icon */
  #downloadButton {
      width: 34px;
      height: 33px;
      margin-top: 6px;
  }
</style>

  videojs.getPlayer('myPlayerID').ready(function() {

    var myPlayer = this,
      controlBar,

      newLink = document.createElement('a'),
      newImage = document.createElement('img');
    // Assign id and classes to div for icon
    newElement.id = 'downloadButton';
    newElement.className = 'downloadStyle vjs-control';
    // Assign properties to elements and assign to parents
    newImage.setAttribute('src','http://solutions.brightcove.com/bcls/brightcove-player/download-video/file-download.png');
    newLink.setAttribute('href','http://www.brightcove.com');
    newLink.appendChild(newImage);
    newElement.appendChild(newLink);
    // Get controlbar and insert before elements
    // Remember that getElementsByClassName() returns an array
    controlBar = document.getElementsByClassName('vjs-control-bar')[0];
    // Change the class name here to move the icon in the controlBar
    insertBeforeNode = document.getElementsByClassName('vjs-volume-menu-button')[0];
    // Insert the icon div in proper location
    controlBar.insertBefore(newElement,insertBeforeNode);
  });

    Usar el espaciador

    Aunque las técnicas anteriores funcionan para colocar iconos en la barra de control, es posible que no sean compatibles con otro código que manipule la barra de control. Además, lo más probable es que no desee abstraer el código en un complemento para su reutilización general.

    Las debilidades del párrafo anterior se pueden superar utilizando el elemento espaciador especial insertado en la barra de control, especialmente para añadir iconos. El inconveniente del espaciador es que no tiene control total sobre dónde se puede colocar el icono, mientras que lo hace utilizando las técnicas mostradas anteriormente en esta sección.

    El elemento espaciador se encuentra en la barra de control entre la información de tiempo a la izquierda y el icono de pantalla completa a la derecha, como se muestra en la siguiente captura de pantalla (resaltada en el óvalo verde). Tenga en cuenta que en la sección Elementos de las Herramientas de desarrollo el elemento se define como HTML <div>.

    Para colocar un icono en el elemento espaciador, utilice el siguiente código, que es muy similar al código mostrado anteriormente en esta sección. El resultado del código se muestra en la captura de pantalla que sigue al fragmento de código.

    // Get the spacer element
spacer = document.getElementsByClassName('vjs-spacer')[0];
// Place the new element in the spacer
spacer.appendChild(newElement);

    Si desea colocar el icono en el lado derecho del espaciador, se debe usar otra línea de código. El diseño del espaciador está controlado por un Flexbox HTML. Por lo tanto, añadiendo un estilo Flexbox puede empujar el icono añadido a la derecha. El resultado del código se muestra en la captura de pantalla que sigue al fragmento de código.

    // Set the content of the spacer to be right justified
spacer.setAttribute("style", "justify-content: flex-end;");

    Degradado de cursor

    Cuando el reproductor se carga por primera vez y al pasar el ratón sobre el reproductor, verá el título del vídeo mostrado, así como un degradado negro a transparente sobre la parte superior del reproductor. Esta sección le mostrará cómo modificar el degradado.

    El gradiente real está controlado por CSS linear-gradient función en el vjs-dock-text HTML <div>. El tamaño del gradiente se puede controlar mediante la altura del vjs-dock-text HTML <div>. Aquí está ese elemento, y sus hijos, del jugador:

    <div class="vjs-dock-text">
  <h1 class="vjs-dock-title">Tiger</h1>
  <h2 class="vjs-dock-description"></h2>
</div>

    Los valores predeterminados para el degradado utilizan la función rgba() (rojo-verde-azul-alfa), como se muestra aquí:

    linear-gradient(180deg,rgba(0,0,0,.8) 25%,transparent 100%);

    Estos valores hacen que el degradado se desvanezca horizontalmente de negro, con un alfa de .8, a transparente. El valor de 25% es un valor de corte, lo que significa que el valor establecido en la rgba función se utilizará para el primer 25% del espacio antes de que comience el desvanecimiento a transparente.

    Para fines de instrucción, ya que no utilizaría este degradado en producción, el siguiente estilo crea el degradado en la captura de pantalla.

    .video-js .vjs-dock-text {
  background: linear-gradient(45deg, rgba(0,0,255,.8) 35%, transparent);
  height: 80%;
}

    En este ejemplo, se aplica un degradado azul con un alfa .8 en un ángulo descendente de 45 grados. El color/alfa establecido en el gradiente lineal se utilizará para el primer 35% del gradiente antes de que comience a desvanecerse. Además, se usa el 80% de la altura del elemento.

    Especificidad CSS

    La especificidad de CSS se refiere a la forma en que los navegadores determinan qué valores de propiedad se aplican a un elemento en función de su relevancia. Para obtener más detalles sobre este concepto, revise el artículo Specificity .

    En Chrome, inspeccione el botón de juego grande. Observe que la propiedad se establece utilizando varias clases.background Esto significa que no podemos usar solo la clase para anular esta propiedad..vjs-big-play-button

    Aquí hay tres enfoques para personalizar el botón de juego grande:

    • Haga referencia a una identificación asignada al jugador.
    • Utilice varias clases para aumentar la relevancia de las propiedades.
    • Utilice JavaScript para establecer el atributo id en el reproductor y luego hacer referencia a él en el CSS.

    Hacer referencia a un identificador

    Dado que un atributo tiene una alta relevancia con respecto a la especificidad de CSS, agregar un al selector CSS anulará la propiedad.ididbackground-color

    Si aún no lo ha hecho, agregue un atributo al elemento de la siguiente manera:idvideo

    <video-js id="video_1"
  data-account="1752604059001"
  data-player="68f30408-8fb4-431d-accb-8c5baa8c4790"
  data-embed="default"
  class="video-js" controls></video-js>

    En el CSS, agregue el valor id al selector de .vjs-big-play-button clase. Anule el color de fondo y la opacidad de la siguiente manera:

    <style>
  #video_1 .vjs-big-play-button {
      background-color: #B37D5B;
      opacity: .6;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
</style>

    Deberías ver que el botón de juego grande ahora es naranja.

    Uso de varias clases

    Dado que la background propiedad para el botón de juego grande se establece usando dos clases, necesitará al menos dos clases en el selector CSS para anular esta propiedad.

    Añade el .video-js clase a la .vjs-big-play-button selector de clases. Recuerde que la clase está configurada en el contenedor de video..video-js

    Con estos valores, estamos diciendo seleccionar todos los elementos con un nombre de clase de los vjs-big-play-button cuales son decedents de elementos con un nombre de clase de .video-js.

    <style>
  .video-js .vjs-big-play-button {
    background-color: #B37D5B;
    opacity: .6;
  }
  .video-js {
    width: 640px;
    height: 360px;
  }
</style>

    Deberías ver que el botón de juego grande ahora es naranja. Trabajar con múltiples clases puede ser complicado y puede afectar a múltiples elementos, mientras que un id es específico de un elemento.

    Utilizando JavaScript

    También puede usar JavaScript para agregar dinámicamente un id al botón de juego grande. Luego puede hacer referencia a ese id desde el CSS.

    Agregue código para agregar un id al elemento del botón de juego grande.

    • Línea 173: Obtiene una referencia al primer elemento que tiene una clase de vjs-big-play-button.
    • Línea 174: Añade un id atributo al botón de juego grande.

    var playButton = document.getElementsByClassName("vjs-big-play-button")[0];

  </script>

    <style>
    #myPlayButton {
      background-color: #B37D5B;
      opacity: .6;
    }
    .video-js {
      width: 640px;
      height: 360px;
    }
  </style>

    Deberías ver que el botón de juego grande ahora es naranja.

    {PLAYER_CLASS}

    El {PLAYER_CLASS} es un selector y será reemplazado por el selector específico de la clase real para el jugador, por ejemplo:

    .bc-player-oH9IdGudo_default

    {PLAYER_CLASS} le ayuda a hacer que su CSS sea lo suficientemente específico como para sobrescribir los estilos incorporados sin tener que hacer CSS para cada jugador individualmente o usar algo como:

    .video-js.video-js

    Por ejemplo, supongamos que desea anular el spinner de carga, puede usar:

    .video-js.video-js .vjs-loading-spinner {...}

    Eso es lo suficientemente específico como para anular el valor predeterminado, pero si tiene un segundo jugador en la página. El uso de lo siguiente NO afectaría a ningún otro jugador de la página:

    .video-js{PLAYER_CLASS} .vjs-loading-spinner {...}

    Tenga en cuenta que el {PLAYER_CLASS} selector solo se puede utilizar en archivos CSS que se agregan a una matriz de hojas de estilo de configuración del reproductor (por ejemplo, a través de Studio o la Player Management API) y se incorporan directamente en el reproductor. Este selector no funcionará con código de implementación avanzada CSS incluido por separado a través de <style></style> o <link></link> etiquetas.
