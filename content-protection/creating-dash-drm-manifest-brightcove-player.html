    Creación de un manifiesto DRM de DASH para el reproductor de Brightcove

    En este tema, aprenderá a crear un manifiesto MPEG-DASH (archivo.MPD) con protección DRM para Brightcove Player.

    Resumen

    El manifiesto MPEG-DASH (archivo.MPD) describe un vídeo con representaciones MPEG-DASH cifradas mediante cifrado común (CENC) y es compatible con los módulos de descifrado de contenido (CDM) Playready, Widevine y Marlin. Solo Playready y Widevine son compatibles con el reproductor web de Brightcove, dependiendo del navegador que se utilice.

    Compatibilidad con navegador

    La reproducción sólo funciona en los navegadores Macintosh y Windows de escritorio compatibles.

    Tipos de explorador Tecnología de reproducción Tipo de DRM
    Cromo 34+ Reproducción nativa MSE/EME con Shaka-Player* Widevine
    IE 11 en Windows 8 Metro, MS Edge en Windows 10 Reproducción nativa MSE/EME con Shaka-Player* Playready
    Todos los demás navegadores, incluidos Safari, IE y Firefox Plugin de Silverlight Playready
    *Shaka-Player proporciona soporte DASH. Antes de la versión 4 del complemento DRM de Brightcove Player, la tecnología de reproducción utilizada para la compatibilidad con DASH se creó sobre Dash.js.

    Contenido DRM externo

    Requisitos para el contenido DRM creado fuera de Brightcove Video Cloud

    Para reproducir contenido de vídeo con protección DRM aplicada mediante el Brightcove Player, el vídeo debe codificarse siguiendo estas directrices.

    1. Debe ser MPEG-DASH Live Profile (no «bajo demanda», no «hbbtv»)
    2. Todas las copias deben tener el mismo intervalo de fotogramas clave y velocidad de fotogramas (para compatibilidad con nuestro plugin Silverlight)
    3. Debe estar encriptado mediante cifrado común (CENC) y tener habilitado el soporte para Playready y Widevine
    4. Deben proporcionarse las URL del servidor de licencias Playready y Widevine. La URL del servidor de licencias Playready debe incluirse en el archivo de manifiesto (.mpd). La URL del servidor de licencias Widevine se puede proporcionar en el manifiesto o como parámetro en Javascript
    5. El vector de inicialización Playready debe ser de 8 bytes.

    Información de licencia

    Para crear su propia secuencia DASH-DRM compatible con Brightcove Player, el sistema de codificación debe crear un manifiesto (MPD) similar al proporcionado anteriormente que incluya información sobre la licencia DRM. Tu MPD puede omitir las secciones Marlin que no utiliza el Brightcove Player en la actualidad.

    Especificación de la clave de licencia PlayReady

    Brightcove Player requiere que la información de licencia PlayReady esté incrustada en el MPD como una cadena codificada en base 64. Puede ver el valor en este ejemplo en la <mspr:pro> etiqueta anterior. La información de clave codificada se muestra descodificada a continuación. Puede encontrar información sobre el formato del encabezado PlayReady aquí: http://www.microsoft.com/playready/documents/.

    <WRMHEADER xmlns="http://schemas.microsoft.com/DRM/2007/03/PlayReadyHeader" version="4.0.0.0"><DATA><PROTECTINFO><KEYLEN>16</KEYLEN><ALGID>AESCTR</ALGID></PROTECTINFO><KID>gK+XXUIk6U+xqvwk7Lb6KA==</KID><CHECKSUM>UdkKbIAxnNg=</CHECKSUM><LA_URL>https://prlic.brightcove.com/license/3723496277001</LA_URL></DATA></WRMHEADER>

    Especificación de la URL de licencia de Widevine

    Observe aquí que la URL del servidor de licencias Widevine se proporciona en el MPD en esta línea:

    <ContentProtection schemeIdUri="urn:uuid:edef8ba9-79d6-4ace-a3c8-27dcd51d21ed" bc:licenseAcquisitionUrl="https://wvlic.brightcove.com/proxy/4324249227001" xmlns:bc="urn:brightcove:2015"/>

    Alternativamente, puede proporcionar la URL del servidor de licencias Widevine como parámetro para el reproductor en JavaScript en la página donde está incrustado. A continuación se muestra un ejemplo de cómo llamar al Brightcove Player especificando un servidor de licencias con la URL de origen:

    myPlayer.src({
  "src": sourceUrl,
  "type": "application/dash+xml",
  "keySystemOptions": [{
    "name": "com.widevine.alpha",
    "options": {"licenseUrl": licenseServerUrl}
  }]
});

    Precarga

    Hay una serie de configuraciones desde las que puede elegir controlar la precarga de datos de video. Consulte la Guía de configuración del reproductor, sección de precarga para obtener más detalles. El comportamiento predeterminado es que Brightcove Player precargará los metadatos de vídeo (por ejemplo, título y duración), junto con varios segundos de datos de vídeo. Si desea cambiar el comportamiento de precarga, puede especificar la siguiente opción en la configuración del reproductor:

    "preload": "none"
