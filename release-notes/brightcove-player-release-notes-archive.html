    Brightcove Player Notas informativas

    Esta página realiza un seguimiento de la información de lanzamiento de Brightcove Player

    Nota: Por defecto, las actualizaciones se aplican automáticamente a todos los reproductores. Para evitar la actualización automática de un reproductor, debe configurarlo en actualización manual (también conocida como bloqueo de un reproductor). Puede hacerlo en el módulo Studio Players (sitio externo) o mediante la API de administración de Player.

    La opción de no recibir actualizaciones automáticas debe ser una medida temporal, aplicada mientras resuelve cualquier problema que cree la actualización. Si no actualiza los jugadores a la versión actual, se está perdiendo nuevas características y correcciones de errores, y las versiones anteriores del reproductor dejarán de ser compatibles en algún momento

    Además, puede ver las versiones de Brightcove Player y los lanzamientos de complementos en el Grupo Google Anuncios de lanzamiento de Brightcove Player.

    VERSIÓN 5 - 5.28.1

    Ya no se admite la versión 5 de Brightcove Player. Los reproductores publicados seguirán funcionando y no se deshabilitarán, pero Brightcove no puede garantizar la compatibilidad futura con navegadores, API, formatos de vídeo, integraciones y otros cambios que puedan ocurrir fuera del reproductor.

    Nuestra recomendación es actualizar a la versión más reciente de la versión 6. Si necesita ayuda para hacerlo, póngase en contacto con el servicio de soporte de Brightcove.

    • El uso de «responsive» en este contexto significa que hemos habilitado la nueva opción responsive Video.js en Brightcove Player de forma predeterminada (mientras que está desactivada de forma predeterminada en Video.js).     Nuevas funciones

    Brightcove Player versión 5 tiene nuevo soporte para vídeo HLS sin usar Flash. En esta versión, en todos los navegadores compatibles con Media Source Extensions API (MSE), Brightcove Player puede reproducir vídeos HLS sin Flash. Esto significa que Flash no se utiliza en Firefox, Chrome, Safari o Microsoft Edge. Flash todavía se usará para versiones anteriores de IE, pero se puede deshabilitar mediante configuración.

    MSE HLS ofrece varios beneficios enormes:

    1. Los vídeos se cargarán rápidamente y se reproducirán sin problemas, incluso con altas resoluciones y velocidades de fotogramas altas, incluyendo vídeo 4K y 60FPS, en hardware modesto .
    2. Utiliza aproximadamente -50% de la carga de la CPU para renderizar los mismos vídeos, lo que significa que las baterías durarán más tiempo y el ventilador de tu portátil no se enciende
    3. Ahorro de energía y otros ajustes del navegador para limitar el uso de Flash ya no tendrán ningún efecto en el reproductor independientemente de las dimensiones del jugador.

    Otras características de esta versión incluyen:

    1. Nueva piel - Basado en importantes investigaciones UX, hemos construido una nueva piel que incorpora muchos cambios. Estos incluyen un nuevo botón de juego grande centrado, nuevos colores, un nuevo comportamiento de la barra de búsqueda y una nueva barra de título llamada el dock. Los participantes de pruebas de usabilidad les encantó especialmente la nueva línea de tiempo que se expande al flotar y hace que sea más fácil buscar a un momento específico. Ahora puedes elegir entre el nuevo aspecto predeterminado, llamado Luna, y el antiguo aspecto 1.x, ahora llamado Grafito.
    2. Compatibilidad inicial con subtítulos en vivo : esta versión ahora soporta muchos sistemas de codificación de subtítulos 608/708. Esto incluye soporte para los populares métodos emergentes y enrollables y el estilo básico
    3. Modo de compatibilidad para todos los plugins 1.x y Video.js 4.x. (excepto los plugins DASH/DRM, FreeWheel y OnCeux - actualizaciones a seguir durante el primer trimestre)

    También en esta versión, hemos realizado muchas actualizaciones para mejorar el modelo de programación para la personalización, incluyendo simplificaciones de nuestro modelo de skinning y la API para plugins. Estos cambios nos ponen al día con nuestra última versión de código abierto, Video.js 5. También se dará cuenta de que también hemos sincronizado nuestra numeración de versiones entre las versiones Video.js y Brightcove Player.
