Introducción

La Incrustación avanzada (in-page) El código le permite tener el reproductor en la página HTML directamente, no en un iframe. Esto ofrece los beneficios de acceder fácilmente al reproductor y a las propiedades y eventos asociados. Al usar el reproductor sin iframe, pierde la facilidad de uso del reproductor como una sola unidad contenida en las aplicaciones de redes sociales y para compartir. Ver el Elegir el código de inserción correcto documento para obtener más información.

Uso

Nota: Si eres cliente de Brightcove Player, no utilizarás el data-video-id atributo para asignar un vídeo a tu reproductor. Puede asignar el video en el Jugadores módulo o usando curl.

Usar el incrustación in-page código siga estos pasos:

  1. Utilizar el JUGADORES módulo para crear un jugador.
  2. Ve a la MEDIOS DE COMUNICACIÓN módulo y publique un video usando el reproductor recién creado.
  3. Copia el Avanzado código de inserción. El HTML será similar al siguiente: 
      <video-js data-video-id="4093372393001"
    data-account="1507807800001"
    data-player="ry3j6rsze"
    data-embed="default"
    data-application-id
    class="video-js"
    controls></video-js>
  <script src="https://players.brightcove.net/1507807800001/ry3j6rsze_default/index.min.js"></script>
  4. Copia el HTML del navegador y pégalo en la body de una página HTML completa.
  5. Navega por la página HTML para ver cómo funciona el reproductor.

Al publicar vídeos mediante el módulo multimedia, puede utilizar la URL del reproductor para obtener una vista previa del vídeo o copiar el iframe o el código de incrustación en la página web para pegarlo en su página web o aplicación.

Nota: Haga clic en el Acortar junto a la URL de vista previa para generar una URL de vista previa abreviada.

Código de inserción en la página (avanzado)

El código de incrustación típico de la página aparecerá de la siguiente manera:

  <video-js data-video-id="5076962725001"
  data-account="1507807800001"
  data-player="BkTGbgSq"
  data-embed="default"
  data-application-id
  class="video-js"
  controls></video-js>
  <script src="https://players.brightcove.net/1507807800001/BkTGbgSq_default/index.min.js"></script>

Aunque la integración del código de publicación en la página puede ser más compleja, utilizar el código de la página es mejor cuando la página que contiene el reproductor necesita comunicarse con el reproductor. Algunos ejemplos de cuándo utilizar el código de incrustación en la página incluyen:

  • El código de la página que contiene necesita escuchar y actuar en función de los eventos de los jugadores
  • El reproductor utiliza estilos de la página principal
  • El código iframe provocará que falle la lógica de la aplicación, como un redireccionamiento desde la página principal

Incluso si su implementación final no utiliza el código de incrustación de iframe, puede seguir utilizando el código In-Page con un complemento para su JavaScript y un archivo separado para su CSS. Esto encapsula tu lógica para que puedas utilizarla fácilmente en varios reproductores.

Recomendación

Se considera una mejor práctica utilizar la implementación de iframe a menos que alguna lógica de aplicación requiera el uso del código In-Page. Si está utilizando el módulo Audiencia para realizar un seguimiento de la participación de los espectadores, debe utilizar el código de inserción In-Page (avanzado).

Atributos

Hay numerosos atributos disponibles para <video> etiqueta para proporcionar información adicional sobre cómo debe comportarse un jugador. La siguiente tabla detalla los atributos disponibles.

Cuando se utilizan atributos de etiqueta que solo pueden ser verdaderos o falsos (tipo de datos Booleano), simplemente incluye el atributo (sin signo igual) para activarlo.

DERECHO: controls

EQUIVOCADO: controls="true"

Para usar booleanos, el atributo debe estar deshabilitado de forma predeterminada en los ajustes de configuración del reproductor, luego se habilita al incluir el atributo en el código de inserción en la página. Por ejemplo, si autoplay está habilitado en la configuración del reproductor, no hay forma de deshabilitar esa función en el código de inserción en la página. Esto significa que si desea controlar un atributo booleano, desea asegurarse de que esté deshabilitado en la configuración del reproductor, entonces puede habilitarlo opcionalmente colocando el atributo apropiado en el código de inserción en la página. Habilitar el atributo en la configuración del reproductor anula la omisión del atributo en el código de inserción en la página.

autoplay Indica que el jugador debe empezar a jugar inmediatamente, en plataformas en las que esté permitido.

Si autoplay se establece en un valor booleano, se utiliza el comportamiento de reproducción automática nativo del navegador. Si se establece en uno de los tres valores de cadena válidos siguientes, se produce lo siguiente:
  • play: El jugador intentará llamar manualmente al juego.
  • muted: El reproductor silenciará manualmente el reproductor y llamará a la reproducción.
  • any: En primer lugar, el jugador intentará llamar al juego. Si esto falla, silencie el reproductor y llame a play.
Si alguno de los anteriores falla, el jugador mostrará el «Botón de juego grande» como si la reproducción automática se estableciera en false.

detalles de precarga

Nota: Actualmente, el valor de precarga establecido en el Configuración de Studio Player siempre anulará el valor establecido en el video atributos de etiqueta, por lo que debe configurarlo en Studio.
Nota: La configuración de precarga es en realidad solo sugerencias en la especificación HTML5 y, como resultado, puede haber un comportamiento diferente en diferentes navegadores. Establecer el valor en sí potencialmente podría tener un impacto significativo en el consumo de ancho de banda.

El atributo de precarga informa al navegador si los datos de vídeo deben comenzar a descargarse en cuanto se carga la etiqueta de vídeo. Las opciones son auto , metadata , y none. Para la tecnología de reproducción HLS, los ajustes significan cosas ligeramente diferentes y se indican a continuación.

  • auto(defecto): Comience a cargar el vídeo inmediatamente (si el navegador está de acuerdo). Algunos dispositivos móviles como iPhones y iPads no cargarán previamente el vídeo para proteger el ancho de' banda de sus usuarios. Es por eso que el valor se llama auto y no algo más final como true.

    Para la tecnología de reproducción HLS, automático significa que el reproductor descarga el archivo de manifiesto m3u8 y solo 1 segmento en lugar de 3-4, como era el caso anterior a la versión 5.3 de Brightcove Player.
  • metadata: Cargue solo los metadatos del vídeo, que incluye información como la duración y las dimensiones del vídeo.
    Para la tecnología de reproducción HLS, los metadatos son los mismos que los automáticos.
  • none: No cargues previamente ninguno de los datos de vídeo. Esto esperará hasta que el usuario haga clic en jugar para comenzar a descargar. Tenga en cuenta que en iOS puede haber descargas de segmentos incluso con esta configuración. Ver el Brightcove Player e iOS documento para obtener más información.
    Para la tecnología de reproducción HLS, ninguno significa que el reproductor solo descargará el manifiesto m3u8 para que se pueda calcular la duración del video.
  <video-js preload ...>
  or
  { "preload": "auto" }

Notas:

  • Al utilizar la inserción de anuncios del lado del servidor (SSAI), Brightcove recomienda configurar preload=none. Si preload está configurado en auto o metadata, los primeros fotogramas del vídeo se solicitará la transmisión, que cuenta el anuncio como entregado incluso si el usuario no inicia la reproducción.
  • Debido a un error en Internet Explorer, la configuración de precarga puede ignorarse al usar IE.
