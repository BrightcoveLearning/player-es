Opciones disponibles para el código de inserción estándar (iframe)

{{ page.description }}

Introducción

La iframe El código de implementación del reproductor proporciona la facilidad de uso del reproductor como una sola unidad contenida. Esto a menudo es necesario para las aplicaciones de redes sociales y para compartir.

Uso

Nota: Si es cliente de Brightcove Player, no utilizará el videoId parámetro de consulta para asignar un video a su reproductor. Puede asignar el video en el Jugadores módulo o usando curl.

Usar el iframe código siga estos pasos:

  1. Utilizar el JUGADORES módulo para crear un jugador.
  2. Ve a la MEDIOS DE COMUNICACIÓN módulo y publique un video usando el reproductor recién creado.
  3. Copia el Estándar código de inserción. El HTML será similar al siguiente: 
          <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
  4. Copia el HTML del navegador y pégalo en la body de una página HTML completa.
  5. Navega por la página HTML para ver cómo funciona el reproductor.

Parámetros de consulta

Hay numerosos parámetros de consulta disponibles para iframe etiqueta para proporcionar información adicional sobre cómo debe comportarse un jugador. La siguiente tabla detalla los parámetros disponibles.

Cuando se utilizan parámetros de consulta que solo pueden ser verdaderos o falsos (tipo de datos Booleano), simplemente incluye el parámetro de consulta para activarlo o excluirlo para desactivarlo. Aquí se muestra un ejemplo usando muted.

<iframe src="https://players...index.html?videoId=4784463159001&muted"

reproducción automática Indica que el jugador debe empezar a jugar inmediatamente, en plataformas en las que esté permitido.

Si autoplay se establece en un valor booleano, se utiliza el comportamiento de reproducción automática nativo del navegador. Si se establece en uno de los tres valores de cadena válidos siguientes, se produce lo siguiente:
  • play: El jugador intentará llamar manualmente al juego.
  • muted: El reproductor silenciará manualmente el reproductor y llamará a la reproducción.
  • any: En primer lugar, el jugador intentará llamar al juego. Si esto falla, silencie el reproductor y llame a play.
Si alguno de los anteriores falla, el jugador mostrará el «Botón de juego grande» como si la reproducción automática se estableciera en false.

Hacer que un vídeo se reproduzca automáticamente puede ser una situación compleja. Ver el Consideraciones sobre la reproducción automática documento para más detalles.

detalles de precarga

El parámetro de precarga informa al navegador si los datos de video deben comenzar a descargarse tan pronto como se cargue la etiqueta de video. Las opciones son auto , metadatos , y ninguno. Para la tecnología de reproducción HLS, los ajustes significan cosas ligeramente diferentes y se indican a continuación.

  • auto (defecto): Comience a cargar el vídeo inmediatamente (si el navegador está de acuerdo). Algunos dispositivos móviles como iPhones y iPads no cargarán previamente el vídeo para proteger el ancho de' banda de sus usuarios. Es por eso que el valor se llama auto y no algo más final como true.

    Para la tecnología de reproducción HLS, automático significa que el reproductor descarga el archivo de manifiesto m3u8 y solo 1 segmento en lugar de 3-4, como era el caso anterior a la versión 5.3 de Brightcove Player.

  • metadatos: Cargue solo los metadatos del vídeo, que incluye información como la duración y las dimensiones del vídeo.

    Para la tecnología de reproducción HLS, los metadatos son los mismos que los automáticos.

  • ninguno: No cargues previamente ninguno de los datos de vídeo. Esto esperará hasta que el usuario haga clic en jugar para comenzar a descargar. Tenga en cuenta que en iOS puede haber descargas de segmentos incluso con esta configuración. Ver el Brightcove Player e iOS documento para obtener más información.

    Para la tecnología de reproducción HLS, ninguno significa que el jugador ya no cargará nada, incluido el manifiesto m3u8, hasta que el usuario presione reproducir.
      <video-js preload ...>
      or
      { "preload": "auto" }
Nota: Al utilizar la inserción de anuncios del lado del servidor (SSAI), Brightcove recomienda configurar preload=none. Si preload está configurado en auto o metadata, los primeros fotogramas del vídeo se solicitará la transmisión, que cuenta el anuncio como entregado incluso si el usuario no inicia la reproducción.
