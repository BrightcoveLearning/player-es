    Validación de etiquetas de anuncio SSAI de Video Cloud

    En este tema, aprenderá a rastrear las llamadas de anuncios especificadas con Inserción de anuncios en el lado del servidor (SSAI).

    Resumen

    Para ayudar con la depuración de anuncios del lado del servidor, la Dynamic Delivery con SSAI proporciona puntos finales de API para realizar el seguimiento de las llamadas publicitarias.

    Notas:
    • Para utilizar SSAI, su cuenta de Video Cloud debe estar configurada para Entrega dinámica y habilitada para SSAI. Póngase en contacto con su administrador de cuentas para empezar a usar esta función.
    • Esta interfaz de depuración es sólo para VOD SSAI.
    • Para obtener una lista de limitaciones, consulte el documento Visión general de SSAI .

    Para depurar anuncios del lado del servidor con el contenido de vídeo almacenado en Video Cloud, siga estos pasos:

    1. Revisar información general para la ruta de acceso y autorización de API
    2. Ejecutar un seguimiento para una configuración de anuncio
    3. Recuperar detalles de seguimiento

    Información general

    La siguiente información se refiere a todas las solicitudes de API SSAI.

    URL base

    La URL base para la API SSAI es:

      https://ssai.api.brightcove.com/v1

    Ruta de la cuenta

    En todos los casos, se solicitarán una cuenta de Video Cloud específica. Por lo tanto, siempre agregará el término seguido de su ID de cuenta a la URL base:accounts

      https://ssai.api.brightcove.com/v1/accounts/your account id

    Autorización

    Las solicitudes de API requieren un encabezado de autorización:

      Authorization: Bearer your access token

    El es un token de acceso temporal de OAuth2 que debe obtenerse del servicio OAuth de Brightcove.access_token Para obtener información detallada sobre cómo obtener credenciales de cliente y usarlas para recuperar tokens de acceso, consulte la descripción general de OAuth de Brightcove.

    Operaciones

    Cuando solicite credenciales de cliente, deberá especificar el tipo de acceso a la cuenta u operaciones que desea. A continuación se muestra una lista de las operaciones actualmente soportadas para la API SSAI:

    • Datos de SSAI:

      video-cloud/ssai/read
      video-cloud/ssai/all

    Ejecutar un seguimiento para una configuración de anuncio

    Para ejecutar un seguimiento de una configuración de anuncio, siga estos pasos:

    Recopilar información

    Recopilar la siguiente información para el cuerpo de su solicitud de API:

    Parámetros Tipo Descripción
    account_id Cadena ID de cuenta de usuario
    playback_config Objeto Los campos se definen en la sección Detalles del campo Configuración del documento de SSAI API.
    title_metadata Objeto Indica al sistema cuánto tiempo dura el contenido para generar la respuesta del anuncio correcta.

    Por ejemplo, lo siguiente indica al sistema que el contenido es de 1 minuto de duración. Se utiliza para insertar anuncios en consecuencia. 
      "title_metadata": {  "duration": "1m" }
    videocloud_metadata Objeto Opcional.
    Sólo es necesario si utiliza variables de plantilla que hacen referencia a campos de metadatos. Estos se definen en la sección Variables de anuncio del documento de SSAI API.

    Solicitud

    Cree un seguimiento para una configuración de anuncio SSAI.

    Método PUBLICAR
    URL https://ssai.api.brightcove.com/v1/accounts/{account_id}/ssai_debug_vmap/debug.xml
    Encabezados Autorización: Bearer access_token (consulte Obtención de tokens de acceso)
    Tipo de contenido: aplicación/json
    Cuerpo de la muestra 
      {
    "playback_config":{
      "name": "config_name",
      "vmap_response_namespace": "config_namespace",
      "account_id": "account_id",
      "ad_config": {
        "enable_ads": true,

    Recuperar detalles de seguimiento

    Una vez que haya ejecutado un seguimiento de anuncios como se muestra en la sección anterior, puede recuperar información detallada sobre el seguimiento utilizando el ID de sesión.

    El session_id especifica la sesión de almacenamiento en caché. Cada sesión tiene su propia duración basada en la duración del contenido de vídeo. Puede obtener el id de la respuesta para la llamada a la API en el paso anterior.

    El ID de sesión se puede encontrar en las respuestas de VMAP de la sección anterior. Un ejemplo:

      <bc:BrightcoveDebug sessionID="your session id"></bc:BrightcoveDebug>

    Solicitud

    Para recuperar la información de seguimiento de la llamada publicitaria proporcionando un identificador de sesión, utilice una solicitud GET similar a la siguiente:

    Método CONSEGUIR
    URL https://ssai.api.brightcove.com/v1/accounts/{account_id}/ssai_traces/{session_id}/ad_calls
    Encabezados Autorización: Bearer access_token (consulte Obtención de tokens de acceso)
    Tipo de contenido: aplicación/json
