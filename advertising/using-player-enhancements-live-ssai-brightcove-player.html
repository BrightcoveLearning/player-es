Uso de mejoras de reproductor para Live SSAI con Brightcove Player

{{ page.description }}

Introducción

Player Enhancements de Brightcove para Live SSAI le permite mejorar las pausas publicitarias en transmisiones en vivo con recuentos de anuncios, temporizadores de cuenta atrás de anuncios, anuncios de clics y anuncios complementarios. Estas transmisiones en vivo se pueden reproducir con Brightcove Player.

Si es nuevo en esta función, consulte lo siguiente:

  • Descripción general: Mejoras del jugador para Live SSAI
  • Implementación de mejoras de jugador para Live SSAI

Pasos

Para reproducir transmisiones en vivo con mejoras del reproductor con Brightcove Player, siga estos pasos:

  1. Cree o elija un reproductor existente de Video Cloud Studio Jugadores módulo.

  2. Brightcove Player debe configurarse con el complemento SSAI. Siga el Agregue el complemento SSAI pasos descritos en el Implementar SSAI programáticamente sección del documento del complemento SSAI.

    Para las mejoras del reproductor con Live SSAI, Brightcove Player requiere el complemento SSAI (videojs-ssai). Actualmente, no puede usar Video Cloud Studio para agregar el complemento SSAI a su reproductor, ya que requiere el ID de configuración del anuncio.
  3. Siga el flujo de trabajo en el Implementación de mejoras de jugador para Live SSAI documento, incluyendo la creación de su token de reproducción.
  4. Elija una implementación:
    • Código incrustado estándar
    • Código incrustado avanzado
    • Fuente SSAI en vivo
  5. Continúe con el flujo de trabajo en el Implementación de mejoras de jugador para Live SSAI documento.

Código incrustado estándar

Al publicar un video de Video Cloud Studio Medios de comunicación módulo, puede utilizar el código de inserción estándar para reproducir su transmisión en vivo. Sigue estos pasos:

  1. Desde el Medios de comunicación módulo, seleccione su video remoto para transmisión en vivo y publíquelo con un reproductor habilitado para SSAI.
  2. Copie el código de inserción estándar y péguelo en su página web.

  3. En el <iframe> etiqueta, debería ver una src atributo. Al final de esto, agregue el adConfigId Parámetro de URL con el valor de su token de reproducción.

    <iframe src="https://players.brightcove.net/6027103979001/IqljF4Olr_default/
index.html?videoId=6089818309001&adConfigId=live.aaB61234aabbccdd567890"
  allowfullscreen=""
  allow="encrypted-media"
  width="960" height="540">
</iframe>
  4. El reproductor está listo para reproducir su transmisión en directo.

Código incrustado avanzado

Al publicar un video de Video Cloud Studio Medios de comunicación módulo, puede utilizar el código de inserción avanzado para reproducir su transmisión en vivo. Sigue estos pasos:

  1. Desde el Medios de comunicación módulo, seleccione su video remoto para transmisión en vivo y publíquelo con un reproductor habilitado para SSAI.
  2. Copia el código de inserción avanzado y pégalo en tu página web.

  3. En el <video-js> etiqueta, agrega una data-ad-config-id atributo con el valor de tu token de reproducción.

    <video-js
  data-account="6027103979001"
  data-player="IqljF4Olr"
  data-embed="default"
  controls=""
  data-video-id="6089818309001"
  data-ad-config-id="live.aaB61234aabbccdd567890"
  data-playlist-id=""
  data-application-id=""
  width="960" height="540">
</video-js>
<script src="https://players.brightcove.net/6027103979001/IqljF4Olr_default/index.min.js"></script>
  4. El reproductor está listo para reproducir su transmisión en directo.

Fuente SSAI en vivo

A diferencia de las fuentes VOD SSAI, las fuentes Live SSAI no incluyen un VMAP inicial. Esto se debe a que las transmisiones en vivo de SSAI no saben de antemano cuándo se incluirán las pausas publicitarias. En cambio, las pausas publicitarias se activan a partir de puntos de referencia que agrega al trabajo en vivo.

Las transmisiones de Brightcove Live SSAI comienzan con una URL de manifiesto HLS estándar con un esquema de nomenclatura específico:

https://.../playlist_ssaiM.m3u8

El complemento SSAI busca el _ssaiM.m3u8 sufijo y el tipo normal HLS MIME

Al publicar un video de Video Cloud Studio Medios de comunicación módulo, puede reproducir una transmisión SSAI en vivo de la siguiente manera:

  1. Desde el Medios de comunicación módulo, seleccione su video remoto para transmisión en vivo y publíquelo con un reproductor habilitado para SSAI.
  2. Copia el código de inserción avanzado y pégalo en tu página web.

  3. Al código de su página, agregue el siguiente código:

    <video-js id="myPlayerID"
  data-account="6027103979001"
  data-player="IqljF4Olr"
  data-embed="default"
  controls=""
  data-video-id=""
  data-playlist-id=""
  data-application-id=""
  width="960" height="540">
</video-js>
<script src="https://players.brightcove.net/6027103979001/IqljF4Olr_default/index.min.js"></script>
<script>
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this;
    myPlayer.src({
      type: 'application/x-mpegURL',
      src: 'https://path/to/playlist_ssaiM.m3u8'
    })
  });
</script>
  4. El reproductor está listo para reproducir su transmisión en directo.
