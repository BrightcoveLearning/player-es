    AMP Uso de un vídeo externo

    En este tema, aprenderá cómo se pueden utilizar las páginas móviles aceleradas (AMP) con Brightcove Player y un vídeo externo que no sea de Video Cloud.
    Este documento utiliza AMP con Brightcove Player y un vídeo NO ingerido en Video Cloud. Si es cliente de Video Cloud, consulte AMP Using a Video Cloud Video.

    Resumen

    Accelerated Mobile Pages es un proyecto de Google que tiene como objetivo permitir «la creación de sitios web y anuncios que sean consistentemente rápidos, hermosos y de alto rendimiento en todos los dispositivos y plataformas de distribución». Puede utilizar Brightcove Player con AMP porque el proyecto incluye un componente que permite a los editores incrustar reproductores de Brightcove en documentos HTML de AMP.amp-brightcove En este ejemplo se utiliza un complemento personalizado que acepta una dirección URL de un vídeo externo (que no sea en la nube de vídeo) que se le pasa a través del amp-brightcove componente utilizando el data-param-video-url atributo personalizado.

    Ejemplo

    A continuación se muestra una página HTML de AMP de ejemplo que incluye un reproductor de Brightcove. El reproductor funciona normalmente, por lo que el HTML es la parte interesante de esta muestra. El código HTML se examina más adelante en este documento.

    Vea el vídeo externo Pen AMP-BrightCove Player-External de Brightcove Learning Services (@rcrooks1969 ) en CodePen.

    Tenga en cuenta que debido a la forma en que funciona AMP, debe considerar cuidadosamente sus limitaciones antes de usarlo. Por ejemplo:
    • AMP tiene reglas muy estrictas sobre lo que HTML puede y no se puede usar en una página. Los detalles sobre esto se pueden ver en la especificación HTML AMP.
    • Brightcove Analytics no informará correctamente de la página de referencia para las páginas AMP; en su lugar, la página referrer_domain estará en blanco y se source_type registrará como direct.
    • No todos los atributos utilizados con la implementación avanzada de páginas de Brightcove Player son válidos con amp-brightcove.
    • AMP no funcionará con reproductores restringidos de dominio.

    Recursos de AMP

    El proyecto AMP desarrolló un componente especial que muestra el reproductor de Brightcove.amp-brightcove El componente se detalla en el documento amp-brightcove.

    AMP proporciona un ejemplo de amp-brightcove implementación con detalles en los siguientes documentos:

    • amp-brightcove
    • amplificador por ejemplo
    • Parque infantil AMP

    Configuración del reproductor

    Brightcove proporciona un complemento para mejorar el uso de AMP con Brightcove Player. El plugin añade soporte para API de interfaz de vídeo de AMP que permite algunas características avanzadas:

    • Integración con amp-analytics: Permite el seguimiento de las vistas a análisis de terceros en relación con el dominio de la página AMP.
    • Integración con amp-bind: La reproducción puede ser controlada por otros elementos de la página AMP.

    Los jugadores sin el complemento todavía funcionan en AMP, pero sin esa funcionalidad.

    Instalación del plugin

    Al igual que con todos los complementos de Brightcove Player, necesita el nombre y la URL del complemento para el JavaScript del plugin, que se proporcionan aquí (no es necesario ningún archivo CSS para el uso de este plugin):

    Nombre del complemento

      ampSupport

    URL JavaScript

      //players.brightcove.net/videojs-amp-support/1/videojs-amp-support.js

    Examen del código del complemento

    La forma en que funciona AMP es que el amp-brightcove componente crea una implementación iframe del Brightcove Player. En la URL del iframe será el atributo personalizado, como se muestra aquí:

      <iframe frameborder="0" allowfullscreen="true" src="https://players.brightcove.net/3676484087001/ByrhJWAPf_default/index.html?videoUrl=%2F%2Fsolutions.brightcove.com%2Fbcls%2Fvideos%2Fcalm-day-oregon-coast.mp4"; class="i-amphtml-fill-content" kwframeid="1"></iframe>

    Si se desplaza por el código iframe, verá la cadena de consulta que se muestra aquí:

    Observe que el nombre de la cadena de consulta es videoUrl. También tenga en cuenta que la URL ha sido codificada por URL. Ambos problemas se reflejarán en el código del complemento, que se muestra aquí:

    • Líneas 21-38: Función auxiliar que encuentra el valor asociado a una cadena de consulta, que se pasa a la función como parámetro.
    • Línea 7: Recupere el valor de la cadena de videoUrl consulta utilizando la función auxiliar.
    • Línea 8: Utilice el decodeURIComponent() método de JavaScript para obtener una URL válida para el vídeo.
    • Líneas 11-14: Utilice la URL decodificada y coloque el vídeo en el reproductor. Si va a utilizar formatos de vídeo que no sean MP4, necesitará pasar el type valor correcto, o escribir lógica para leer la extensión del vídeo y asignar el valor correcto.
    • Líneas 17-19: Por supuesto que no es necesario, pero si desea que el video se reproduzca al inicio, puede usar este código.
      videojs.registerPlugin('ampPassVideo', function() {
    var myPlayer = this,
      encodedVideoQueryParam,
      decodedVideoQueryParam;
  
    // Get the query string and un-URL encode it
    encodedVideoQueryParam = getQuerystring('videoUrl');
    decodedVideoQueryParam = decodeURIComponent(encodedVideoQueryParam);
  
    // Load the video into the player
    myPlayer.src({
      'type': 'video/mp4',
      'src': decodedVideoQueryParam
    });
  
    // On loadedmetadata event, play the video
    myPlayer.on('loadedmetadata', function(){
      myPlayer.play();
    });
  
    /*
     * usage: foo = getQuerystring("bctid", null);
     * foo will be equal to value of query param bctid
     * note that the default_ parameter is what you will get back
     * if the key isn’t found
     */
    function getQuerystring(key, default_) {
      var regex, qs;
      if (default_ == null) default_ = "";
      key = key.replace(/[\[]/, "\\\[").replace(/[\]]/, "\\\]");
      regex = new RegExp("[\\?&]" + key + "=([^&#]*)");
      qs = regex.exec(window.location.href);
      if (qs === null) {
        return default_;
      } else {
        return qs[1];
      }
    }
  });

    Funciones

    Aunque se discute en la documentación de AMP, aquí se destacan algunas características que son de especial importancia:

    Parámetros personalizados

    Es posible que desee pasar información adicional a su reproductor para que sus complementos accedan. Puede hacerlo agregando atributos adicionales en el elemento que se debe nombrar, donde es el nombre de su propiedad.amp-brightcovedata-param-**

    Los parámetros se pasan como nombres de atributos con casquillas de camel anexados a la URL del reproductor. Las claves y los valores están codificados por URL. Por ejemplo:

    • data-param-language="de" se convierte en &language=de
    • data-param-ad-vars="key:val;key2:val2" se convierte en &adVars=key%3Aval%3Bkey2%3Aval2

    Soporte de referencia externo

    Brightcove Player v6.25.0+ admite la configuración de un referrer arbitrario en un iframe. El componente AMP ahora admite pasar a través de su propio referrer al jugador, añadiendo un atributo su amp-brightcove incrustar.referrer="EXTERNAL_REFERRER"

    EXTERNAL_REFERRER es la macro propia de AMP; consulte https://github.com/ampproject/amphtml/blob/19135a3aac7813a9ff263a9f863fe35aeb316582/spec/amp-var-substitutions.md#external-referrer para obtener más información.

    Análisis AMP

    Un componente de amp-analytics se puede utilizar en un documento AMP para realizar un seguimiento de datos a cualquier análisis arbitrario. Si los editores desean incluir eventos de vídeo, ahora pueden hacerlo.

    El documento de AMP sobre la sintaxis de AMP Analytics (que es engorroso) está aquí: https://www.ampproject.org/docs/reference/components/amp-analytics. Y específicamente analítica de vídeo: https://github.com/ampproject/amphtml/blob/master/extensions/amp-analytics/amp-video-analytics.md.

    Hay un ejemplo disponible para realizar un seguimiento a example.com para que veas las balizas como errores en la pestaña Red: http://solutions.brightcove.com/bclifford/ampexamples/analytics.html.

    Vincular AMP

    AMP Bind es la API de AMP para interactuar entre componentes. En un nivel básico, le permite tener algo externo al reproductor para controlar la reproducción. Por ejemplo, el siguiente código reproduciría un vídeo, donde myPlayer es el id del reproductor amp-brightcove.

        <button on="tap:myPlayer.play">Play</button>

    AMP no confía en eventos de vídeo, por lo que actualmente no puede hacer algo más útil, como actuar en un final de vídeo. Un ejemplo de esto está disponible aquí: http://solutions.brightcove.com/bclifford/ampexamples/bind.html.

    Publicidad

    Publicidad incluyendo pre-olls funciona en jugadores utilizados en AMP.

    El plugin publicitario y su configuración sólo tienen que incluirse en la configuración del reproductor. Si necesita incluir valores específicos del artículo en las llamadas al servidor de anuncios, puede pasar datos personalizados a un complemento de reproductor como se describe a continuación.

    Como se mencionó anteriormente, AMP es muy estricto en lo que se puede agregar a una página AMP válida. Por ejemplo, NO PUEDE agregar un id a la amp-brightcove etiqueta y luego usar un script bloque para configurar, por ejemplo, el complemento IMA3. Para reiterar lo que se dijo en el párrafo anterior, la publicidad debe implementarse en la configuración del reproductor, ya sea utilizando Studio o la API de administración de Player.
