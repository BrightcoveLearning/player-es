    Catálogo de jugadores

    En este tema, aprenderá a utilizar la biblioteca de catalog de métodos de Brightcove Player para realizar solicitudes al catalog de Video Cloud. La biblioteca del catálogo facilita la información sobre los medios de Video Cloud y cargarlos en un reproductor.

    El contenido de este documento sólo se aplica a Video Cloud.

    Sintaxis básica

    La sintaxis básica para usar el catalog es la siguiente:

    • Métodos: playerName.catalog.methodName()

    Varios de los catalog métodos pasarán un objeto de parámetros que definen la solicitud. El objeto decatalog parámetros se define en la siguiente sección, y se hará referencia en las descripciones de los métodos, cuando corresponda.

    Nota: En todo el contenido de esta página, cuando un parámetro está rodeado de corchetes ([ ]) indica que el parámetro es opcional.

    catalog parameters (objeto)

    En la tabla siguiente se describen las propiedades del CatalogParams objeto. Este true objeto (no un constructor) define el formato de los parámetros que se utilizarán con muchas solicitudes de catalog.

    Nombre Tipo Descripción
    type string

    El tipo de solicitud que se va a realizar. Debe ser uno de vídeo, lista de reproducción o búsqueda.
    policyKey string

    La clave de directiva de esta cuenta. De forma predeterminada, será la clave de política del reproductor.** Tenga en cuenta que si pasa un ID de cuenta diferente como parámetro, también debe pasar la clave de directiva correspondiente para esa cuenta.
    deliveryConfigId string

    El ID de las reglas de entrega. Consulte Visión General: Reglas de entrega.
    [id] string

    Un ID de vídeo o lista de reproducción o un ID de referencia con el prefijo «ref:». Required para obtener solicitudes de vídeo y listas de reproducción. DEPRECATED: Para la compatibilidad con versiones anteriores, esto se admite como una consulta de búsqueda en lugar de q. Esto se puede proporcionar como un sub-objeto, que también es un CatalogParams objeto, que se puede usar para proporcionar parámetros.
    [q] string

    Una consulta de búsqueda. Required para solicitudes de búsqueda, ignoradas para otros.
    [adConfigId] string

    Un ID de configuración de anuncio SSAI de vídeo Podría.
    [tveToken] string

    Un token TVE opcional que se enviará como parámetro de cadena de consulta.
    [limit] string

    Sólo admite y tipos.playlistsearch Limite el número de vídeos devueltos.
    [offset] string

    Sólo admite y tipos.playlistsearch El número de vídeos que se deben omitir.
    [sort] string

    Solo se admite search tipo. Cómo se deben ordenar los vídeos para búsquedas.
    [bcovAuthToken] string Para su uso con el Playback Authorization Service (PAS) de Brightcove.

    **Nota: De forma predeterminada, la clave de política generada con un reproductor NO está habilitada para la búsqueda. Esto significa que debe (elegir uno):

    • Utilice el policyKey parámetro para pasar una clave de política habilitada para búsquedas.

    • Utilice la API de administración del reproductor para sobrescribir la clave de directiva predeterminada del reproductor. Puede utilizar una instrucción curl similar a la siguiente para hacerlo:

      curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
    "video_cloud": {
      "policy_key": "YOUR SEARCH ENABLED POLICY KEY"
    }
  }' \
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

      Asegúrese de tener en cuenta cualquier problema de seguridad al asociar una clave de política habilitada para búsquedas con el reproductor.

      Para obtener información sobre cómo crear una clave de directiva habilitada para búsquedas, consulte la descripción general: Policy API - sección Buscar vídeos .

    getVideo( ) método

    El getVideo(params | videoID,callback,[adConfigId]) => XMLHttpRequest método realiza una catalog solicitud para el video con el id especificado e invoca una devolución de llamada cuando se completa la solicitud.

    Parámetros

    • params | videoID: Un objeto de catalog parámetros de tipo O una cadena que contiene el ID de vídeo o un identificador de referencia con el prefijo ref:.

    • callback: Función que se llamará cuando se complete la solicitud, ya sea que sea exitosa o no. Se invoca con dos argumentos:

      • El primer argumento contendrá un objeto de error, si se produce un error. Este objeto contiene detalles del error. Si no se produce ningún error, este objeto es null.

      • Si no se produce ningún error, el segundo argumento es el objeto de vídeo solicitado. Si se produce un error, este valor contiene la cadena vacía.

    • [adConfigId]: DESAPROBADO Un ID de configuración de anuncio de Video Cloud que se pasará a la API de reproducción. En su lugar, pase adConfigID en el objeto params inicial.

    Valor devuelto

    La llamada en sí devuelve un objeto de tipo.catalogXMLHttpRequest A continuación se muestra un ejemplo de este objeto:

    Aunque la llamada devuelve un XMLHttpRequest objeto, lo más probable es que nunca tratará con ese objeto. La razón es que en la función de devolución de llamada la respuesta JSON se analiza automáticamente y un objeto utilizable es lo que se pasa a la función de devolución de llamada, en este caso un objeto de video.

    Ejemplo: ID de vídeo como cadena

    En el ejemplo siguiente se muestra un uso del getVideo() método:

    <video-js id="myPlayerID"

  data-player="3bc25a34-086a-4060-8745-dd87af3d53b4"
  data-embed="default"
  data-video-id=""
  class="video-js" controls></video-js>

    Ejemplo: Uso de CatalogParams

    El equivalente del código justo arriba, pero el uso del objeto CatalogParams aparece de la siguiente manera:

    var myPlayer = this,
      catalogParams = {};
    catalogParams.type = 'video';
    catalogParams.id = '2114345471001';
    myPlayer.catalog.getVideo(catalogParams, function(error, video){
    //deal with error
    myPlayer.catalog.load(video);
  });
});
</script>

    getPlaylist( ) método

    El getPlaylist(params | playlistID,callback,[adConfigId]) => XMLHttpRequest método realiza una catalog solicitud para la lista de reproducción con el id especificado e invoca una devolución de llamada cuando se completa la solicitud.

    Parámetros

    • params | playlistID: Un objeto de catalog parámetros de tipo O una cadena que contiene el ID de la lista de reproducción o un identificador de referencia con el prefijo ref:.

    • callback: Función que se llamará cuando se complete la solicitud, ya sea que sea exitosa o no. Se invoca con dos argumentos:

      • El primer argumento contendrá un objeto de error, si se produce un error. Este objeto contiene detalles del error. Si no se produce ningún error, este objeto es null.

      • Si no se produce ningún error, el segundo argumento es el objeto de vídeo solicitado. Si se produce un error, este valor contiene la cadena vacía.

    • [adConfigId]: DESAPROBADO Un ID de configuración de anuncio de Video Cloud que se pasará a la API de reproducción. En su lugar, pase adConfigID en el objeto params inicial.

    Valor devuelto

    La catalog llamada en sí devuelve un XMLHttpRequest. Aunque la llamada devuelve un XMLHttpRequest objeto, lo más probable es que nunca tratará con ese objeto. La razón es que en la función de devolución de llamada la respuesta JSON se analiza automáticamente y un objeto utilizable es lo que se pasa a la función de devolución de llamada, en este caso un objeto de lista de reproducción.

    Ejemplo

    En el siguiente ejemplo se muestra un uso del getPlaylist() método siguiente:

    <video-js id="myPlayerID"
  data-account="1752604059001"
  data-player="f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe"
  data-embed="default"
  class="video-js" controls></video-js>

<script src="https://players.brightcove.net/1752604059001/f50a2d3c-af51-4d8c-84e3-0c7cdec0edbe_default/index.min.js"></script>

<script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this;
    myPlayer.catalog.getPlaylist('1754200320001', function(error, playlist){
      //deal with error
      myPlayer.catalog.load(playlist);
    })
  });
</script>
    Si desea cargar la lista de reproducción, pero NO tener el primer vídeo de la lista de reproducción colocado en el reproductor, en el método puede establecer un segundo parámetro en.playlist()-1 Por ejemplo, el siguiente código recupera una lista de reproducción, la coloca en el reproductor, pero no carga el primer vídeo en el reproductor: 
    myPlayer.catalog.getPlaylist(playlistID, function(error, myPlaylist) {
  // deal with error
  // +++ Place playlist in player, but -1 parameter does not load playlist video in player +++
  myPlayer.playlist(myPlaylist, -1);
});
    Por supuesto, entonces tiene que cargar el video manualmente. Consulte el ejemplo del reproductor de Brightcove: Reproducir vídeo antes de la lista de reproducción para un ejemplo de esta técnica.

    load (), método

    El load(mediaobject) método carga un objeto de vídeo o lista de reproducción en el reproductor. Este método actualizará la mediainfo propiedad del reproductor y actualizará la fuente de vídeo y el póster. En la mayoría de los casos, llamaría a este método con el resultado de una llamada a getVideo() o getPlaylist(). Si desea recibir una notificación cuando se mediainfo haya actualizado, escuche el loadstart evento.

    Parámetros

    • mediaobject: Un objeto de vídeo o lista de reproducción que se va a cargar. Este objeto debe tener el mismo formato que el objeto de respuesta de una llamada a getVideo() o getPlaylist()método.

    get (), método

    El get(params, [callback]) ⇒ Promise | XMLHttpRequest método permite buscar el catalog uso de CMS/Playback API: Sintaxis de búsqueda de vídeos. Bajo el capó, este método utiliza la API de reproducción.

    Parámetros

    • params: Objeto con catalog parámetrosde tipo.

    • callback: Función que se llamará cuando se complete la solicitud, ya sea que sea exitosa o no. Se invoca con dos argumentos:

      • El primer argumento contendrá un objeto de error, si se produce un error. Este objeto contiene detalles del error. Si no se produce ningún error, este objeto es null.

      • Si no se produce ningún error, el segundo argumento es el objeto de vídeo solicitado. Si se produce un error, este valor contiene la cadena vacía.

    Valor devuelto

    El método devuelve un Promise o un objeto de tipo XMLHttpRequest. Si está utilizando una función de devolución de llamada, la respuesta JSON se analiza automáticamente y un objeto utilizable es lo que se pasa a la función de devolución de llamada. La estructura exacta de los datos devueltos en la devolución de llamada variará dependiendo de lo que se haya buscado.

    Ejemplo de devolución de llamada

    El siguiente ejemplo hace lo siguiente:

    1. Construye un objeto CatalogParams que se pasará como argumento con la función. La búsqueda consultará vídeos con la cadena Desert en el vídeo name.

    2. Realiza la búsqueda utilizando el get() método. Tenga en cuenta que la función de devolución de llamada utilizará los parámetros nombrados y.errorObjvideosReturned

    3. Utilice el método para asignar los vídeos devueltos al Reproductor de Brightcove.playlist()

    <script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      catalogParams = {};
    catalogParams.type = 'search';
    catalogParams.policyKey = 'BCpkADawqM3dv_...Cv8-nlTX';
    catalogParams.q = 'name:Desert';
    console.log('catalogParams',catalogParams);
    myPlayer.catalog.get(catalogParams,function(errorObj,videosReturned){
      console.log('videosReturned',videosReturned);
      myPlayer.playlist(videosReturned);
    });
  });
</script>

    La siguiente captura de pantalla muestra los dos resultados en el código anterior.console.log() El catalogParams muestra los parámetros de la búsqueda y la videosReturned matriz muestra los tres vídeos devueltos por la búsqueda.

    Ejemplo de promesa

    En este código la misma búsqueda se realiza utilizando el get() método que se muestra justo arriba, pero el resultado se maneja como una Promesa.

    <script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      catalogParams = {};
    catalogParams.type = 'search';
    catalogParams.policyKey = 'BCpkADawqM3dv_...Cv8-nlTX';
    catalogParams.q = 'name:Desert';
    myPlayer.catalog.get(catalogParams).then(function(videosReturned){
      console.log('videosReturned',videosReturned);
      myPlayer.playlist(videosReturned);
    }).catch(function(errorObj){
      console.log('errorObj',errorObj);
    })
  });
</script>

    Para obtener más información sobre Promesas, consulte Uso de promesas de JavaScript.

La estructura exacta de los datos devueltos en la devolución de llamada variará dependiendo de lo que se haya buscado.

    GetSequence (), método

    El getSequence(sequences, callback, [adConfigId]) ⇒ Array.<XMLHttpRequest> método permite pasar varias catalog operaciones y recupera todos los objetos de vídeo de esas catalog operaciones múltiples. Puede tener tres tipos de operaciones, las cuales son:

    1. buscar
    2. video
    3. lista de reproducción

    En términos prácticos, esto significa que puede recuperar las siguientes combinaciones y los datos devueltos son una matriz de objetos de vídeo:

    • Múltiples listas de reproducción
    • Una lista de reproducción y varios vídeos individuales adicionales
    • Usando la búsqueda, todos los vídeos con una determinada cadena en el nombre, descripción o etiqueta.

    Por supuesto, las secuencias que acaban de enumerar son sólo unas pocas de un número casi infinito de combinaciones.

    Parámetros

    • sequences: Un objeto de catalog parámetros de tipo O una matriz de objetos que representa una secuencia de catalog operaciones.

    • callback: Función que se llamará cuando se complete la solicitud, ya sea que sea exitosa o no. Se invoca con dos argumentos:

      • El primer argumento contendrá un objeto de error, si se produce un error. Este objeto contiene detalles del error. Si no se produce ningún error, este objeto es null.

      • Si no se produce ningún error, el segundo argumento es el objeto de vídeo solicitado. Si se produce un error, este valor contiene la cadena vacía.

    • [adConfigId]: Un ID de configuración de anuncio que se transmitirá junto con cada solicitud.

    Tenga en cuenta que si se devuelve un vídeo específico en más de una de las operaciones, estará en el resultado varias veces.

    Valor devuelto

    La llamada en sí devuelve un objeto de tipo.catalogXMLHttpRequest Aunque la llamada devuelve un XMLHttpRequest objeto, lo más probable es que nunca tratará con ese objeto. La razón es que en la función de devolución de llamada la respuesta JSON se analiza automáticamente y un objeto utilizable es lo que se pasa a la función de devolución de llamada. La estructura exacta de los datos devueltos en la devolución de llamada variará dependiendo de lo que se haya buscado.

    Ejemplo

    En el ejemplo siguiente se utiliza una secuencia de tres catalog operaciones. El primero es una búsqueda de los 10 vídeos más antiguos del catalog, el segundo se añade un vídeo individual y, por último, los vídeos de una lista de reproducción se agregan al resultado.

    mySequences[1]={};
    mySequences[2]={};

    // Search will retrieve 10 oldest videos in catalog

    getLazySequence () (método)

    El getLazySequence(sequences, callback, [adConfigId]) ⇒ XMLHttpRequest método se comporta del mismo modo que el método getSequence () , excepto que se emplea la carga perezosa de vídeos. Esto significa que cuando se recuperan los vídeos de una operación de secuencia, los vídeos de la siguiente operación de secuencia no se recuperarán hasta que se alcance el último vídeo de la operación anterior. Existe un caso especial si una secuencia devuelve sólo un solo vídeo, en cuyo caso se realiza la siguiente operación de secuencia y se recuperan más vídeos.

    AutoFindAndLoadMedia () (método)

    El autoFindAndLoadMedia([opts]) ⇒ Object | undefined método busca el último ID de vídeo o lista de reproducción definido, lo solicita desde la API y lo carga en el reproductor. Este método define el comportamiento predeterminado para el Brightcove Player.

    Parámetros

    • options: Se reconocen cualquiera de las siguientes opciones:

      • adConfigId ( número): Valor numérico de una configuración de anuncio que se pasará a la API de reproducción. Se llamará ad_config_id al final.

      • embedHasPoster ( booleano): Si el incrustado original tenía o no un póster.

      • embedHasSources ( booleano): Ya sea que el incrustado original tuviera o no fuentes.

      • embedHasTracks ( booleano): Si el incrustado original tenía o no pistas.

      • playlistId ( cadena): Un ID de lista de reproducción proporcionado en la configuración del reproductor.

      • search ( cadena | Objeto): Una cadena de búsqueda/objeto que se proporcionó en la configuración del reproductor o incrustar. Consulte el método para obtener más información sobre los parámetros admitidos.get()

      • sequence ( Objeto | Array.Object): Una cadena de búsqueda/objeto que se proporcionó en la configuración del reproductor o incrustar. Consulte el método para obtener más información sobre los parámetros admitidos.get()

      • videoId ( cadena): Un ID de vídeo proporcionado en la configuración del reproductor.

    TransformVideoResponse (), método

    El transformVideoResponse(data) método convierte un objeto de vídeo sin procesar de la CMS API o de la Playback API en un formato compatible con el reproductor y los navegadores. Esto incluye tiempos de duración y compatibilidad de imágenes https. También reorganiza las fuentes para asegurarse de que las fuentes que se espera que proporcionen la experiencia de visualización de la más alta calidad en esta plataforma se intentan primero.

    Al usar el catalog.getVideo() o catalog.getPlaylist() métodos, el transformVideoResponse() se llama automáticamente, de modo que los datos devueltos estén en el formato apropiado.

    Parámetros

    • data: Objeto de vídeo sin procesar analizado a partir de una respuesta catalog API.

    A continuación se muestra un ejemplo de uso del método:

     for (var i=0; i<limitValue; i++) {

    catalog errores

    El catalog arrojará errores cuando haya problemas para recuperar o reproducir inicialmente el vídeo o la lista de reproducción. Los errores se detallan a continuación.

    catalog Error Descripción
    Video ID as string El reproductor se ha configurado con un ID de cuenta de Video Cloud no válido.
    VIDEO_CLOUD_ERR_RESOURCE_NOT_FOUND El ID predeterminado de la lista de reproducción no es válido.
    VIDEO_CLOUD_ERR_VIDEO_NOT_FOUND El vídeo predeterminado no es válido.
    VIDEO_CLOUD_ERR_NOT_PLAYABLE El vídeo de Video Cloud no se puede reproducir. Las causas frecuentes de este error incluyen cuando se desactiva un ID de vídeo o cuando la disponibilidad programada ha caducado.

    Mango bc-catalog-error

    Es posible que el manejo de errores en la ready() sección normal del script bloque pueda causar problemas. Por ejemplo, puede suceder que el bc-catalog-error evento se pueda enviar antes de que el jugador esté listo, y si escucha el error en la ready() sección, no podrá manejar el error. Este problema puede producirse cuando se utiliza el filtrado geográfico, un vídeo no se publica, un vídeo está fuera del rango de programación o en una cuenta diferente. Es posible que encuentre que no hay un problema en su código, pero el problema puede depender del navegador, así que prueba a fondo.

    Si esta es una posibilidad en su implementación, su código de manejo de errores debe usar el método de manejo de one() eventos para escuchar el bc-catalog-error en un bloque de código separado en lugar de dentro del ready() sección.

    Uso de error personalizado

    La mejor solución para manejar un específico bc-catalog-error tal vez usando un error personalizado. El siguiente código muestra un complemento que utiliza un error personalizado para notificar al espectador de un vídeo inactivo por qué el vídeo no se reproducirá:

    videojs.registerPlugin('inactiveErrorCheck', function(options) {
  var myPlayer = this;
  myPlayer.one('bc-catalog-error', function(){
    var specificError;
    myPlayer.errors({
      'errors': {
        'inactive-error': {
          'headline': options.inactiveMessage,
          'dismiss': false
        }
      }
    });
    if (typeof(myPlayer.catalog.error) !== 'undefined') {
      specificError = myPlayer.catalog.error.data[0];
      if (specificError !== 'undefined' & specificError.error_code == "VIDEO_NOT_PLAYABLE") {
        myPlayer.error({code:'inactive-error'});
      };
    };
  });
});
