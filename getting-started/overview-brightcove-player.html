Descripción general: Jugador Brightcove

Fortalezas

El reproductor proporciona una plataforma sobre la que construir una experiencia de reproducción de video flexible y extensible.

Las fortalezas clave son:

  • Reproductor de video HTML5 ligero
  • Entorno de complementos enriquecido para agregar características y funcionalidad
  • Uso de HTML, JavaScript y CSS para personalizar la apariencia del reproductor
  • Implementaciones de dos jugadores para diferentes entornos
  • Amplio conjunto de funciones como puntos de referencia, subtítulos, localización, publicidad, etc.
  • SDK de reproductor nativo
  • Reproduce videos de la biblioteca de Video Cloud y videos con URL accesibles a través de Internet
Este sitio en el que está viendo este documento, https://player.support.brightcove.com , es para el web versión de Brightcove Player. Puede encontrar información sobre las versiones SDK de Brightcove Player en https://sdks.support.brightcove.com/. También tenga en cuenta que los cambios en la versión web, como el estilo o las configuraciones especiales, NO se realizan automáticamente en las versiones del SDK y viceversa.

Publicación de vídeos

Cuando crea un reproductor, ve tres implementaciones para publicar el reproductor. Estas implementaciones son:

  • URL
  • Incrustación en la página
  • iframe

Cada una de estas implementaciones puede adoptar dos enfoques. El enfoque más común es utilizar un reproductor dinámico. Este reproductor puede reproducir muchos videos diferentes. Para lograr esto, la identificación del video se pasa al reproductor en tiempo de ejecución, a través de un parámetro de URL, un atributo de etiqueta de video o API de JavaScript. Esto es técnicamente similar a una página web de plantilla que se puede usar para muchas instancias diferentes de contenido. Un caso de uso de este enfoque es un reproductor de video genérico al que se le asignan diferentes videos según el artículo o video al que acceda el usuario.

Un enfoque más especializado es vincular estáticamente el video al reproductor incluyéndolo en la configuración del reproductor. Este enfoque es similar a la creación de una página web estática. La ventaja es reducir la cantidad de llamadas que debe hacer el jugador para mostrar una imagen de póster o comenzar a reproducir el video, aumentando así el rendimiento. Esto se puede hacer en Studio o usando la API directamente. Un caso de uso de este enfoque es una página de producto creada con un reproductor que contiene un video que detalla el producto individual.

Nota: Aunque es posible cambiar el video vinculado estáticamente a un reproductor, no es una buena práctica. Si haces esto, Es posible que vea el póster del video enlazado estáticamente aparecer brevemente en el reproductor antes de que se muestre el video agregado dinámicamente. . Por esta razón, es una buena práctica crear un reproductor que no tenga un video asociado inicialmente para su uso en una situación en la que el video cambiará dinámicamente.

Gestión de jugadores

 

Los reproductores se pueden crear, personalizar y administrar a través de Studio o mediante programación con Player Management API.

Estudio

Studio proporciona una interfaz fácil de usar para realizar estas tareas.

La Introducción a Studio El documento proporciona un excelente punto de partida para aprender sobre Video Cloud Studio.

API de administración de reproductores

Las características de la API de administración de jugadores se dividen en dos categorías principales. Estas categorías y fortalezas detalladas se enumeran aquí:

  • Gestión centralizada
    • Actualizaciones de versión automáticas
    • Crea, actualiza y activa jugadores a través de API
    • Actualiza miles de jugadores a la vez
    • Control de versiones y reversión del reproductor
  • Rendimiento optimizado
    • Compilación previa de activos de jugador
    • Insertar URL de miniaturas o videos en el reproductor
    • Entregue complementos a través de la misma CDN

Puede encontrar más información sobre la API de administración de jugadores en el Gestión de reproductores de Brightcove documentación.

Publica jugadores vinculados estáticamente

Como se señaló anteriormente, puede vincular estáticamente un video a un reproductor. Esto se puede hacer en la interfaz de usuario del reproductor o mediante la API de administración del reproductor. Para obtener información, consulte el Publicar videos y listas de reproducción con el módulo de medios y el Paso a paso: Gestión de jugadores documentos.

En estos casos, no es necesario incluir la identificación del video en la URL del video utilizada en el iframe o para un enlace directo, pero debe incluirse en el <video> etiqueta para una inserción en la página.

Comparación del tipo de jugador

Hay tres tipos de reproductores diferentes que componen el entorno de reproducción de Brightcove. Son:

  • Video.js: Una biblioteca de código abierto JavaScript y CSS que proporciona un marco para facilitar el trabajo y la creación de videos HTML5. El proyecto Video.js está patrocinado por Brightcove y Brightcove contribuye en gran medida al proyecto.
  • Brightcove Player (módulo de reproductores en Studio): Un reproductor de video desarrollado por Brightcove que utiliza Video.js como marco de reproducción subyacente. Si bien puede usar Video.js por sí solo, la mayoría de las veces querrá combinarlo con complementos y personalizaciones adicionales. Brightcove cultiva los complementos, gestiona su compatibilidad y lo mantiene todo actualizado a medida que pasa el tiempo. Brightcove Player constituye la base de todas las experiencias de vídeo de Brightcove.
  • Jugador de Brightcove: El servicio de alto rendimiento para crear y gestionar experiencias de reproductores de vídeo. Diseñado para satisfacer las necesidades de los principales editores de video del mundo, Brightcove combina Brightcove Player, el primer reproductor de video HTML5 multiplataforma líder, con un sólido conjunto de API de administración de reproductores y servicios de optimización del rendimiento. También se incluyen los SDK nativos de iOS y Android de Brightcove.

La siguiente información explica por qué elegir Brightcove Player en lugar de Video.js de código abierto puede ser una ventaja para usted.

  • Optimización del rendimiento: El servicio de administración de reproductores es responsable de optimizar la entrega del reproductor, optimizando cada reproductor mediante la compilación previa de complementos, los activos de máscara y las miniaturas para minimizar el tamaño de descarga, y luego los entrega a través de un CDN de alto rendimiento y optimizado globalmente.
  • Gestión de jugadores: Las capacidades de gestión del reproductor permiten a los usuarios realizar un seguimiento de las configuraciones del reproductor y realizar cambios. Esto es especialmente útil para clientes con múltiples configuraciones de reproductor, incluidas diferentes personalizaciones, complementos y estilos de reproductor, para diferentes partes de su sitio o aplicación. El servicio de administración de reproductores les permite actualizar la configuración de los reproductores que cambian complementos, estilos y otras configuraciones, sin cambiar el código de cada página. Los clientes pueden simplemente volver a publicar el reproductor y estos cambios se envían automáticamente al CDN.
  • Integraciones de terceros a prueba de futuro: Brightcove trabaja directamente con socios como comScore, Nielsen, Google / Doubleclick, Freewheel y muchos más para asegurarse de que los complementos de análisis y publicidad funcionen correctamente. Además, Brightcove se mantiene actualizado sobre todas las tendencias con navegadores y dispositivos conectados, para garantizar que nuestro reproductor funcione a la perfección con cualquier actualización de tecnologías web de terceros. Algunas funciones proporcionadas en los complementos de Brightcove no están disponibles en código abierto, como el cambio automático entre Flash y HTML5 para la visualización de anuncios mediante el SDK de IMA de Google, que se utiliza para mostrar anuncios de DFP. Brightcove garantiza que los complementos se prueben con Brightcove Player, pero no con todas las versiones de Video.js.
  • Soporte líder en la industria: Otra razón importante para elegir Brightcove Player es que el servicio es compatible con Brightcove. En general, esto significa que usted, como cliente, tiene un socio experimentado que lo ayudará en cada paso del camino. Con Brightcove, obtiene un tiempo de comercialización más rápido y un sistema que está garantizado para funcionar, y alguien a quien llamar cuando no funciona o si tiene alguna pregunta.
  • Costos de desarrollo y tiempo de comercialización: Es posible crear todas las funciones en torno a un reproductor de video HTML usted mismo, como lo ha hecho Brightcove, pero no es barato y llevará una gran cantidad de tiempo. Es probable que este tipo de trabajo no sea la competencia principal de las empresas a las que Brightcove vende y, por lo tanto, es poco probable que una empresa pueda hacerlo tan bien como Brightcove o por menos dinero. El código abierto parece más barato al principio, pero requiere mucho más tiempo y esfuerzo para construir una solución completa, y luego existen costos de mantenimiento continuos para mantener actualizados el reproductor, los complementos y las integraciones.
  • Una vez Integración: Brightcove Player está diseñado específicamente para admitir la reproducción de transmisiones Once con anuncios del lado del servidor codificados en la transmisión. Además, el reproductor está integrado con Once UX, lo que permite experiencias publicitarias ricas e interactivas basadas en el contenido VMAP, que indica cuándo se unen los anuncios en el lado del servidor.
