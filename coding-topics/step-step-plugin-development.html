    Paso a paso: Desarrollo de complementos

    En este tema aprenderás a través de una introducción práctica sobre el desarrollo y uso de plugins.

    Método de desarrollo

    El JavaScript y CSS que desarrolle para complementos eventualmente necesitarán almacenarse en una ubicación accesible a Internet, pero se recomienda que durante el desarrollo cree y pruebe localmente. Para hacer esto, realizará lo siguiente:

    • Crea un archivo para contener el código del complemento JavaScript
    • Cree un archivo para contener su código de complemento CSS (si es necesario)
    • Cree un archivo HTML para probar. Usar la embed_in_page implementación para el reproductor
    • Agregar un id a la <video-js> etiqueta
    • Enlace a las páginas JavaScript y CSS
    • Usar una <script> etiqueta para llamar a la función
    • Desarrollar y depurar su código

    El resto de este documento le guía a través de estos pasos para construir e implementar un complemento simple.

    Construir los conceptos básicos

    Para comenzar el proceso de construcción de un complemento, debe realizar algunos de los pasos fundamentales y de alto nivel mencionados anteriormente.

    1. Cree una carpeta denominada plugin-dev que se pueda examinar utilizando un servidor HTTP real. El servidor es necesario para las pruebas de implementación del iframe más adelante en este documento.
    2. En la carpeta, cree tres archivos con los siguientes nombres:
      • plugin-dev.html ( Insertar los elementos básicos de una página HTML en el archivo)
      • plugin-dev.js
      • plugin-dev.css
    3. Usando el Players módulo de Studio, cree un nuevo reproductor.
    4. En el Media módulo, seleccione un video y publíquelo con el reproductor recién creado.
    5. Mediante la VIDEO CONTENT sección, asocie un vídeo con el reproductor y, a continuación, guarde y publique el reproductor.
    6. Copie el Advanced código y péguelo en el cuerpo de la plugin-dev.html página.
    7. Desde el Embed Code & URL > Published Player menú, copie el Advanced Embed Code código y péguelo en el cuerpo de la plugin-dev.html página.
    8. Agregue un id atributo a la <video-js> etiqueta con un valor de player.
    9. Asegúrese de que su página HTML parezca similar a la siguiente (los clientes de Brightcove Player no tendrán una data-video-id propiedad): 
      <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Quick Start Plugin Dev</title>
</head>

<body>
  <video-js id="player"
    data-video-id="4607746980001"
    data-account="1507807800001"
    data-player="Bk6LLayNQ"
    data-embed="default"
    class="video-js" controls></video>
  <script src="https://players.brightcove.net/1507807800001/Bk6LLayNQ_default/index.min.js"></script>
</body>

</html>
    10. Navegue por la página para asegurarse de que su vídeo se está reproduciendo.

    Crear JavaScript

    A continuación, creará y probará el código JavaScript para construir una superposición para el video.

    1. Abra el archivo plugin-dev.js y pegue en el siguiente código JavaScript: 
      videojs.registerPlugin('pluginDev', function() {
  var player = this,
  overlay = document.createElement('p');
  overlay.className = 'vjs-overlay';
  overlay.innerHTML = "Becoming a plugin developer";
  player.el().appendChild(overlay);
});
      Nota: para la versión 5.x y los reproductores anteriores, debe usar videojs.plugin() en lugar de videojs.registerPlugin()
    2. Compruebe su comprensión de cada línea insertada:
      • Las líneas 1 y 7 son la sintaxis estándar para comenzar y finalizar un nuevo plugin de jugador. En este caso, el nombre del plugin es pluginDev.
      • La línea 2 es una forma estándar de tener una manera de conseguir un control para el jugador. Esto es necesario para llamar a los métodos del jugador, lo que hará más tarde.
      • La línea 3 crea un elemento de párrafo en el documento y lo asigna a la overlay variable.
      • La línea 4 asigna una clase a la superposición que se usará más adelante junto con CSS.
      • La línea 5 añade texto al elemento de párrafo.
      • La línea 6 utiliza el el() método del reproductor para recuperar el elemento DOM del jugador y, a continuación, añade el nuevo elemento de párrafo a ese DOM.
      Nota: Se requiere el punto y coma final al final del código JavaScript del complemento (línea 7 anterior).
    3. En el archivo HTML, agregue el siguiente código justo debajo de las <script> etiquetas existentes. Este código incluye el archivo JavaScript y luego llama al método definido en ese JavaScript. 
      <script type="text/javascript" src="plugin-dev.js"></script>
<script>videojs.getPlayer('myPlayerID').pluginDev();</script>
    4. Examine la página HTML de nuevo, y verá que nada visible ha cambiado. El problema es que la superposición está allí, pero no visible. Cambiarás esto más tarde.
    5. Para comprobar que la superposición está presente, utilice las herramientas de desarrollo de su navegador. En la sección Elementos , compruebe el <div> elemento del reproductor y verá el elemento de párrafo recién insertado, como se muestra aquí:

    Estilo del plugin

    Sabes que la superposición ahora forma parte del jugador, pero no es visible. A continuación, dará estilo a la superposición para que se vuelva visible. En esta sección del documento, usará un CSS muy simple para asegurarse de que la superposición es visible.

    1. Abra el archivo plugin-dev.css y péguelo en los estilos siguientes: 
      .vjs-overlay {
    background-color: #333333;
    color: white;
    position: absolute;
    margin-top: 100px;
    margin-left: 20px;
}
    2. En el archivo HTML, agregue el siguiente código justo debajo de la <link> etiqueta existente. Este código se vincula a su archivo CSS recién creado. 
      <link href="plugin-dev.css" rel="stylesheet">
    3. Navegue por la página HTML y verá que la superposición está ahora visible.

    Pasar datos al complemento

    A menudo será el caso de que desee cambiar el comportamiento de su plugin en el momento de la inicialización. Esto es posible pasando datos al plugin usando la options propiedad. En este ejemplo pasará el texto que se mostrará en la superposición.

    1. Abra la página HTML y modifique las <script> etiquetas para que options se cree una variable llamada y se le asigne un objeto con un par clave-valor de «OverlayText» :"Estos datos se suministran en la inicialización». Además, pase la options variable como argumento al llamar al pluginDev() método. Los cambios deben aparecer de la siguiente manera: 
      <script type="text/javascript" src="plugin-dev-copy.js"></script>
<script type="text/javascript">
  var options = {"overlayText": "This data supplied at initialization"};
</script>
<script>videojs.getPlayer('myPlayerID').pluginDev(options);</script>
    2. Ahora necesita alterar el JavaScript del complemento para usar los datos pasados a la función. La línea 110 muestra la función que acepta los datos como un parámetro, y la línea 114 utiliza los datos del objeto. 
      videojs.registerPlugin('pluginDev', function(options) {
  var player = this,
  overlay = document.createElement('p');
  overlay.className = 'vjs-overlay';
  overlay.innerHTML = options.overlayText;
  player.el().appendChild(overlay);
});
    3. Navegue por la página HTML y verá el nuevo texto que se está utilizando.
    Nota: El proceso que se muestra en esta sección del documento, incluyendo y llamando al plugin a través de JavaScript, es para fines de desarrollo y útil para la configuración inicial y las pruebas, pero no se considera una práctica recomendada. La mejor práctica es incluir el plugin como parte de su estudio de configuración de reproductor (como se muestra en la siguiente sección) incluso si está utilizando la implementación embed_in_page.

    Implementar el complemento

    Una vez que tenga el plugin, CSS y reproductor funcionando correctamente, debe implementar los activos para su uso adecuado. A continuación se proporciona una descripción general de los pasos necesarios para la implementación:

    • Copiar/mover los archivos JavaScript y CSS a su ubicación remota
    • Utilice Studio para agregar la configuración del plugin al reproductor.
    • Crear un archivo HTML para probar y utilizar la implementación del iframe para el reproductor
    • Prueba de anomalías

    Ahora serás guiado a través de estos pasos.

    1. Mueva su plugin JavaScript y archivos CSS a una ubicación accesible a Internet de su elección.
      Si lo elige, puede utilizar los archivos en la ubicación que se muestra a continuación en el ejemplo. También puede usar la API del sistema de entrega para almacenar los archivos en su propio repositorio.
    2. En la carpeta en la que está trabajando, cree otro archivo llamado plugin-dev-iframe.html.
    3. Utilice el Players módulo de Studio para editar el reproductor que creó anteriormente.
    4. Haga clic en Plugins en el menú de navegación izquierdo.
    5. A continuación, haga clic en Agregar un complemento.
    6. Para el nombre del complemento , escriba pluginDev.
    7. Para la URL de JavaScript , escriba (o su URL): 
       https://solutions.brightcove.com/bcls/brightcove-player/plugins/plugin-dev.js
    8. Para la URL CSS , introduzca (o su URL): 
       https://solutions.brightcove.com/bcls/brightcove-player/plugins/plugin-dev.css
    9. Introduzca las opciones de configuración en el cuadro de texto Opciones (JSON) . 
       {"overlayText": "This data is supplied at initialization"}
    10. El cuadro de diálogo de configuración debe aparecer similar al siguiente:
    11. Haga clic en Guardar.
    12. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    13. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.
    14. En el Media módulo, seleccione un video y publíquelo con el reproductor recién actualizado y publicado.
    15. Copie el Standard código y péguelo en el cuerpo de la plugin-dev-iframe.html página. Su página debería aparecer similar a la siguiente:
    16. Desde el Embed Code & URL > Published Player menú, copie el Standard Embed Code código y péguelo en el cuerpo de la plugin-dev-iframe.html página. Su página debería aparecer similar a la siguiente: 
      <!doctype html>
<html>

<head>
  <meta charset="UTF-8">
  <title>Quick Start Plugin Dev - iframe</title>
</head>

<body>
    <iframe src='https://players.brightcove.net/1507807800001/Bk6LLayNQ_default/index.html' allowfullscreen webkitallowfullscreen mozallowfullscreen></iframe>
</body>

</html>
    17. Navegue por la página HTML y verá el complemento funcionando correctamente con el iframe.

    Bibliotecas de terceros

    Si su complemento tiene dependencias en bibliotecas de terceros, por ejemplo jQuery, debe incluirlas de una de estas dos maneras:

    • Agregue la biblioteca como otro archivo JavaScript en la Plugins sección del Players módulo de Video Cloud Studio.
    • Agregue varias entradas en la scripts sección de la configuración del reproductor en una instrucción curl.

    ASEGÚRESE de poner las bibliotecas de las que depende su plugin antes de la entrada JavaScript de su plugin en ambos escenarios enumerados anteriormente. El orden de entrada SÍ importa.
