    Paso a paso: Generador de complementos Video.js

    En este tema, aprenderá a usar el generador de plugins video.js para crear y probar complementos para Brightcove Player.

    El generador de plugins video.js crea los archivos y carpetas necesarios para crear un complemento convencional. Incluye un entorno de prueba para que pueda ver fácilmente sus cambios a medida que desarrolla su plugin. Incluso genera los archivos finales del plugin que puede usar cuando esté listo para lanzarlo.

    Este es un proyecto de código abierto y no está oficialmente respaldado por Brightcove. Dado que es utilizado por muchos desarrolladores, por lo general se mantiene actualizado.

    Resumen

    En este tema, realizará las siguientes tareas:

    • Configurar su entorno
    • Crear la base del plugin
    • Ver los archivos de origen
    • Pon a prueba tu plugin
    • Editar el archivo JavaScript
    • Editar el archivo CSS
    • Construye tu plugin
    • Pasar datos a su plugin
    • Recomendaciones
    • Recursos

    Para obtener una descripción general del vídeo, consulte lo siguiente:

    Configurar su entorno

    Para configurar su entorno para ejecutar el generador de complementos, siga estos pasos:

    1. Si aún no lo tiene, descargue e instale el motor de ejecución de JavaScript Node.js. Esto incluye npm, que es un ecosistema de paquetes que incluye una gran colección de bibliotecas de código abierto.

      Utilice la versión recomendada para la mayoría de usuarios de Node.js.

    2. Cree una carpeta local en el equipo. Para este ejemplo, vamos a crear una pruebacon nombre. Aquí es donde irán los archivos del generador y sus archivos de plugin.

      Para evitar problemas de permisos, generalmente es mejor colocar esta carpeta en el directorio raíz del usuario.
    3. Revise la información README para el generador de plugins video.js. En la carpeta de documentos , abra y revise las convenciones del plugin video.js. Este documento habla sobre las reglas recomendadas para crear su plugin.
    4. Abra la aplicación Terminal en la carpeta que creó en un paso anterior. Para este ejemplo, abra la aplicación Terminal en la carpeta de prueba .

    5. En la aplicación Terminal, instale el generador de complementos junto con Yeoman:

      $ npm install -g yo generator-videojs-plugin

      El generador de plugins video.js utiliza un generador Yeoman, que es una herramienta de andamio para configurar las bases para cualquier proyecto. Esto configura los archivos y carpetas básicos. No necesitas entender a Yeoman. Solo necesita instalarlo con el generador de complementos.

    Crear la base del plugin

    En esta sección, creará las carpetas y archivos de base para su plugin.

    1. En el Terminal, ejecute el siguiente comando:

      $ yo videojs-plugin
      Puede ejecutar este comando tantas veces como desee. Esto es útil si decide agregar una opción a su plugin después de trabajar en él. La herramienta recordará sus selecciones anteriores, pero es posible que tenga que sobrescribir algunos archivos.

    2. A continuación, se le hará una serie de preguntas sobre los detalles de su plugin. Aquí hay una lista de las opciones, así como una breve descripción para cada una.

      Opción Descripción Valores para este ejemplo

    Ver los archivos de origen

    En esta sección, revisaremos los archivos fuente que fueron creados por el generador de videojs.

    1. En un editor, abra la carpeta superior donde coloque su proyecto de plugin. Abra la carpeta src . Aquí encontrará lo siguiente:

      • Un archivo plugin.js . Este archivo contiene el código de su plugin.
      • Un archivo plugin.css , si ha seleccionado Sí a la opción de herramientas CSS.

    2. (Si no ha seleccionado Sí a la opción de herramientas CSS, puede omitir este paso.) Abra el archivo src > plugin.css .

      Su código debe ser similar al siguiente:

      /* Note: all vars must be defined here, there are no "local" vars */
  :root {
    --main-color: red;
    --base-font-size: 9;
    --font-size: 7;
  }

  .video-js {

    &.vjs-demo {
      /* This class is added to the video.js element by the plugin by default. */
      display: block;

      & .remove-me, & .remove-me-too, &.finally-remove-me {
        /* examples of postcss syntax, you probably want to remove this */

        color: var(--main-color);

        /**
         * Note that you have to use calc and multiply by a value with a unit
         * prepending the unit like `var(--base-font-size)px` or
         * `calc(10 * var(--base-font-size)em` will NOT work!
         */
        font-size: calc(var(--font-size) * 8 * var(--base-font-size) * 1px);

      }
    }
  }

    3. Abra el archivo src > plugin.js .

      El generador utiliza ES6, que es la última versión de JavaScript. Obtenga más información sobre las nuevas características de ECMAScript 6. El generador traduce su código ES6 en código ES5 para que su código de distribución se ejecute en la mayoría de los navegadores.

      Su código debe ser similar al siguiente:

      • Líneas 25-27: llamar a la onPlayerReady() función, cuando el jugador esté listo. Aquí, puede pasar variables a su plugin usando el options objeto.
      • Línea 26: añade la vjs-demo clase a tu jugador. Por defecto, esto es lo único que hace el código esqueleto. Aquí es donde puede agregar funcionalidad a su plugin.
      • Línea 48: registra su plugin con video.js.
      • Línea 51: agrega un parámetro de versión a su plugin. Cuando ejecute el script de la versión npm , actualizará esta variable a la versión en la que se encuentre.
      import videojs from 'video.js';
import {version as VERSION} from '../package.json';

// Default options for the plugin.
const defaults = {};

// Cross-compatibility for Video.js 5 and 6.
const registerPlugin = videojs.registerPlugin || videojs.plugin;
// const dom = videojs.dom || videojs;

/**
 * Function to invoke when the player is ready.
 *
 * This is a great place for your plugin to initialize itself. When this
 * function is called, the player will have its DOM and child components
 * in place.
 *
 * @function onPlayerReady
 * @param    {Player} player
 *           A Video.js player object.
 *
 * @param    {Object} [options={}]
 *           A plain object containing options for the plugin.
 */
const onPlayerReady = (player, options) => {
  player.addClass('vjs-demo');
};

/**
 * A video.js plugin.
 *
 * In the plugin function, the value of `this` is a video.js `Player`
 * instance. You cannot rely on the player being in a "ready" state here,
 * depending on how the plugin is invoked. This may or may not be important
 * to you; if not, remove the wait for "ready"!
 *
 * @function demo
 * @param    {Object} [options={}]
 *           An object of options left to the plugin author to define.
 */
const demo = function(options) {
  this.ready(() => {
    onPlayerReady(this, videojs.mergeOptions(defaults, options));
  });
};

// Register the plugin with video.js.
registerPlugin('demo', demo);

// Include the version number.
demo.VERSION = VERSION;

export default demo;

    Pon a prueba tu plugin

    El generador de complementos hace que sea fácil desarrollar y probar su plugin utilizando un entorno de prueba local. Siga estos pasos para probar su plugin:

    1. En el Terminal, escriba el siguiente comando para iniciar el servidor de desarrollo:

      $ npm start

    2. En un navegador, introduzca lo siguiente para abrir el servidor de desarrollo:

      http://localhost:9999/

      Deberías ver el reproductor con un video de prueba. El generador le da un reproductor de demostración que se ejecuta en la página. En las herramientas de desarrollador del navegador, abra la pestaña Elementos para ver el HTML del reproductor.

    3. Inspeccione los Elementos de esta página web. Deberías ver que la vjs-demo clase se ha añadido al jugador. Recuerde que agregamos esta clase al reproductor en el código del plugin.

    4. Ahora, vamos a intentar agregar código al plugin para comenzar a reproducir automáticamente el video cuando el reproductor se carga. Vuelva al archivo src > plugin.js en su editor.

    5. En el onPlayerReady() método, agregue código para iniciar la reproducción del vídeo.

      const onPlayerReady = (player, options) => {
  player.addClass('vjs-demo');
  player.play();
};

    6. Guarde el archivo plugin.js en su editor. Cuando actualice el navegador, debe ver que el vídeo comienza a reproducirse en el entorno de prueba.

      A medida que desarrolla su plugin en el archivo plugin.js y guarda los cambios, la herramienta reconstruye y vuelve a cargar automáticamente el reproductor en el navegador. Esto hace que sea fácil de desarrollar y probar su plugin.

    7. Elimine la línea de código para comenzar a reproducir el vídeo.

      player.play();
      En el Terminal, presione CTRL-C para detener el servidor de desarrollo.

    Editar el archivo JavaScript

    En esta sección, agregará código al archivo fuente de JavaScript para agregar un elemento de párrafo con texto personalizado al reproductor.

    1. En su editor, regrese al archivo src > plugin.js .

    2. En la onPlayerReady() función, agregue código para agregar un <p> elemento con texto personalizado al reproductor.

      const onPlayerReady = (player, options) => {
  player.addClass('vjs-demo');
  var textDisplay = document.createElement('p');
  textDisplay.className = 'vjs-text';
  textDisplay.innerHTML = "Becoming a plugin developer";
  player.el().appendChild(textDisplay);
};
    3. Guarde el archivo. Recuerde que sus cambios se actualizarán automáticamente en el navegador de prueba.

    4. Vuelva al navegador de prueba. Verás que nada ha cambiado en el jugador. El problema es que el texto está ahí, pero no es visible. Lo arreglaremos a continuación.

      Para comprobar que el elemento de texto se ha añadido al reproductor, utilice las herramientas de desarrollo en el navegador. En la sección Elementos , expanda el <div> elemento del jugador. Debería ver la etiqueta de párrafo recién agregada.

      Haremos visible el texto en la siguiente sección usando CSS.

    Editar el archivo CSS

    En esta sección, agregará código al archivo fuente CSS para mostrar texto sobre el reproductor.

    1. En su editor, regrese al archivo src > plugin.css .

    2. Añada el .vjs-text selector con estilos para mostrar el texto personalizado en el reproductor.

      /* Note: all vars must be defined here, there are no "local" vars */
  :root {
    --main-color: red;
    --base-font-size: 9;
    --font-size: 7;
  }

  .vjs-text {
    background-color: #333333;
    color: white;
    position: absolute;
    font-size: 2em;
    text-align: center;
    width: 100%;
    margin-top: 10%;
  }
    3. Guarde el archivo. Recuerde que sus cambios se actualizarán automáticamente en el navegador de prueba.

    4. Vuelva al navegador de prueba. Ahora debería ver el texto personalizado que se muestra sobre el reproductor.

    5. Detenga el servidor de desarrollo presionando CTRL-C en la aplicación Terminal.

      Ahora, usted está listo para preparar su plugin para su distribución. Lo haremos a continuación.

    Construye tu plugin

    En esta sección, va a construir su plugin. Esto toma su código fuente y crea archivos CSS y JavaScript distribuibles.

    1. En el Terminal, ejecute el siguiente comando:

      $ npm run build

      La compilación toma su código fuente ES6 y lo convierte en código ES5 JavaScript.

    2. En la carpeta del proyecto, expanda la carpeta dist . Aquí es donde encontrará una versión distribuible de su plugin. Aquí, debe encontrar los siguientes archivos (suponiendo que seleccionó Sí para herramientas CSS):

      • videojs-demo.css
      • videojs-demo.js
      • videojs-demo.min.js
       

      También encontrará estos archivos menos utilizados:

      • videojs-demo.cjs.js
        • Este es un módulo CommonJS que se utilizará cuando se requiera su proyecto en Node, o si está utilizando Browserify para agrupar JavaScript.
      • videojs-demo.es.js
        • Este es un módulo ES6 para proyectos modernos que usan WebPack o Rollup para agrupar sus dependencias.

    3. En la carpeta dist , abra el archivo videojs-demo.js .

      Algunas cosas a tener en cuenta sobre este archivo de distribución:

      • Se ha agregado un banner de licencia a la parte superior del archivo.
      • Su plugin ha sido envuelto en un espacio de nombres browserify . Esto significa que no hay variables globales que puedan entrar en conflicto con el código de su página.
      • El plugin se registra internamente con videojs. Esto significa que puede crear múltiples plugins, cada uno de ellos trabajando de forma independiente, para un jugador.

    Pasar datos a su plugin

    Esta sección es opcional. Puede omitir esta sección a menos que desee aprender a pasar datos a su plugin.

    1. Usaremos la options propiedad para pasar datos de nuestra página HTML al plugin. Para obtener más información sobre el uso de esta propiedad, consulte el documento Pass Data to the Plugin .

    2. En su editor, regrese al archivo src > plugin.js .

    3. En la onPlayerReady() función, agregue código para utilizar el valor de texto de la options propiedad si existe; de lo contrario, utilice un valor de texto predeterminado.

    Recomendaciones

    Se recomienda utilizar este generador para todos sus complementos, incluso los simples. De esta manera, siempre tendrá la misma configuración para todos sus complementos. También le libera de la tarea de tener que crear scripts para realizar ciertas funciones, como el linting o la compresión.

    Con este generador, puede centrarse en desarrollar y probar su plugin sin tener que preocuparse por las herramientas subyacentes.

    Puede mantener su plugin local o ponerlo en su repositorio privado de GitHub. Crear un plugin de videojs significa que funcionará con Brightcove Player.
