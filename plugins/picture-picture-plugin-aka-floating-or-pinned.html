    Plugin de imagen en imagen (también conocido como «flotante» o «anclado»)

    En este tema, aprenderá a utilizar el complemento Picture-in-picture para habilitar este modo para Brightcove Player.

    Resumen

    El modo Picture-in-Picture permite a los usuarios realizar varias tareas. Una vez que se haya iniciado la reproducción de vídeo, a medida que el usuario se desplaza hacia abajo por la página, el modo de imagen en imagen se reubicará y fijará el reproductor a una esquina de la página web.

    El complemento Picture-in-Picture sólo funciona con el código Advanced (video) incrustado. No funcionará con la implementación Standard (iframe) .

    ¿Por qué se llama Picture-in-Picture

    El comportamiento que este complemento habilita también se conoce comúnmente como comportamiento «flotante» o «anclado». Brightcove utiliza el término «imagen en imagen» debido a la adopción de esta verboria por Apple y Google para sus navegadores nativos. Los estándares W3C también hacen referencia a «picture in-picture».

    Ejemplo de reproductor

    Inicie la reproducción de vídeo. (A continuación, puede detenerlo). Desplácese hacia abajo por la página para ver el reproductor de imágenes en imagen en la esquina derecha de la página web. Puede cerrar el reproductor de imagen en imagen o desplazarse hacia arriba hasta el reproductor de tamaño completo.

    Consulte el plugin Plugin Picture-in-Picture de Brightcove Learning Services (@bcls1969 ) en CodePen.

    Implementar usando el módulo Players

    Para implementar el complemento Picture-in-Picture utilizando el módulo Players, siga estos pasos:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. Seleccione Plugins en el menú de navegación de la izquierda.

    4. A continuación, seleccione el botón Agregar un complemento y, a continuación, seleccione Brightcove Plugin.

    5. Expanda el menú desplegable Brightcove Plugin y seleccione Picture-in-Picture.

    6. Opcional: Introduzca las opciones de configuración en el cuadro de texto Opciones (JSON) . Aquí se muestra un ejemplo para ajustar la alineación horizontal del reproductor cuando está en modo imagen en imagen:

      Consulte la sección de opciones para obtener más detalles.

    7. Seleccione el botón Guardar. Ahora verá el plugin Picture-in-Picture añadido a la lista de complementos para su reproductor.

    8. Para publicar el reproductor, seleccione Publicar e incrustar > Publicar cambios.
    9. Para cerrar el cuadro de diálogo abierto, seleccione Cerrar.

    10. Vuelva al módulo MEDIA y publique su vídeo con el reproductor que acaba de actualizar para Picture-in-Picture.

    11. En un editor, copie el código de inserción del reproductor en su página web.

    12. El plugin Picture-in-Picture requiere que su reproductor esté envuelto por un elemento contenedor con la clase establecida en vjs-pip-container. Su código debería verse similar a esto:

      <div class="vjs-pip-container">
  <video-js id="myPlayerID"
    data-video-id="5701202551001"
    data-account="1752604059001"
    data-player="default"
    data-embed="default"
    class="video-js"
    width="640" height="360"
    controls></video-js>
  <script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>
</div>

      Si el <div> elemento anterior no se encuentra como el padre del reproductor, el plugin no se inicializará y obtendrá la siguiente advertencia de registro:

      VIDEOJS: WARN: expected player to be a child of a "vjs-pip-container" element, cannot continue with picture-in-picture plugin
    13. Cuando el vídeo comience a reproducirse, desplácese hacia abajo para ver el reproductor de imagen en imagen anclado al final de la página.

    Implementar usando código

    Para implementar el complemento utilizando código personalizado, configurará las siguientes propiedades del complemento:

    • scripts - JavaScript suministrado para el plugin y no cambiará para diferentes implementaciones de plugin
    • stylesheets - CSS suministrado a usted para el plugin y no cambiará para diferentes implementaciones de plugin
    • plugin name - Siempre pip
    • plugin options - Contiene una matriz de propiedades y valores

    Para agregar el complemento a su código, siga estos pasos:

    1. Añada los estilos para el plugin picture in-picture. 
      <link href="https://players.brightcove.net/videojs-pip/1/videojs-pip.css" rel="stylesheet">
    2. Agregue el archivo JavaScript para incluir el plugin picture in-picture. 
      <video-js id="myPlayerID"
  data-video-id="5701202551001"
  data-account="1752604059001"
  data-player="default"
  data-embed="default"
  class="video-js"
  width="640" height="360"
  controls></video-js>
<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

<!-- Script for the picture-in-picture plugin -->
<script src="https://players.brightcove.net/videojs-pip/1/videojs-pip.min.js"></script>

    3. Envuelva el código de inserción del reproductor en un elemento contenedor con la clase establecida en vjs-pip-container. El plugin Picture-in-Picture requiere que su reproductor esté envuelto por este elemento contenedor.

      <div class="vjs-pip-container">
    <video-js id="myPlayerID"
      data-video-id="5701202551001"
      data-account="1752604059001"
      data-player="default"
      data-embed="default"
      class="video-js"
      width="640" height="360"
      controls></video-js>
    <script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>
  </div>

        <!-- Script for the picture-in-picture plugin -->
        <script src="https://players.brightcove.net/videojs-pip/1/videojs-pip.min.js"></script>

      Si el <div> elemento anterior no se encuentra como el padre del reproductor, el plugin no se inicializará y obtendrá la siguiente advertencia de registro:

      expected player to be a child of a "vjs-pip-container" element, cannot continue with picture-in-picture plugin
    4. Agregue código de script personalizado que haga lo siguiente:
      • Cuando el jugador esté listo, obtiene una referencia al Brightcove Player. En este ejemplo, estamos creando una variable llamada myPlayer y asignándole una referencia al jugador.
      • Inicializa el plugin de imagen en imagen.
      videojs.getPlayer('myPlayerID').ready(function() {
  // When the player is ready, get a reference to it
  var myPlayer = this;
  // Initialize the picture-in-picture plugin
  myPlayer.pip();
});

      Para obtener más información, consulte el ejemplo de lápiz de código anterior.

    Opciones

    Puede pasar un objeto de opciones al plugin tras la inicialización. Este objeto puede contener cualquiera de las siguientes opciones:

    PermitirOnMobile

    • allowOnMobile:
      • Tipo:boolean
      • Predeterminado:false

      • De forma predeterminada, el modo de imagen en imagen no funcionará en dispositivos móviles Android o iOS. La razón de esto es porque estos posicionamiento fijo CSS no funcionan en dispositivos con zoom con múltiples ventanas gráficas de la misma manera que en dispositivos de escritorio.

        Si desea habilitar Picture-in-Picture en dispositivos móviles compatibles, puede hacerlo con esta opción. Hay algunos casos en los que es posible que desee habilitar el soporte:

        • El integrador está dispuesto a asumir la responsabilidad de gestionar el posicionamiento para el modo de reproductor de imagen en imagen.
        • Es poco probable que se amplíe el sitio web que utiliza el reproductor.
        • El zoom se ha deshabilitado mediante una user-scalable=no directiva. El uso de esta directiva hará position: fixed que se comporte como lo hace en los escritorios, pero no se recomienda específicamente porque puede ser un problema de accesibilidad. 
          <meta name="viewport" content="width=device-width, user-scalable=no">

      • Ejemplo:

        // Allow mobile (iOS and Android) devices to enter PIP mode.
player.pip({allowOnMobile: true});
    En dispositivos iOS anteriores a la versión 12.2 y dispositivos Android, una vez que el reproductor se haya movido a su nueva ubicación, no volverá a aparecer en la ubicación original del reproductor, incluso cuando la ubicación del reproductor original sea totalmente visible en el navegador.

    cerrable

    • closeable:
      • Tipo:boolean
      • Predeterminado:true
      • De forma predeterminada, el usuario puede desactivar el modo de imagen en imagen haciendo clic en el botón X situado en la esquina superior derecha del reproductor. Esta función se puede deshabilitar pasando false por esta opción.

      • Ejemplo:

        // Do not allow the user to close the PIP mode player.
player.pip({closeable: false});

    escala

    • scale:
      • Tipo:number
      • Predeterminado:2 / 3
      • Factor de escala aplicado al reproductor cuando se encuentra en modo imagen en imagen. Este valor debe ser un número mayor que cero e inferior o igual que 1.

      • Ejemplo:

        // Detach the player, but do not change its size.
player.pip({scale: 1});

      • Ejemplo 2:

        // Detach the player, and change its size to 1/2.
player.pip({scale: 0.5});

    altura y ancho

    • height y width:
      • Tipo:number
      • Predeterminado:null

      • Por defecto, el plugin reducirá las dimensiones del jugador por un factor determinado por la scale opción. Sin embargo, proporcionar un height (o width ) anulará la escala predeterminada y establecerá el tamaño del reproductor reducido explícitamente.

        Si solo se proporciona una dimensión, la otra se reducirá para mantener la relación de aspecto. Si se proporcionan ambas dimensiones, el jugador se ajustará al tamaño exacto y especificado, permitiendo un cambio en la relación de aspecto.

      • Ejemplo:

        // Detach the player and set its width to 300 pixels. Scale its height to
// maintain its current aspect ratio.
player.pip({width: 300});

    POSx

    • posX:
      • Tipo:string
      • Predeterminado:"right"

      • La alineación horizontal del reproductor cuando está en modo imagen en imagen - ya sea «derecha» o «izquierda».

      • Ejemplo:

        // When the player is in PIP mode, align it to the left side of the viewport.
player.pip({posX: 'left'});

    PoSy

    • posY:
      • Tipo:string
      • Predeterminado:"bottom"

      • La alineación vertical del reproductor cuando está en modo imagen en imagen - ya sea «arriba» o «inferior».

      • Ejemplo:

        // When the player is in PIP mode, align it to the top of the viewport.
player.pip({posY: 'top'});

    visible

    • viewable:
      • Tipo:number
      • Predeterminado:0.8

      • El umbral en el que el jugador se considera visible. En otras palabras, cuando este porcentaje del área total del jugador (altura y ancho) es visible en la ventana gráfica del navegador, se considera visible.

        Por ejemplo, con el valor predeterminado de 0.8, el reproductor no se considera visible a menos que el 80% del mismo sea visible en la ventana gráfica. El viewable valor debe ser un número mayor o igual que 0 y menor o igual que 1.

      • Ejemplo:

        // If more than half the player is outside of the viewport, activate PIP mode.
player.pip({viewable: 0.5});

    ManualContainerTamaño

    • manualContainerSize:
      • Tipo:boolean
      • Predeterminado:false

      • Por defecto, un reproductor con este plugin habilitado mantendrá las dimensiones físicas del elemento contenedor especial sincronizadas con las dimensiones del reproductor. Sin embargo, este comportamiento puede no funcionar para todos los casos de uso, por lo que puede deshabilitarse configurando esta opción en true.

        Al hacerlo, el elemento contenedor se comportará como un elemento de bloque normal. Esto significa que los usuarios del plugin tendrán que administrar su tamaño por su cuenta.

        Esta opción también se puede establecer en el código de inserción a través del data-manual-container-size atributo booleano en el contenedor. Su presencia establecerá esta opción en true. Tenga en cuenta que cualquier valor pasado al complemento tendrá prioridad sobre cualquier valor definido en el código de inserción.

      • Ejemplo:

        // Implementation will handle sizing the container.
player.pip({manualContainerSize: true});

      • Ejemplo usando el data-manual-container-size atributo en el contenedor:

        <div class="vjs-pip-container" data-manual-container-size>
  <video-js class="video-js vjs-default-skin">
  </video-js>
</div>

    Uso de las opciones

    Tiene dos formas de utilizar la opción:

    1. En la sección > PLUGINS PLAYERS de Studio.
    2. Usar JavaScript con el reproductor.

    Uso de Studio

    En Studio, edite el reproductor y vuelva a la sección Plugins . Haga clic en Picture-in-Picture en la lista de nombres. Usando el formato JSON adecuado, enumere la opción (entre comillas), seguida del valor apropiado. Si los valores son de cadena, deben estar entre comillas. Si son numéricos o booleanos, no se pueden incluir comillas:

    Utilizando JavaScript

    Para implementar las opciones en el código, usted crea un objeto, asigna las opciones deseadas sus respectivos valores, luego pasa el objeto options al llamar al plugin:

    videojs.getPlayer('myPlayerID').ready(function() {
  // When the player is ready, get a reference to it
  var myPlayer = this,
      options = {};

  options.scale = 0.5;
  options.posX = "left";
  // Initialize the picture-in-picture plugin
  myPlayer.pip(options);
});

    Métodos

    Los siguientes métodos le permiten interactuar con el plugin picture-in-picture:

    Método Descripción
    myPlayer.pip().enable() Habilitar el comportamiento automático de imagen en imagen
    myPlayer.pip().disable() Deshabilitar el comportamiento automático de imagen en imagen
    myPlayer.pip().toggle() Activar o desactivar manualmente el modo de imagen en imagen en función del estado actual
    myPlayer.pip().activate() Coloque manualmente el reproductor en modo de imagen en imagen
    myPlayer.pip().deactivate() Sacar manualmente el reproductor del modo de imagen en imagen

    Eventos

    Los siguientes eventos se desencadenan desde el plugin picture-in-picture:

    Evento Descripción
    beforepipactive Se dispara antes de que se active el modo pip
    pipactive Se dispara después de activar el modo pip
    beforepipinactive Se dispara antes de que se desactive el modo pip
    pipinactive Se dispara después de que se desactive el modo pip
    beforepipenabled Se activa antes de que el plugin videojs-pip esté habilitado
    pipenabled Se activa después de que el plugin videojs-pip está habilitado
    beforepipdisabled Se activa antes de que el complemento videojs-pip esté deshabilitado
    pipdisabled Se dispara después de que el plugin videojs-pip está deshabilitado
