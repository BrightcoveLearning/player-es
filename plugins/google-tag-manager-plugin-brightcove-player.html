    Complemento de Google Tag Manager para Brightcove Player

    En este tema, aprenderá a utilizar el complemento Google Tag Manager (GTM) para Brightcove Player.

    Resumen

    El complemento Google Tag Manager (GTM) le permite integrar la funcionalidad de GTM con Brightcove Player.

    Los administradores de etiquetas le permiten agregar y publicar etiquetas sin tener que usar código. Debido a que son una solución del lado del cliente, los gestores de etiquetas le permiten recopilar datos en diferentes tecnologías de marketing. Esto permite a los vendedores al darles la capacidad de recopilar datos de usuarios y jugadores a través de un conjunto diverso de puntos de recopilación.

    Si ya está utilizando Google Analytics, Adwords y DoubleClick, es posible que también desee utilizar Google Tag Manager, ya que se integra bien con estas herramientas. GTM ofrece un gran número de opciones variables y de activación.

    El complemento Google Tag Manager le permite realizar un seguimiento de los eventos de Brightcove Player mediante el panel de control de Google Analytics. Se pueden realizar un seguimiento de los siguientes eventos del reproductor: 

    • Cargas del reproductor
    • Impresión del vídeo
    • Visionados del vídeo
    •  Interacción de vídeo
    • Solicitud de reproducción
    • Fin del vídeo
    • Eventos de reproductores avanzados, como inicio del anuncio, fin del anuncio
    • Errores que interrumpen la experiencia de reproducción
    El complemento Google Tag Manager es compatible con la versión 6.36.5+ de Brightcove Player.

    Terminología de gestión de etiquetas

    Estos son algunos de los términos comunes que encontrará al usar Google Tag Manager (GTM).

    Plazo Descripción
    Contenedor Su cuenta de sistema de administración de etiquetas puede tener varios contenedores (perfiles), con sus propias etiquetas y variables. Cada contenedor puede tener múltiples entornos, normalmente configurados para el desarrollo, la puesta en escena y la producción.
    Capa de datos La capa de datos se refiere a la recopilación de datos que se encuentra en el código de su página web. Este objeto JavaScript se utiliza para pasar información de su sitio web a su contenedor de administrador de etiquetas. A continuación, puede utilizar esta información para rellenar variables y activar desencadenadores en las configuraciones de etiquetas.
    Evento Los eventos son puntos de datos utilizados para enviar datos a su herramienta de informes. Estos están relacionados con la carga y reproducción de vídeo.
    Etiqueta Las etiquetas son fragmentos de código colocados en su página web para recopilar datos de interacción del usuario. Los sistemas de administración de etiquetas simplifican el proceso de agregar etiquetas a su sitio, lo que requiere que solo proporcione el identificador de etiqueta y el tipo de evento de etiqueta.
    Etiquetar marketplace Las plantillas de mercado de etiquetas proporcionan fragmentos de código para implementar una etiqueta. Si bien esto reduce la cantidad de conocimiento de código necesario, se requiere cierta familiaridad con JavaScript y HTML para una implementación exitosa.
    Rastreador Los rastreadores son objetos que recopilan, almacenan datos y envían esos datos a Google Tag Manager.
    Gatillo Los activadores indican cuándo se debe agregar código de etiqueta a una página. Estas son condiciones definidas por el usuario.

    Introducción

    Para configurar el Administrador de etiquetas de Google, sigue estos pasos:

    Pasos

    1. Revisa la documentación de Google para Configurar e instalar Tag Manager.

    2. En Tag Manager, cree una cuenta o utilice una cuenta existente. Para crear una nueva cuenta, selecciona el botón Crear cuenta .

      De forma predeterminada, se crea un nuevo contenedor para una cuenta nueva. Puede crear contenedores adicionales dentro de cada cuenta.

    3. Introduzca valores para la nueva página de cuenta, incluidos el nombre de cuenta y el nombre del contenedor. Seleccione una plataforma de destino.

    4. Seleccione el botón Crear.

    Revisión de eventos para realizar un seguimiento

    Revise la tabla siguiente y determine qué eventos desea realizar un seguimiento.

    Tipo Evento Descripción
    Cargas del reproductor bcvideo_player_load Se envía cuando se ha cargado un reproductor y está listo para su reproducción
    Número de impresiones del vídeo bcvideo_video_impression Se envía cuando un vídeo se ha cargado en un reproductor y está listo para su reproducción
    Vista de vídeo bcvideo_video_view Se envía cuando un vídeo ha comenzado a reproducirse a través de la reproducción automática o debido a la interacción del usuario
    Nivel de interacción con el vídeo bcvideo_video_engagement Un usuario vio un intervalo de segundos de la línea de tiempo de un vídeo. Este evento es un latido para el seguimiento de la interacción del vídeo y probablemente se enviará muchas veces durante la reproducción, dependiendo de la interacción del usuario y de la duración del vídeo.
    Solicitud de reproducción bcvideo_play_request Se envía cuando la reproducción es iniciada por el usuario haciendo clic expresamente en el botón de reproducción, o automáticamente cuando la plataforma activa la reproducción en un escenario de reproducción automática
    Inicio del anuncio bcvideo_ad_start Se envía cuando la plataforma de reproducción traspasa el control a un agente publicitario
    Fin del anuncio bcvideo_ad_end Se envía cuando el control se devuelve desde el agente publicitario a la plataforma de reproducción
    Error bcvideo_error Se envía cuando se detectan errores fatales que interrumpen la experiencia de reproducción
    Vídeo completado bcvideo_video_complete Se envía cuando finaliza la reproducción de vídeo

    Creación de variables, etiquetas y desencadenadores

    Cada evento de Brightcove que se va a realizar un seguimiento debe crearse como etiqueta de evento en Google Tag Manager. Los identificadores de eventos de GTM se utilizan dentro del reproductor, lo que permite que los datos de eventos de Brightcove se transmitan a eventos GTM.

    Para enviar estos eventos a Google Analytics (GA), es necesario crear disparadores GTM para cada etiqueta. Los parámetros personalizados asociados a los eventos también se crean en la ficha Variables GTM.

    Adición de variables a GTM

    Los parámetros contienen datos adicionales y se pasan con eventos. Todos los parámetros de Brightcove que se grabarán deben crearse como una variable en Google Tag Manager. El parámetro nombrado se utilizará dentro del reproductor, permitiendo que los parámetros de Brightcove se pasen a las variables de Google Tag Manager.

    Para crear variables en Google Tag Manager, sigue estos pasos

    1. Inicie sesión en su cuenta de Google Tag Manager

    2. Seleccione Variables en la navegación del lado izquierdo.
    3. Para Variables definidas por el usuario, seleccione el botón Nuevo.

    4. Seleccione el tipo de variable como Variable de capa de datos.

    5. Introduzca los detalles:

      • Nombrede variable de capa de datos: nombre de la variable
      •  Versión de capa de datos - Versión 2

    6. Parámetros que se crearán:

      • Cuenta : nombre de cuenta de Brightcove
      • Jugador - ID de jugador
      • Nombre del jugador - Nombre del jugador
      • Versión de plataforma : número de versión de la plataforma
      • Sesión - Identificador de sesión
      • Vídeo - ID de vídeo
      • Nombre del vídeo - Nombre del vídeo
      • Rango - El rango de segundos visualizados en el formato StartSecond.. EndSecond
      • Segundos de vídeo visualizados - Recuento de segundos vistos
      • Duración del vídeo - Duración del vídeo en segundos
      • Porcentaje de vídeo visto - Porcentaje de vídeo visto
      • Código de error: código de error específico de la plataforma asociado con el evento
      • Hito de vídeo - Hito (25% ,50% ,75% ,100%) completado por el video
      • Video Playhead - El tiempo en el vídeo en el que se desencadena el evento
      • País - código de país ISO-3166 (alfa 2), p. ej.: EE. UU.
      • Nombre del país - Nombre de país legible por humanos , por ejemplo: Estados Unidos
      • Región - código de región ISO-3166 (alfa 2)
      • Nombre de región - Nombre de región legible por humanos
      • Ciudad - Nombre de la ciudad
      • Navegador - Nombre completo y versión del navegador utilizado
      • Tipo de navegador - Nombre común del navegador utilizado (es decir, Chrome, Edge, Firefox)
      • Destination - URI que originó el evento
      • Origen : URI que envió al usuario final al destino
      • Tipo de dispositivo : anula para especificar el sistema operativo del dispositivo que originó el evento en los casos en los que el agente de usuario no es confiable
      • Sistema operativo de dispositivo : el nombre del sistema operativo que utiliza el dispositivo
      • Versión del sistema operativo de dispositivo : la versión del sistema operativo que utiliza el dispositivo

    Adición de etiquetas a GTM

    Después de determinar qué eventos desea realizar un seguimiento, las etiquetas deben crearse en Google Tag Manager.

    Para crear etiquetas en Google Tag Manager, sigue estos pasos

    1. Inicie sesión en su cuenta de Google Tag Manager

    2. Seleccione Etiquetas en el panel de navegación izquierdo y seleccione el botón Nuevo .

    3. Seleccione el tipo de etiqueta como Google Analytics: Analítica universal.

    4. Introduzca los detalles de la etiqueta:

      • Tipo de pista - Establecer en Evento
      • Categoría - Variable de categoría de evento
      • Acción - Variable de acción de evento
      • Etiqueta - Variable de etiqueta de evento
      • Valor - Ninguna
      • Hit sin interacción - Establecer en True
      • Configuraciónde Google Analytics - Configuración predeterminada
      • Identificadorde seguimiento: ID de seguimiento del panel de GA. Seleccione Variable de ID de seguimiento.

    5. Vaya a Configuración avanzada y defina los índices de dimensión personalizados y los valores de dimensión relacionados. Estos índices deben coincidir con los de Google Analytics.

    Integración con Brightcove

    Para integrar GTM con Brightcove Player, agregue el complemento Google Tag Manager a su reproductor.

    Uso de Studio

    Los siguientes pasos detallan el uso de Video Cloud Studio para cargar el complemento.

    1. En Video Cloud Studio, abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea agregar el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. Seleccione Plugins en el menú de navegación de la izquierda.

    4. A continuación, seleccione el botón Agregar un complemento y, a continuación, seleccione Brightcove Plugin.

    5. Amplíe el menú desplegable de Brightcove Plugin y seleccione Google Tag Manager.

    6. Para indicar al reproductor qué eventos y dimensiones personalizadas desea realizar un seguimiento, cree un objeto JSON para pegarlo en el cuadro de texto Opciones (JSON) .

      Descripciones de campo:

      • tracker - Tu ID de seguimiento de Google Tag Manager (GTM-XXXXXXX)
      • eventsToTrack - Eventos que quieres rastrear
      • paramsToTrack - Dimensiones personalizadas que desea rastrear

      Un ejemplo:

      {
  "tracker":"your Google Tag Manager id",
  "eventsToTrack": {
    "error":"Error",
    "play_request":"Play Request",
    "video_engagement":"Video Engagement"
  },
  "paramsToTrack": {
    "bcvideo_video_seconds_viewed":"dimension1",
    "bcvideo_video_playhead":"dimension2",
    "bcvideo_account":"dimension3",
    "bcvideo_player":"dimension4",
    "bcvideo_player_name":"dimension5",
    "bcvideo_session":"dimension8",
    "bcvideo_video":"dimension6",
    "bcvideo_video_name":"dimension7",
    "bcvideo_platform_version":"dimension9",
    "bcvideo_video_duration":"dimension11",
    "bcvideo_range":"dimension10",
    "bcvideo_video_percent_viewed":"dimension12",
    "bcvideo_error_code":"dimension13",
    "bcvideo_video_milestone":"dimension14",
    "bcvideo_destination":"dimension15"
  }
}
      Asegúrese de que las dimensiones personalizadas del objeto JSON coinciden con el nombre y el índice de Google Analytics.

    7. Seleccione el botón Guardar. Ahora verá el complemento Google Tag Manager añadido a la lista de complementos para su reproductor.

    8. Para publicar el reproductor, seleccione Publicar e incrustar > Publicar cambios.
    9. Para cerrar el cuadro de diálogo, seleccione Cerrar.

    10. Vuelve a MEDIA y publica tu vídeo o lista de reproducción con el reproductor que acabas de actualizar para Google Tag Manager.

      Cuando el reproductor se carga y el usuario interactúa con el reproductor, se rastrearán los eventos y parámetros especificados.

    Creación de cotas personalizadas

    Las dimensiones personalizadas son métricas que usted mismo crea en Google Analytics. Para obtener más información, consulta el documento de métricas y dimensiones personalizadas de Google.

    Con Google Scripts, puede automatizar el proceso de agregar dimensiones personalizadas en Google Analytics.

    Pasos

    Siga estos pasos para utilizar un script de aplicación de Google para crear dimensiones personalizadas:

    1. Inicie sesión en Google.

    2. Navega hasta el Hub para desarrolladores de G Suite y selecciona el nuevo proyecto en blanco de Apps Script.

    3. Seleccione Proyecto sin título y asigne un nombre a su proyecto.

    4. Copie el siguiente código de script de aplicación personalizado en un editor y actualice los siguientes valores:

      • accountId - El ID de tu cuenta de Google Analytics
      • webPropertyId - Identificador de seguimiento de tu cuenta de Google
      • dimensions - Las dimensiones personalizadas que se van a añadir. La lista actual contiene todas las dimensiones personalizadas. Observe que las cotas tienen el prefijo bcvideo_. Esto es para garantizar que no haya conflictos de nomenclatura con estas variables. 
      //Set values for GA Account ID and web property ID to which you have to set custom dimensions.
var accountId='your account id';
var webPropertyId='your tracking code';
//Add required custom dimensions to be created to this list
var dimensions=['bcvideo_video_seconds_viewed','bcvideo_video_playhead','bcvideo_account','bcvideo_player','bcvideo_player_name','bcvideo_video','bcvideo_video_name','bcvideo_session','bcvideo_platform_version','bcvideo_range','bcvideo_video_duration',
'bcvideo_video_percent_viewed','bcvideo_error_code','bcvideo_milestone','bcvideo_destination'];


//Function to create custom dimensions in GA
function createGACustomDimensions() {

  //This loop iterates through the custom dimensions in order to set the custom dimension scope, active values into our GA property
  for(var index=0;index<dimensions.length;index++){
  var customDimensions ={
    'name': dimensions[index],
    'scope':'HIT',
    'active':true,
  }

  //This create a request to the new custom dimension
  Analytics.Management.CustomDimensions.insert(customDimensions,accountId,webPropertyId);

  // Pause for 1 second between adding each custom dimension, this ensure that the GA property has time
  // to process the request fully before moving to the next custom dimension.
  Utilities.sleep(1000)
  }
}
      En este ejemplo, estamos poniendo las dimensiones en un objeto llamado customDimensions. Para agregarlas como variables de capa de datos, debe hacer referencia a cada nombre de variable como customDimensions.{dimension}

    5. Reemplace el bloque de function myFunction(){} código, como se muestra a continuación, con su script de aplicación personalizado.

    Eventos y parámetros

    Los siguientes eventos y parámetros están disponibles para enviarlos a Google Tag Manager.

    Eventos

    Estos eventos vienen de su jugador.

    • player_load
    • video_impression
    • play_request
    • video_view
    • video_engagement
      • video_engagement - Hito del 25%
      • video_engagement - Hito 50%
      • video_engagement - 75% hito
    • ad_start
    • ad_end
    • error
    • video_complete - Hito 100%

    El porcentaje de vídeo visto (video_engagement) se puede calcular de la siguiente manera (similar al evento video_engagement en Google y Adobe Analytics): 

    Video watched percentage = duration the user actually watched / total video duration

    Por ejemplo, en un vídeo de 20 segundos, el hito del 25% se puede lograr en los dos casos siguientes

    • Relojes de usuario de 0 a 5 segundos
    • El usuario observa desde 0-1er segundo y saltar al 11º segundo y ver hasta el 15 segundo

    En los dos casos anteriores, el usuario observó 5 segundos, por lo que el hito se informará como 25%.
