Resumen

El complemento Google Tag Manager (GTM) le permite integrar la funcionalidad GTM con Brightcove Player.

Los gestores de etiquetas le permiten añadir y publicar etiquetas sin tener que usar código. Debido a que son una solución del lado del cliente, los gestores de etiquetas le permiten recopilar datos en diferentes tecnologías de marketing. Esto permite a los especialistas en marketing proporcionándoles la capacidad de recopilar datos de usuarios y jugadores en un conjunto diverso de puntos de recopilación.

Si ya está utilizando Google Analytics, Adwords y DoubleClick, es posible que también desee utilizar Google Tag Manager, ya que se integra bien con estas herramientas. GTM ofrece una gran cantidad de opciones variables y de activación.

El complemento Google Tag Manager le permite realizar un seguimiento de los eventos de Brightcove Player mediante el panel de Google Analytics. Se pueden rastrear los siguientes eventos de jugador: 

  • Carga del jugador
  • Impresión de video
  • Vista de video
  • Participación de video
  • Solicitud de reproducción
  • Video completo
  • Eventos de reproductor avanzados, como inicio y finalización del anuncio
  • Errores que interrumpen la experiencia de reproducción
El complemento Google Tag Manager es compatible con Brightcove Player versión 6.36.5+.

Terminología de gestión de etiquetas

Estos son algunos de los términos comunes que encontrará al utilizar Google Tag Manager (GTM).

Plazo Descripción
Envase La cuenta del sistema de administración de etiquetas puede tener varios contenedores (perfiles), con sus propias etiquetas y variables. Cada contenedor puede tener múltiples entornos, generalmente configurados para desarrollo, preparación y producción.
Capa de datos La capa de datos se refiere a la recopilación de datos que viven en el código de su página web. Este objeto JavaScript se utiliza para pasar información de su sitio web al contenedor del administrador de etiquetas. Luego, puede usar esta información para completar variables y activar activadores en sus configuraciones de etiquetas.
Evento Los eventos son puntos de datos que se utilizan para enviar datos a la herramienta de informes. Están relacionados con la carga y reproducción de vídeos.
Etiqueta Las etiquetas son fragmentos de código colocados en la página web para recopilar datos de interacción del usuario. Los sistemas de administración de etiquetas simplifican el proceso de adición de etiquetas a su sitio, lo que requiere que solo proporcione el identificador de etiqueta y el tipo de evento de etiqueta.
Etiqueta marketplace Las plantillas de mercado de etiquetas proporcionan fragmentos de código para implementar una etiqueta. Si bien esto reduce la cantidad de conocimientos de código necesarios, se requiere cierta familiaridad con JavaScript y HTML para una implementación satisfactoria.
Tracker Los rastreadores son objetos que recopilan, almacenan datos y envían esos datos a Google Tag Manager.
Desencadenar Los activadores indican cuándo se debe agregar un código de etiqueta a una página. Se trata de condiciones definidas por el usuario.

Empezar

Para configurar Google Tag Manager, siga estos pasos:

Pasos

  1. Revise la documentación de Google para Configurar e instalar Tag Manager.

  2. En Administrador de etiquetas , cree una cuenta o use una cuenta existente. Para crear una nueva cuenta, seleccione el Crear una cuenta botón.

    De forma predeterminada, se crea un nuevo contenedor para una nueva cuenta. Puede crear contenedores adicionales dentro de cada cuenta.

  3. Ingrese valores para la página de su nueva cuenta, incluido el nombre de la cuenta y el nombre del contenedor. Seleccione una plataforma de destino.

  4. Selecciona el Crear botón.

Revisar eventos para rastrear

Revise la tabla a continuación y determine qué eventos desea rastrear.

Tipo Evento Descripción
Carga del reproductor bcvideo_player_load El reproductor se ha cargado y está listo para reproducirse.
Impresión de vídeo bcvideo_video_impression El video se cargó en un reproductor y está listo para reproducirse
Vista de video bcvideo_video_view El video ha comenzado a reproducirse a través de la reproducción automática o debido a la interacción del usuario
Participación de vídeo bcvideo_video_engagement Un usuario vio un rango de segundos de la línea de tiempo de un video. Este evento es un latido para rastrear la interacción de vídeo y es probable que se envíe muchas veces durante la reproducción, según la interacción del usuario y la duración del vídeo.
Solicitud de juego bcvideo_play_request La reproducción se inicia cuando el usuario hace clic expresamente en el botón de reproducción o automáticamente cuando la plataforma activa la reproducción en un escenario de reproducción automática.
Inicio de anuncio bcvideo_ad_start El control es entregado a un agente publicitario por la plataforma de reproducción.
Fin de anuncio bcvideo_ad_end El control se devuelve del agente publicitario a la plataforma de reproducción.
Error bcvideo_error Ha ocurrido un error fatal que interrumpe la experiencia de reproducción
Video completo bcvideo_video_complete Finaliza la reproducción de video

Creación de variables, etiquetas y disparadores

Cada evento de Brightcove que desee rastrear debe crearse como una etiqueta de evento en Google Tag Manager (GTM). Los ID de eventos de GTM se utilizarán dentro del módulo de Jugadores. Esto permite que los datos de eventos de Brightcove se pasen a los eventos de GTM.

Para cada etiqueta de evento, debe crear un activador GTM para enviar el evento a Google Analytics. Luego, en el GTM Variables pestaña, creará los parámetros personalizados asociados con cada evento.

Agregar variables a GTM

Los parámetros de Brightcove transportan datos adicionales y se transmiten con eventos. Cada parámetro de Brightcove que se registrará debe crearse como una variable en Google Tag Manager. El nombre del parámetro se utilizará dentro del reproductor, lo que permitirá que los parámetros de Brightcove se pasen a las variables de Google Tag Manager.

Para crear variables en Google Tag Manager, siga estos pasos

  1. Inicie sesión en su cuenta de Google Tag Manager

  2. Seleccione Variables en la navegación del lado izquierdo.

  3. En el Variables definidas por el usuario sección, seleccione la Nuevo botón.

  4. Para tipo de variable , Seleccione Variable de capa de datos.

  5. Ingrese los detalles:

    • Nombre de variable de capa de datos - Nombre de la variable
    • Versión de la capa de datos - Versión 2

  6. Selecciona Guardar.

  7. Parámetros de Brightcove que puede crear como variables de GTM:

    • Cuenta - Nombre de la cuenta de Brightcove
    • Jugador - ID de jugador
    • Nombre del jugador - Nombre del jugador
    • Versión de la plataforma - Número de versión de la plataforma
    • Sesión - Identificador de sesión
    • Video - ID de video
    • Nombre del video - Nombre del video
    • Distancia - El rango de segundos visualizados en el formato StartSecond..EndSecond
    • Segundos de video vistos - Cuenta de segundos vistos
    • Duración del video - La duración del video en segundos.
    • Porcentaje de video visto - Porcentaje de video visto
    • Código de error - Código de error específico de la plataforma asociado con el evento
    • Hito del video - Hito (25%, 50%, 75%, 100%) completado por el video
    • Cabezal de reproducción de video - La hora en el video en la que se activa el evento.
    • País - Código de país ISO-3166 (alfa 2), por ejemplo: nosotros
    • Nombre del país - Nombre de país legible por humanos, por ejemplo: Estados Unidos
    • Región - Código de región ISO-3166 (alfa 2)
    • Nombre de la región - Nombre de la región legible por humanos
    • Ciudad - Nombre de la ciudad
    • Navegador - Nombre completo y versión del navegador utilizado
    • Tipo de navegador - Nombre común del navegador utilizado (es decir, Chrome, Edge, Firefox)
    • Destino - URI que originó el evento
    • Fuente - URI que envió al usuario final al destino
    • Tipo de dispositivo - Anular para especificar el sistema operativo del dispositivo que originó el evento en los casos en que el agente de usuario no es confiable
    • SO del dispositivo - El nombre del sistema operativo que está utilizando el dispositivo
    • Versión del SO del dispositivo - La versión del sistema operativo que utiliza el dispositivo

Agregar etiquetas a GTM

Después de determinar qué eventos desea rastrear, las etiquetas deben crearse en Google Tag Manager.

Para crear etiquetas en Google Tag Manager, siga estos pasos

  1. Inicie sesión en su cuenta de Google Tag Manager

  2. Seleccione Etiquetas en la barra de navegación del lado izquierdo y seleccione Nuevo botón.

  3. Selecciona el Tipo de etiqueta como Google analitico: Universal Analytics.

  4. Ingrese los detalles de la etiqueta:

    • Tipo de pista - Ajustado a Evento
    • Categoría - Variable de categoría de evento
    • Acción - Variable de acción de evento
    • Etiqueta - Variable de etiqueta de evento
    • Valor - Nulo
    • Hit sin interacción - Ajustado a Cierto
    • Configuración de Google Analytics - Configuración predeterminada
    • ID de rastreo - ID de seguimiento del panel de GA. Seleccione Variable de ID de seguimiento.

  5. Vaya a Configuración avanzada y establezca los índices de dimensión personalizada y los valores de dimensión relacionados. Estos índices deben coincidir con los de Google Analytics.

  6. Sus etiquetas deberían tener un aspecto similar a esto en GTM:

Agregar activadores a GTM

Para crear activadores en Google Tag Manager, siga estos pasos

  1. Inicie sesión en su cuenta de Google Tag Manager

  3. Selecciona el Tipo de disparador como Evento personalizado.

  4. Ingrese el nombre del evento.

  5. Cree un disparador para cada evento del que desee realizar un seguimiento.

Integración con Brightcove

Para integrar GTM con Brightcove Player, agregue el complemento Google Tag Manager a su reproductor.

Uso de Studio

En los siguientes pasos se detalla el uso de Video Cloud Studio para cargar el complemento.

  1. En Video Cloud Studio, abra el JUGADORES módulo y cree un nuevo reproductor o localice el reproductor al que desea agregar el complemento.
  2. Selecciona el enlace del reproductor para abrir las propiedades del reproductor.
  3. Selecciona Plugins en el menú de navegación izquierdo.

  4. A continuación, seleccione el Agregar un complemento y luego seleccione Complemento Brightcove.

  5. Ampliar la Complemento Brightcove desplegable y seleccione Administrador de etiquetas de Google.

  6. Para decirle al jugador qué eventos y dimensiones personalizadas desea rastrear, cree un objeto JSON para pegarlo en el Opciones (JSON) caja de texto.

    Descripciones de campos:

    • tracker- Su ID de seguimiento de Google Tag Manager (GTM-XXXXXXX)
    • eventsToTrack- Eventos de los que desea realizar un seguimiento
    • paramsToTrack- Dimensiones personalizadas que desea rastrear

    He aquí un ejemplo:

    "bcvideo_player_name":"player_name",
    "bcvideo_video":"video",
    "bcvideo_video_name":"video_name",
	"bcvideo_session":"session",
    "bcvideo_platform_version":"platform_version",
	"bcvideo_range":"range",
    "bcvideo_video_duration":"video_duration",
    "bcvideo_video_percent_viewed":"video_percent_viewed",
    "bcvideo_error_code":"error_code",
    "bcvideo_video_milestone":"video_milestone",

    "video_impression": "Video Impression",

    Asegúrese de que sus dimensiones personalizadas en el objeto JSON coincidan con el nombre y el índice dentro de Google Analytics.

  7. Selecciona el Ahorrar botón. Ahora verá el Administrador de etiquetas de Google complemento agregado a la lista de complementos para su reproductor.

  8. Para publicar el reproductor, seleccione Publicar e incrustar > Publicar cambios.
  9. Para cerrar el cuadro de diálogo, seleccione Cerrar.

  10. Volver a la MEDIOS DE COMUNICACIÓN y publique su video o lista de reproducción con el reproductor que acaba de actualizar para Google Tag Manager.

    Cuando el reproductor se carga y el usuario interactúa con el reproductor, se realizará un seguimiento de los eventos y parámetros especificados.

Creando dimensiones personalizadas

Las dimensiones personalizadas son métricas que crea usted mismo en Google Analytics. Para obtener más detalles, consulte el Dimensiones y métricas personalizadas documento.

Con Google Scripts, puede automatizar el proceso de agregar dimensiones personalizadas en Google Analytics.

Pasos

Siga estos pasos para utilizar un script de aplicación de Google para crear dimensiones personalizadas:

  1. Inicie sesión en Google.

  2. Navega al Centro de desarrolladores de G Suite y seleccione el nuevo proyecto en blanco de Apps Script.

  3. Seleccione Proyecto sin título y dale un nombre a tu proyecto.

  4. Copie el siguiente código de secuencia de comandos de la aplicación personalizada en un editor y actualice los siguientes valores:

    • accountId- Su ID de cuenta de Google Analytics
    • webPropertyId- ID de seguimiento de su cuenta de Google
    • dimensions- Las dimensiones personalizadas que se agregarán. La lista actual contiene todas las dimensiones personalizadas. Observe que las dimensiones tienen el prefijo bcvideo_. Esto es para asegurar que no haya conflictos de nombres con estas variables. 
    //Set values for GA Account ID and web property ID to which you have to set custom dimensions.
var accountId='your account id';
var webPropertyId='your tracking code';
//Add required custom dimensions to be created to this list
var dimensions=['bcvideo_video_seconds_viewed','bcvideo_video_playhead','bcvideo_account','bcvideo_player','bcvideo_player_name','bcvideo_video','bcvideo_video_name','bcvideo_session','bcvideo_platform_version','bcvideo_range','bcvideo_video_duration',
'bcvideo_video_percent_viewed','bcvideo_error_code','bcvideo_milestone','bcvideo_destination'];


//Function to create custom dimensions in GA
function createGACustomDimensions() {

  //This loop iterates through the custom dimensions in order to set the custom dimension scope, active values into our GA property
  for(var index=0;index<dimensions.length;index++){
  var customDimensions ={
    'name': dimensions[index],
    'scope':'HIT',
    'active':true,
  }

  //This create a request to the new custom dimension
  Analytics.Management.CustomDimensions.insert(customDimensions,accountId,webPropertyId);

  // Pause for 1 second between adding each custom dimension, this ensure that the GA property has time
  // to process the request fully before moving to the next custom dimension.
  Utilities.sleep(1000)
  }
}
    En este ejemplo, estamos poniendo las dimensiones en un objeto llamado customDimensions. Para agregarlos como variables de capa de datos, debe hacer referencia a cada nombre de variable como customDimensions.{dimension}

  5. Reemplace la function myFunction(){} bloque de código, como se muestra a continuación, con la secuencia de comandos de su aplicación personalizada. 

    function myFunction() {

}
  6. Guarde el proyecto. Debe guardar el proyecto para poder utilizar la API de Google Analytics.

  7. Para habilitar la API de Google Analytics, en el editor de secuencias de comandos, seleccione Recursos > Servicios avanzados de Google.

  8. Entonces busca Servicios avanzados de Google y darle la vuelta en. Seleccione OK.

  9. Para ejecutar el script de su aplicación, seleccione Correr > Función de ejecución y elija su guión.

  10. En el cuadro de diálogo de autorización, seleccione Revisar permisos.

  11. Selecciona el Permitir botón.

  1. Inicie sesión en Google Analytics.

  2. Seleccione Administración

  3. Haga clic en Definiciones personalizadas para expandirlo, luego seleccione Dimensiones personalizadas.

  4. Debería ver las dimensiones personalizadas junto con sus índices asociados.

Eventos y parámetros

Los siguientes eventos y parámetros están disponibles para enviar a Google Tag Manager.

Eventos

Estos eventos vienen de tu jugador.

  • player_load
  • video_impression
  • play_request
  • video_view
  • video_engagement
    • video_engagement- Hito del 25%
    • video_engagement- 50% de hito
    • video_engagement- Hito del 75%
  • ad_start
  • ad_end
  • error
  • video_complete- Hito 100%

El porcentaje de video visto (video_engagement ) se puede calcular de la siguiente manera (similar al evento video_engagement en Google y Adobe Analytics): 

Video watched percentage = duration the user actually watched / total video duration

Por ejemplo, en un vídeo de 20 segundos, se puede alcanzar el hito del 25% en los dos casos siguientes

  • Relojes de usuario de 0 a 5 segundos
  • El usuario mira de 0 a 1 segundo y salta al 11º segundo y mira al 15 segundo

En los dos casos anteriores, el usuario observó 5 segundos, por lo que el hito se informará como 25%.
