    Implementación de anuncios del lado del servidor con Brightcove Player

    En este tema, aprenderá a utilizar la inserción de anuncios del lado del servidor (SSAI) con Brightcove Player para entregar anuncios cosidos en sus transmisiones de vídeo.

    Resumen

    Este ejemplo y el código asociado se proporcionan como guía para su desarrollo de producción. El código no está certificado para uso en producción ni es compatible oficialmente.

    La inserción de anuncios en el lado del servidor (SSAI) te permite incrustar anuncios en tus vídeos para que no puedan ser bloqueados por bloqueadores de anuncios en el navegador. Dynamic Delivery es el sistema de ingesta y entrega de próxima generación que reduce el espacio de almacenamiento y empaqueta dinámicamente los medios.

    De forma predeterminada, SSAI obliga a ver todos los anuncios, mostrando un temporizador de cuenta atrás de anuncios mientras se reproducen. Puede personalizar fácilmente esta función para omitir anuncios.

    Puede configurar Brightcove Player para que use anuncios del lado del cliente cuando no estén bloqueados y conmutar automáticamente por error a SSAI cuando se detecte un bloqueador de anuncios. Para obtener más información sobre cómo habilitar esta función, consulte el documento de conmutación por error de anuncio .

    Notas:

    • Para utilizar SSAI, su cuenta de Video Cloud debe estar configurada para Entrega dinámica y habilitada para SSAI. Póngase en contacto con su administrador de cuentas para empezar a usar esta función.
    • SSAI para reproducción en vivo no requiere el complemento SSAI, y las características del lado del cliente para la reproducción SSAI no están disponibles para la reproducción en directo. Para obtener más información, consulte el documento de Brightcove Live API with SSAI .
    • Para obtener una lista de limitaciones, consulte el documento Visión general de SSAI .

    Para obtener una descripción general del vídeo, consulte lo siguiente:

    Ejemplo de reproductor

    En este ejemplo se utilizan anuncios IMA definidos en un archivo VMAP para entregar anuncios del lado del servidor en la transmisión de vídeo. Deberías ver que hay un anuncio de pre-roll, mid-roll y post-roll. El archivo VMAP se define en la configuración del anuncio.

    Vea el plugin Pen 18468-anunciar-ssai-ssai-de Brightcove Learning Services (@bcls1969 ) en CodePen.

    Ver el código fuente.

    Funciones

    Brightcove SSAI es un reemplazo moderno para la publicación de anuncios Once UX. Las características incluyen:

    • Analizar VMAP/VAST más completo
    • Soporte completo para anuncios complementarios de VAST
    • Nuevas API para interactuar con la línea de tiempo y los rollos de anuncios lineales
    • Compatibilidad con listas de reproducción, macros de anuncios y FairPlay
    • SSAI funciona con contenido DRM y no DRM.
    • Una de las funciones principales de SSAI es aplicar ciertos comportamientos en torno a la búsqueda y los anuncios.
    • Compatibilidad doble con vMaps Once UX heredados (utilizando el espacio de nombres uo) y los nuevos VMaps de Dynamic Delivery (utilizando el espacio de nombres bc)

    Introducción

    Para reproducir anuncios del servidor desde Video Cloud, siga estos pasos:

    1. Crear una configuración de anuncio
    2. Crear un Brightcove Player
    3. Implementar SSAI usando Studio

    Eso es todo. El Brightcove Player ahora está configurado para los anuncios del lado del servidor. Si lo prefiere, puede agregar SSAI mediante programación, como se muestra en la sección Implementar SSAI mediante programación .

    Crear una configuración de anuncio

    La configuración del anuncio define varios aspectos de la reproducción de SSAI, incluidas las llamadas a anuncios, los indicadores y otras opciones de configuración. La respuesta al anuncio puede ser reglas de anuncios VAST, VMAP o DFP. Para crear una configuración de anuncio, siga estos pasos:

    1. En Video Cloud Studio, expanda el menú ADMIN y seleccione Configuración de anuncios del lado del servidor.

    2. Añade información para la configuración de tu anuncio y selecciona Guardar.

      3. https://studio.support.brightcove.com/admin/configuring-server-side-ad-settings.html

    Para obtener más información, consulte el documento Configuración de la configuración de anuncios del lado del servidor .

    Para crear una configuración de anuncio mediante la API SSAI, consulte el documento de la API de configuración de anuncios SSAI de Video Cloud.

    Crear un reproductor de Brightcove

    Cree un nuevo Brightcove Player con Video Cloud Studio. Para obtener más información, consulte los conceptos básicos de Video Cloud: Creación de un documento del Reproductor .

    Debe utilizar una versión 6.8.0+ de Brightcove Player.

    Implementación de SSAI con Studio

    La forma más sencilla de configurar el reproductor para anuncios del lado del servidor es con Video Cloud Studio. Una vez que haya creado una configuración y un reproductor de anuncios, estará listo para configurar el reproductor para SSAI de la siguiente manera:

    1. Abra el módulo PLAYERS y localice el reproductor al que desea añadir la funcionalidad de publicidad.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. Seleccione Publicidad en el menú de navegación de la izquierda.
    4. Marque la casilla de verificación Habilitar servidor (SSAI) .
    5. En el menú desplegable Seleccionar configuración , seleccione la configuración del anuncio que desea asociar a este reproductor.

    6. Si desea que las superposiciones se muestren sobre sus anuncios, active la casilla de verificación Habilitar superposiciones de información de anuncios. Esto incluye las superposiciones «Más información» y la cuenta regresiva de anuncios.

      Las superposiciones no se mostrarán hasta que las habilite aquí y las configure en su etiqueta de anuncio de VAST.

    7. El formulario completo debe ser similar a este:

    8. Seleccione Guardar.
    9. Para publicar el reproductor, seleccione Publicar e incrustar > Publicar cambios.

    Cuando se guardan los cambios en las propiedades publicitarias, el plugin SSAI se configurará como parte de la configuración del complemento. El JavaScript y CSS estarán ocultos ya que los agregó a través de la sección Publicidad .

    Reproducir un vídeo con anuncios

    Cualquier vídeo que recupere de Video Cloud que se haya ingerido con Entrega dinámica incluirá los anuncios especificados en el archivo VMAP en la configuración del anuncio. Tenga en cuenta que el vídeo necesita tener una pista de audio asociada a él para que SSAI funcione.

    Implementación de SSAI programáticamente

    Puede agregar SSAI fácilmente a su reproductor mediante Video Cloud Studio, como se muestra en la sección anterior. Si prefiere hacerlo mediante programación, siga estos pasos:

    1. Agregar el complemento SSAI
    2. Asociar anuncios con tu reproductor
    3. Reproducir un vídeo con anuncios

    Agregar el complemento SSAI

    Puede agregar los archivos del plugin SSAI a su código HTML, o puede agregarlo a la configuración de su reproductor como se muestra aquí:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Haga clic en el enlace del reproductor para abrir las propiedades del reproductor.
    3. Haga clic en Plugins en el menú de navegación izquierdo.
    4. A continuación, haga clic en Plugins de las tres opciones.
    5. En Agregar un complemento , seleccione Plugin personalizado.
    6. Para el nombre del complemento , escriba ssai.
    7. Para la URL de JavaScript , escriba: 
      https://players.brightcove.net/videojs-ssai/1/videojs-ssai.js
    8. Para la URL CSS , escriba: 
      https://players.brightcove.net/videojs-ssai/1/videojs-ssai.css
    9. Usted no tiene que introducir ninguna opción para este plugin.
    10. Haga clic en Guardar.
    11. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    12. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.

    Asociar anuncios con tu reproductor

    A continuación, asociará la configuración de su anuncio a su Brightcove Player. Puede hacer esto de una de tres maneras:

    • Código de inserción estándar
    • Código de incrustación avanzado
    • Uso del catalog
    • Actualización de la configuración del reproductor

    Código de inserción estándar (iframe)

    Con el código de inserción estándar, incluya el parámetro de cadena de adConfigId consulta con el valor de su ID de configuración de anuncio:

    <iframe src="https://players.brightcove.net/1752604059001/default_default/
index.html?videoId=5625751316001&adConfigId=your ad config id"
	allowfullscreen
	webkitallowfullscreen
	mozallowfullscreen
	width="640"
	height="360"></iframe>

    Código de incrustación avanzado

    Con el código de inserción avanzado, incluya el data-ad-config-id atributo con el valor de su ID de configuración de anuncio:

    <video-js data-video-id="5625751316001"
  data-account="1752604059001"
  data-player="default"
  data-embed="default"
  data-application-id
  data-ad-config-id="your ad config id"
  class="video-js"
  controls
  width="640"
  height="360"></video-js>
<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

    Uso del catalog

    Puedes usar el player catalog para asociar anuncios a tu vídeo. El catalog funciona haciendo dos solicitudes:

    1. Solicite datos de vídeo del catalog del reproductor. Esto incluirá una URL de VMAP.
    2. El origen del reproductor se establece con la dirección URL de VMAP, lo que desencadena una solicitud de un documento VMAP desde Dynamic Delivery. El origen del reproductor se establece de nuevo con un documento XML VMAP válido.

    Cuando utilice catalog con SSAI, añada el identificador de configuración del anuncio a la getVideo() llamada a la Playback API de la siguiente manera:

    var adConfigId = "your ad config id";
var myPlayer = videojs.getPlayer('myPlayerId');
// If you added the SSAI plugin using the Players module, then the initialization
// step is performed automatically. Uncomment the next line if you
// did not use the Players module.
//myPlayer.ssai();

myPlayer.catalog.getVideo("your video id", function(error, video) {
	if (error) {
		myPlayer.error(error);
	} else {
		myPlayer.catalog.load(video);
	}
}, adConfigId);

    Actualización de la configuración del reproductor

    Otra forma de asociar anuncios es incluir su ID de configuración de anuncio en la configuración de Brightcove Player. Para ello, puede utilizar la API de administración de jugadores de la siguiente manera:

    1. Utilice el PATCH comando para incluir su ad_config_id. A continuación se muestra un ejemplo de actualización de su reproductor usando cURL:

      curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
      "ad_config_id" : "$CONFIG_ID"
  }' \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

    2. A continuación, puede publicar los cambios de la siguiente manera:

      curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request POST \
  https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players/$PLAYER_ID/publish

    3. Compruebe que la configuración de su Brightcove Player ad_config_id incluye el valor de ID de configuración y el plugin ssai con los archivos de plugin asociados. Para ello, siga estos pasos:

      1. Desplácese hasta el reproductor en el módulo Studio Players. Haga clic en el enlace del nombre del jugador para ver los detalles.
      2. Seleccione el código incrustado y la URL (ya sea Vista previa o Publicada funcionarán). Haga clic en el enlace URL del reproductor .
      3. En la barra de direcciones del navegador, cambie index.html a config.json y busque la nueva URL.

      La configuración del reproductor debe ser similar a la siguiente:

      {
  "account_id": "1752604059001",
  "ad_config_id": "d6190656-2095-4ff3-8afe-123abcde",
  "compatibility": true,
  "embed_id": "default",
  "player": {
    "template": {
      "name": "single-video-template",
      "version": "6.9.0"
    }
  },
  "player_id": "rJCECV0RZ",
  "player_name": "SSAI Player",
  "plugins": [
    {
      "name": "ssai"
    }
  ],
  "scripts": [
    "https://players.brightcove.net/videojs-ssai/1/videojs-ssai.js"
  ],
  "stylesheets": [
    "https://players.brightcove.net/videojs-ssai/1/videojs-ssai.css"
  ],
  "updated_at": "2017-11-07T16:03:47.161Z",
  "video_cloud": {
    "policy_key": "ABCDE123456789",
    "video": null
  }
}

    Reproducción de un vídeo con anuncios

    Cualquier vídeo que recupere de Video Cloud que se haya ingerido con Entrega dinámica incluirá los anuncios especificados en el archivo VMAP en la configuración del anuncio. Tenga en cuenta que el vídeo necesita tener una pista de audio asociada a él para que SSAI funcione.

    Opciones

    • debug
      • Si es true, configura los mensajes de depuración en contrib-ads y registra información adicional en presencia de videojs-bc-analytics-logger.
    • hideOverlays
      • Si true, el temporizador de cuenta atrás y el clic de Más información a través de superposiciones no se mostrarán mientras se reproducen los anuncios.
    • trackingBeacons
      • Si es false, no se enviarán las balizas de seguimiento analizadas desde el VMAP para la vista del anuncio, la impresión, los cuartiles, etc.
    • timeout
      • El número de milisegundos después de los cuales un XHR para obtener un VMAP se agotará.

    Estilización

    Hay varias clases HTML útiles aplicadas al reproductor por este plugin que puede ser dirigido para determinar el estado del plugin.

    Clase Uso
    vjs-ssai Indica que se ha creado una instancia del complemento SSAI, pero no está necesariamente habilitado. Esto estará presente incluso cuando no se reproduzca una fuente SSAI.
    vjs-ssai-enabled El plugin SSAI está habilitado actualmente. En otras palabras, se ha establecido una fuente SSAI en el reproductor.
    vjs-ssai-disabled El complemento SSAI no está habilitado actualmente.
    vjs-ssai-waiting El plugin SSAI está esperando datos o algún otro proceso externo.
    vjs-ssai-not-waiting El plugin SSAI no está esperando nada.
    vjs-ssai-hide-overlays La hideOverlays opción se ha establecido en true.
    vjs-ssai-show-overlays La showOverlays opción está establecida en true. Este es el valor predeterminado.

    Métodos/propiedades

    Varios métodos públicos están disponibles cuando se utiliza el plugin SSAI. Recuerde, también puede usar los métodos estándar del reproductor.

    Para usar los métodos SSAI, su código JavaScript tendrá que esperar a que se cargue el complemento, como se muestra aquí:

    <script>
	videojs.getPlayer('myPlayerID').ready(function () {
		var myPlayer = this;
		myPlayer.on("loadedmetadata", function () {
			console.log(myPlayer.ssai().currentTimelineState());
		});
	});
</script>

    Métodos de estado de línea de tiempo

    Los *TimelineState métodos devuelven un TimelineState objeto. Un objeto de estado de línea de tiempo es un objeto simple con una interfaz específica que detalla el estado de reproducción de una secuencia SSAI en cualquier punto de la línea de tiempo absoluta.

    El «tiempo absoluto» se refiere a un punto de la línea de tiempo completa de una transmisión (con anuncios y contenido incluidos). El término «tiempo relativo» se refiere al tiempo relativo al contenido actual (ya sea anuncio o contenido).

    contentTimelineState()

    • Argumento: numérico único time
    • Obtener el estado de la línea de tiempo a la hora del contenido especificada (contenido, sin contar anuncios). Por ejemplo, si el contenido tiene una duración de 100 segundos, puede obtener el estado de la línea de tiempo a mitad del contenido utilizando: 
      contentTimelineState(50)

    absoluteTimelineState()

    • Argumento: numérico único time
    • Obtener el estado de la línea de tiempo a la hora absoluta especificada (contenido + anuncios). Por ejemplo, si el contenido incluye un anuncio pre-roll de 30 segundos, puede obtener el estado de la línea de tiempo en 30 segundos en el contenido mediante: 
      absoluteTimelineState(50)

    Aquí hay una lista de métodos adicionales de estado de línea de tiempo:

    currentTimelineState()
    relativeTimelineState()
    seekInAbsoluteTime()
    seekInRelativeTime()
    seekInContentTime()

    Propiedades del estado de la línea de tiempo

    Para obtener el valor de una propiedad de estado de línea de tiempo, puede utilizar la sintaxis siguiente:

    myPlayer.ssai().absoluteTimelineState().absoluteDuration;

    Notas de configuración

    1. La precarga de anuncios no se debe hacer con SSAI. La razón de esto es que si precarga el reproductor informará de una impresión de anuncio y probablemente las primeras balizas de cuartil antes de que se reproduzca el vídeo. Esto podría dar lugar a análisis publicitarios inexactos. Si configura SSAI en Studio, esto se hará automáticamente, pero si configura SSAI manualmente, debe ser consciente de este problema.
    2. Si el reproductor web utiliza SSAI, y una de sus motivaciones para hacerlo es evitar los bloqueadores de anuncios, debe usar balizas del lado del servidor. Las balizas del lado del cliente no deben usarse, ya que se bloquearán.

    Glosario

    Este plugin distingue los conceptos de tiempo absoluto y contenido dentro de una secuencia SSAI. Los reproductores de vídeo tradicionales solo tienen un concepto de tiempo de contenido ; tiempos entre el inicio y el final del URI que se está reproduciendo actualmente. Debido a que una transmisión SSAI es esencialmente una serie de flujos de contenido cosidos juntos, hemos introducido el concepto de tiempo absoluto que tiene en cuenta el flujo cosido completo incluyendo los anuncios de vídeo.

    Cuando vea el prefijo absoluto en una propiedad o método, las horas esperadas/devueltas son relativas a toda la secuencia cosida. Cuando vea el contenido del prefijo, los tiempos esperados/devueltos son relativos sólo a un fragmento particular de contenido que fue cosido en la secuencia (el contenido principal o anuncio lineal único).

    • Hora absoluta : Se refiere a cualquier punto dado en la línea de tiempo total de una secuencia de SSAI. Por ejemplo, un vídeo a las 2:00 con un anuncio previo a las 0:30 tiene un tiempo absoluto total de 2:30. El tiempo absoluto de 0:15 está en el pre-roll y el tiempo absoluto de 0:31 es el primer segundo de contenido.

    • Tiempo relativo : Hace referencia al tiempo relativo al bloque actual de medios, ya sea contenido o anuncio. Expandiendo en lo anterior, durante el pre-roll, el tiempo relativo 0:15 es sinónimo del tiempo absoluto de 0:15 , pero el tiempo absoluto 0:31 equivaldría a un tiempo relativo de 0:01.

      Generalmente, el tiempo relativo es lo que ves en la interfaz de usuario del reproductor y gran parte del trabajo de este plugin y middleware asociado se traduce de tiempo absoluto a tiempo relativo.

    • El tiempo de contenido se refiere a un punto en el tiempo dentro de la línea de tiempo de contenido de una transmisión de SSAI, ignorando todos los anuncios. Por ejemplo, un vídeo a las 2:00 con un anuncio previo a las 0:30 tiene una hora de contenido de 2:00. El tiempo de contenido de 0:15 es equivalente a un tiempo absoluto de 0:45 (pre-roll más 15 segundos).

    Problemas conocidos

    Estos son los problemas conocidos para usar el complemento SSAI:

    • Safari 10/11 a veces muestra el último fotograma de la postroll al final del video.
    • SSAI no cose los anuncios superposición en la transmisión de vídeo.
