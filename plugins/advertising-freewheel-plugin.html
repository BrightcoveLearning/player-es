    Publicidad con el plugin FreeWheel

    En este tema, aprenderá cómo usar el plugin FreeWheel para habilitar anuncios con reproducción de vídeo, y cómo se puede implementar usando Studio y mediante codificación personalizada.

    Resumen

    Este plugin habilita la tecnología de anuncios FreeWheel, ya sea para Flash o HTML5, en Brightcove Player. La opción de renderizado predeterminada para el plugin FreeWheel está establecida en Html5 (para obtener más información consulte la adTechOrder opción a continuación).

    Uso de versiones del plugin

    Cuando utilice el plugin FreeWheel necesitará usar URL versionadas.

    Versión Tipo URL versionada
    Última versión JS https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
    CSS https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css
    SWF * https://players.brightcove.net/videojs-freewheel/3/videojs.freewheel.swf

    Como referencia, las versiones del complemento y los SWF compatibles se rastrean en el documento de referencia de versión del complemento .

    Comprobación del servidor de anuncios

    Lo primero que debe hacer es verificar la validez de la etiqueta de anuncio que está planeando utilizar. Asegúrese de haber copiado su URL y vaya a la siguiente página: Ad Previewer (al hacer clic en este enlace, la página se abrirá en una nueva ventana o pestaña).

    Pegue la URL de la etiqueta de anuncio en el campo de entrada del formulario indicado. Haga clic en OPEN IN AD PREVIEWER. Se Open In Ad Previewer mostrará una ventana emergente titulada y, a continuación, haga clic en el OPEN botón para probar su anuncio. Haga clic en PLAY y, si funciona correctamente, el anuncio se mostrará intercalado con un vídeo. Si la etiqueta de anuncio no funciona en este entorno de prueba, no funcionará con Brightcove Player.

    Uso de la URL del SDK correcta

    Antes de la versión 6.24 del SDK FreeWheel proporcionaba una URL de «actualización automática» a la última versión del SDK JS AdManger. Esta URL siempre apuntaba a la versión más actual del SDK. Este ya no es el caso.

    Las dos secciones siguientes ofrecen alternativas a la implementación de FreeWheel. En ambas secciones utilizará un sdkurl. En el pasado, simplemente podía usar http://adm.fwmrm.net/p/[]/AdManager.js para el valor de la opción, y saber que apuntaría a la última versión del SDK de FreeWheel. Ahora tendrá que ingresar una URL que apunte a una versión específica, como https://mssl.fwmrm.net/libs/adm/6.24.0/AdManager.js. Además, después de la configuración inicial, también tendrá que actualizar ese valor cuando FreeWheel libere una versión más reciente del SDK, si desea estar en la última versión. Tenga en cuenta que después de realizar este cambio, tendrá que volver a publicar su Brightcove Player.

    Si no actualiza la URL y continúa utilizando la URL de actualización automática, siempre apuntará a la versión 6.23.

    Es posible que desee administrar su propia versión de SDK en su integración con Brightcove Player. Esto implica pasar la opción al plugin FreeWheel:Html5.sdkurl

    player.FreeWheelPlugin({
  Html5: {
    sdkurl: 'preferred-sdk-url'
  }
});

    Implementar usando el módulo Players

    Para implementar el plugin FreeWheel utilizando el módulo Players, siga estos pasos:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Haga clic en el enlace del reproductor para abrir las propiedades del reproductor.
    3. Haga clic en Plugins en el menú de navegación izquierdo.
    4. A continuación, haga clic en Plugins de las tres opciones.
    5. En Agregar un complemento , seleccione Plugin personalizado.
    6. Para el nombre del complemento , escriba FreeWheelPlugin.
    7. Para la URL de JavaScript , escriba: 
      https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
    8. Para la URL CSS , escriba: 
      https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css
    9. Introduzca las opciones de configuración en el cuadro de texto Opciones (JSON) . Las opciones proporcionadas en el formulario configuran la tecnología de reproducción donde Flash y las secciones Html5 de configuración son proporcionadas por FreeWheel. A continuación se muestra una configuración completa ligeramente alterada (por razones de seguridad). Un ejemplo stub se muestra aquí: 
      {
  "adTechOrder": ["html5"],
  "Flash": {
    "sdkurl": "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    "serverUrl": "http://cue.v.fwmrm.net/..."
    "profile": "3aaa_profile",
    ...
  },
  "Html5": {
    "sdkurl": "https://mssl.fwmrm.net/libs/adm/{LATEST_VERSION}/...",
    "serverUrl": "http://cue.v.fwmrm.net/..."
    "profile": "3aaa_profile",
    ...
  }
}
    10. Haga clic en Guardar.
    11. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    12. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.

    https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.js
https://players.brightcove.net/videojs-freewheel/3/videojs-freewheel.min.css

    El nombre del plugin es FreeWheelPlugin. Las opciones configuran la tecnología de reproducción donde Flash y las secciones Html5 de configuración son proporcionadas por FreeWheel. A continuación se muestra una configuración completa ligeramente alterada (por razones de seguridad). Un ejemplo stub se muestra aquí:

    Configuración

    Los siguientes ajustes se utilizan para configurar el plugin FreeWheel:

    • adTechOrder:
      • Tipo: Array Valor predeterminado: ["html5"]
      • La matriz contiene las tecnologías de integración de anuncios que se deben intentar utilizar, en orden de prioridad descendente.
    • Flash:
      • Tipo:Object
      • Un objeto para la configuración de FreeWheel específica de tecnología (proporcionada por FreeWheel). El objeto contiene las siguientes opciones:
        Estas opciones se pueden configurar dinámicamente (se muestra aquí) o como parte de la configuración inicial (se muestra aquí).
      • swfurl:
        • Tipo: string Valor predeterminado: https://players.brightcove.net/videojs-freewheel/3/videojs.freewheel.swf (compatible con FW Flash SDK v6.4.6)
        • Los valores opcionales se pueden utilizar en función de FreeWheel Flash Plugin Version. Estos valores se pueden encontrar en la Using versions of the plugin sección situada cerca de la parte superior de este documento.
        • Debe ser una propiedad secundaria del objeto.Flash Esto se aplicará directamente al elemento objeto si se utiliza la tecnología de anuncios Flash.
      • autoplay:
        • Tipo: boolean Valor predeterminado: true
        • Indica si el contenido se reproduce automáticamente.
        • Debe ser una propiedad secundaria del objeto.Flash Esto se aplicará directamente al elemento objeto si se utiliza la tecnología de anuncios Flash.
      • unattendedPlay:
        • Tipo: boolean Valor predeterminado: false
        • Cuando autoPlay se establece en true, unattendedPlay indica si el usuario es consciente de la reproducción automática.
        • Debe ser una propiedad secundaria del objeto.Flash Esto se aplicará directamente al elemento objeto si se utiliza la tecnología de anuncios Flash.
      • visitorCustomId:
        • Tipo: string Valor predeterminado: unset
        • Permite a los editores pasar un identificador de visitante personalizado para la solicitud de anuncio. Cuando se establece, este valor se pasará en el bloque de visitantes de la carga útil de solicitud como customId="CustomID_001". El ID se devolverá en la respuesta a la solicitud de anuncio en la etiqueta de visitante de apertura como atributo: customId="CustomID_001".
        • Debe ser una propiedad secundaria del objeto.Flash Esto se aplicará directamente al elemento objeto si se utiliza la tecnología de anuncios Flash.
    • Html5:
      • Tipo:Object
      • Un objeto para la configuración de FreeWheel específica de tecnología (proporcionada por FreeWheel). El objeto contiene las siguientes opciones:
        Estas opciones se pueden configurar dinámicamente (se muestra aquí) o como parte de la configuración inicial (se muestra aquí).
      • autoPlayType:
        • Tipo: integer Valor predeterminado: 1
        • Indica que el vídeo de contenido se reproducirá automáticamente y el usuario sabe que el contenido se está reproduciendo automáticamente.
        • Los valores pasados a AutoPlayType establece indicadores en la URL de solicitud de anuncio enviada al servidor de anuncios FreeWheel y pueden ser uno de los tres valores enteros que equivalen a las constantes FreeWheel SDK relacionadas:
          • 1: tv.freewheel.SDK.VIDEO_ASSET_AUTO_PLAY_TYPE_ATTENDED: Los medios autoPlay son true y unattendedPlay son false. Esto establece la bandera = +play-uapl.
          • 2: tv.freewheel.SDK.VIDEO_ASSET_AUTO_PLAY_TYPE_UNATTENDED: significa autoPlay es true y unattendedPlay es también true. Esto establece la bandera = +play+uapl.
          • 3: tv.freewheel.SDK.VIDEO_ASSET_AUTO_PLAY_TYPE_NONE: significa autoPlay es false y unattendedPlay es unset. Esto establece la bandera = -play.
        • Debe ser una propiedad secundaria del objeto.Html5 Esto se aplicará directamente al elemento objeto si se utiliza la tecnología publicitaria Html5.
      • visitorCustomId:
        • Tipo: string Valor predeterminado: (optional)
        • visitorCustomId Esta opción permite a los editores pasar un identificador de visitante personalizado para la solicitud de anuncio. Cuando se establece, este valor se pasará a la URL de solicitud de anuncio como QueryString vcid=CustomID_001. El ID se devolverá en respuesta a la solicitud de anuncio en el bloque JSON visitante como "customId" : "CustomID_001".
        • Debe ser una propiedad secundaria del objeto.Html5 Esto se aplicará directamente al elemento objeto si se utiliza la tecnología publicitaria Html5.
      • subsessionToken:
        • Tipo: int Valor predeterminado: (optional)
        • La subsessionToken opción permite a los editores iniciar una subsesión con el token dado. Cuando se establece, este valor se pasará a la URL de solicitud de anuncio como QueryString: 
          ssto=1234567890
        • Debe ser una propiedad secundaria del objeto.Html5 Esto se aplicará directamente al elemento objeto si se utiliza la tecnología publicitaria Html5.
          Nota: subsessionToken Para que funcione correctamente, también debe incluir una capacidad en la solicitud de anuncio de [flag=+sync] como parámetro.
      • compatibleDimensions:
        • Tipo: array Valor predeterminado: (optional)

        • La compatibleDimensions opción permite a los editores pasar una matriz de dimensiones compatibles. Por ejemplo:

          compatibleDimensions: [ {width: A, height: B}, {width: X, height: Y} ]

          Cuando se establece, este valor se pasará a la URL de solicitud de anuncio como QueryString:

          cd=widthA,heightA|widthX,heightY
        • Debe ser una propiedad secundaria del objeto.Html5 Esto se aplicará directamente al elemento objeto si se utiliza la tecnología publicitaria Html5.
    • loadWithCookie:
      • Tipo: boolean Valor predeterminado: false
      • Habilita o deshabilita las cookies para las solicitudes VAST.
    • requestAdsMode:
      • Tipo: string Valor predeterminado: onload
      • Los valores posibles para la propiedad son:
        • onload: Los anuncios se solicitan cuando el reproductor está completamente cargado.
        • onplay: Los anuncios se solicitan cuando el vídeo comienza a reproducirse, ya sea iniciado por el usuario o reproducción automática.
        • oncue: Los anuncios se solicitan en la hora de inicio del punto de referencia. Al utilizar esta propiedad, también tenga en cuenta:
          • Sólo funcional cuando la useMediaCuePoints propiedad también está establecida en true.
          • Los puntos de referencia de anuncios de Video Cloud deben configurarse en el vídeo de contenido. Consulte el documento Trabajar con puntos de referencia en el módulo multimedia para obtener instrucciones sobre cómo crear puntos de referencia en el vídeo de contenido.
          • Si hay un temporalSlots opción bajo la Flash o Html5 properties, la configuración se ignora.
      Nota: Cuando se utilizan anuncios de puntos de referencia con el plugin FreeWheel, los anuncios postroll no son compatibles.
    • useMediaCuePoints:
      • Tipo: boolean Valor predeterminado: false
      • Debe establecerse en true para utilizar los puntos de referencia de anuncios de Video Cloud para activar anuncios. Debe utilizarse junto con la requestAdsMode: 'oncue' propiedad y el valor asignado.

    Macros de anuncios

    Hay macros de anuncios disponibles para facilitar el trabajo al configurar FreeWheel. Las macros de anuncios se sustituyen por los valores correspondientes en cualquier parte de la configuración.

    La siguiente es la lista completa de variables para las que se utilizarán los valores sustituidos:

    Macro Descripción
    {player.id} Identificador del reproductor
    {mediainfo.id} Id. del vídeo
    {mediainfo.name} Título del vídeo
    {mediainfo.description} Descripción corta (250 chars max)
    {mediainfo.tags} Etiquetas (metadatos) asociadas al vídeo
    {mediainfo.reference_id} Identificador de referencia
    {mediainfo.duration} Duración del vídeo según lo informado por Video Cloud
    {mediainfo.ad_keys} Cadena de texto de formulario libre que se puede agregar y editar en el módulo Media de Studio; debe usar el parámetro de consulta en el formulario 
    cust_params={mediainfo.ad_keys}
    {player.duration} Duración del vídeo medido por el reproductor (posiblemente ligeramente diferente de mediainfo.duration y probablemente más preciso)
    {document.referrer} URL de página de referencia
    {window.location.href} URL de página actual
    {player.url} URL del reproductor
    {marca de tiempo} Hora local actual en milisegundos desde 1/1/70
    {aleatorio} Un número aleatorio 0-1 billón

    Nota: Si desea utilizar el ID de vídeo como FreeWheel videoAssetCustomId, puede hacerlo utilizando la macro de {mediainfo.id} anuncios. Por ejemplo, en las configuraciones siguientes puede hacer lo siguiente:

    'videoAssetCustomId' = '{mediainfo.id}';

    Un problema de configuración de FreeWheel que debe tener en cuenta: Si el ID de vídeo no está configurado como Asset ID en la configuración de FreeWheel Ad Manager en el servidor, FreeWheel devolverá un error de INVALID_ASSET_CUSTOM_ID.

    Eventos

    El complemento emite algunos tipos de eventos personalizados durante la carga, inicialización y reproducción. Puede escuchar los eventos del marco publicitario como lo haría con cualquier otro evento:

    player.on('ads-ad-started', function(event) {
  console.log('event', event);
});
    Evento Se envía cuando:
    solicitud de anuncios A petición de datos publicitarios.
    carga de anuncios Cuando los datos del anuncio estén disponibles tras una solicitud de anuncio.
    ads-ad-iniciado Ha empezado a reproducirse un anuncio.
    ads-ad-finalizado Un anuncio ha terminado de reproducirse.
    pausa de anuncios Un anuncio está en pausa.
    anuncios de reproducción Un anuncio se reanuda a partir de una pausa.
    ads-primer cuartil El anuncio ha reproducido el 25% de su duración total.
    punto medio de anuncios El anuncio ha reproducido el 50% de su duración total.
    ads-tercer cuartil El anuncio ha reproducido el 75% de su duración total.
    anuncios y clic Un espectador hizo clic en el anuncio de reproducción.
    ads-volumechange Se ha cambiado el volumen del anuncio de reproducción.
    ads-pod-iniciada Ha comenzado el primer anuncio de un pod de anuncios lineal (un grupo secuenciado de anuncios).
    ads-pod-finalizado El último anuncio de un pod de anuncios lineal (un grupo secuenciado de anuncios) ha finalizado.
    ads-allpods-completado Todos los anuncios lineales han terminado de reproducirse.
    fw-antes-ad-solicitud Este evento se expone en el objeto del reproductor y se activa antes de enviar una solicitud de anuncio. Por lo general, se utiliza en el contexto de listas de reproducción, para realizar actualizaciones a los ajustes de configuración de FreeWheel a través de: player.FreeWheelPlugin.settings.Html5 o player.FreeWheelPlugin.settings.Flash

    Asignación dinámica de URL de servidor

    Puede utilizar el fw-before-ad-request evento para asignar dinámicamente la dirección URL del servidor. Utilizará el on() método para escuchar la solicitud de anuncio y, a continuación, asignar la URL del servidor deseada. Por supuesto, deberá proporcionar las URL de servidor deseadas para los marcadores de posición en el código:

    player.on('fw-before-ad-request', function () {
  player.FreeWheelPlugin.settings.Html5.serverUrl = '[your server url]'
  player.FreeWheelPlugin.settings.Flash.serverUrl = '[your server url]'
})

    Si ha configurado previamente las URL del servidor, el código que se muestra anulará la configuración anterior.

    Bibliotecas de anuncios del reproductor

    El repositorio de videojs/videojs-contrib-ads GitHub contiene un complemento que proporciona la funcionalidad común necesaria para las bibliotecas de anuncios de vídeo que trabajan con Brightcove Player. El complemento proporciona la funcionalidad común necesaria para las integraciones de anuncios de vídeo y se ocupa de una serie de problemas para los integradores de anuncios, lo que reduce el código que debe escribir para la integración de anuncios.

    Propiedades

    videojs-contrib-ads proporciona algunas propiedades que pueden ser útiles. A saber:

    Nombre Tipo de datos Descripción
    ads.ad.id Cadena Identificador único de un anuncio que se reproduce
    ads.ad.index Número Índice del anuncio que se reproduce en un momento especificado; el índice identificaría el valor ordinal de un anuncio en un pod de anuncios
    ads.ad.duration Número Duración del anuncio en segundos
    ads.ad.type Cadena O bien PREROLL, MIDROLL o POSTROLL
    ADS.AD.CurrentTime () Función Función que devuelve la hora actual de reproducción de anuncios

    Problemas conocidos

    Reutilizar dinámicamente el mismo reproductor con FreeWheel

    Si está cargando vídeos dinámicamente y reutilizando el mismo reproductor, los ajustes de configuración de FreeWheel contendrán los mismos valores que se establecieron en el momento en que se instanció el plugin FreeWheel, con dos excepciones. La primera excepción es si utiliza macros de anuncios. La segunda excepción es que puede actualizar la configuración usando player.FreeWheelPlugin.settings.Html5 o player.FreeWheelPlugin.settings.Flash en un fw-before-ad-request evento.

    Cambiar el tamaño del reproductor durante la reproducción de anuncios

    Si se cambia el tamaño del reproductor durante la reproducción de anuncios o vídeo, el contenido del anuncio no cambiará de tamaño a menos que se llame a la función de dimensiones del reproductor para cambiar el tamaño del reproductor. Cambiar el tamaño del reproductor utilizando otros métodos (por ejemplo: ancho y alto de estilo) no cambiará el tamaño del anuncio.

    Cuando se utiliza player.dimensions(width,height) para cambiar el tamaño del reproductor, tendrá que activar el fw-resizeplayer evento para que el plugin sepa que las dimensiones han cambiado. Esto se debe a que algunos anuncios de VPAID conservan las dimensiones predeterminadas del anuncio y no cambiarán automáticamente el tamaño cuando el reproductor cambie de tamaño.

    Un ejemplo:

    player.dimensions(960,540);
player.trigger('fw-resizeplayer');

    Anuncios de superposición

    Si el plugin FreeWheel está representando un anuncio en Flash, no se puede hacer clic en los anuncios superposición. Si el plugin FreeWheel está representando un anuncio en HTML5, las superposiciones no se muestran con las coordenadas correctas. Al hacer clic en un anuncio de superposición HTML5 no se detendrá el reproductor de contenido cuando se siga la URL de clic.

    FreeWheel Live

    Los anuncios FreeWheel en una transmisión en directo se pueden implementar con la ayuda de Brightcove Global Services (BGS).

    VPAID

    No es posible reanudar los anuncios a través de los controles VPAID si la barra de control de anuncios está habilitada.
