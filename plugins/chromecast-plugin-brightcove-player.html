    Complemento de Chromecast para Brightcove Player

    En este tema, aprenderás cómo enviar tus vídeos a un televisor conectado a Chromecast, utilizando el complemento Chromecast (2.x) para Brightcove Player.

    Resumen

    Google Chromecast es un dispositivo que se conecta al puerto HDMI del televisor. Con tu smartphone o ordenador como mando a distancia, puedes utilizar Chromecast para acceder al contenido de vídeo. El complemento Chromecast para Brightcove Player permite al reproductor transmitir vídeo desde el escritorio o el navegador Android Chrome al dispositivo Chromecast.

    El complemento es compatible con todos los vídeos de Video Cloud o transmisiones externas que utilizan HLS o DASH, incluidas las transmisiones cifradas mediante DRM mediante Widevine. La publicidad se admite mediante la inserción de anuncios del lado del servidor (SSAI) de Video Cloud. Los anuncios del lado del cliente y las listas de reproducción no son compatibles.

    Para utilizar el complemento siga estos pasos:

    1. Cargue el complemento Chromecast para Brightcove Player (detallado a continuación).
    2. Asegúrate de que el dispositivo Chromecast esté en la misma red que el dispositivo que reproducirá el vídeo.
    3. Empieza a reproducir el vídeo.
    4. Haz clic en el botón de lanzamiento y selecciona tu dispositivo Chromecast.

    Componentes de aplicación Chromecast

    El software detrás de Chromecast se compone de los siguientes componentes:

    • Aplicación del remitente: Esta es cualquier aplicación que puede iniciar una sesión de lanzamiento y comunicarse con un receptor. La aplicación remitente puede considerarse la parte del «lado del cliente» del ecosistema Chromecast.

      El complemento Chromecast para Brightcove Player habilita la aplicación remitente en Brightcove Player. Aprenderás al respecto en este tema.

    • Aplicación del receptor: Se trata de una aplicación web personalizada que se aloja en Internet pública y se ejecuta en un dispositivo Chromecast. Maneja la comunicación entre la aplicación remitente y el dispositivo receptor. Se puede pensar como una aplicación HTML de una sola página con activos CSS y JavaScript.

      Para que Chromecast funcione con reproductores restringidos de dominio, debes incluir en la lista blanca el dominio del receptor de Chromecast, que es players.brightcove.net.

    Requisitos

    Los siguientes requisitos son necesarios para el complemento Chromecast:

    • Brightcove Player v6.45.0 o posterior
    • Complemento Brightcove Chromecast v2.0.0 o posterior

    Dispositivos

    Se admiten los siguientes dispositivos:

    • Chromecast (3ª generación)
    • Chromecast Ultra
    • Chromecast con Google TV (soporte preliminar)
    Es posible que las versiones o combinaciones anteriores sigan funcionando, pero no se admiten.

    Cómo funciona

    La implementación actual (plugin/receptor versión 2.x) utiliza la API CAF (Cast Application Framework).

    Cuando se agrega el complemento del receptor de Chromecast, el reproductor mostrará un botón de lanzamiento en la interfaz de usuario del reproductor cuando haya un Chromecast disponible en la red local.

    Cuando se hace clic o toca este botón, se inicia una sesión de lanzamiento con el vídeo actual que se carga en Brightcove Player. Así es como funciona:

    1. El plugin envía un conjunto de parámetros al receptor.
    2. El receptor refleja el reproductor emisor cargándolo desde el CDN de Brightcove.
    3. Una vez cargado, el reproductor del lado del receptor carga el ID de vídeo desde la Playback API que se envió cuando comenzó la sesión de lanzamiento.
    4. A continuación, el reproductor del lado receptor comienza la reproducción en la posición del cabezal de reproducción del reproductor que envía.

    Este proceso es idéntico para fuentes que no son de Video Cloud, excepto que no hay ninguna solicitud de Playback API.

    Analytics

    Actualmente, desde una perspectiva analítica, la sesión de lanzamiento se trata como una sesión de reproducción completamente nueva.

    Desde una perspectiva de UX, el espectador verá su flujo en pausa en el dispositivo remitente y reanudará en el receptor.

    Desde el punto de vista de los datos, debido a que nuestras métricas están anonimizadas por defecto, se registra como un nuevo visor que inicia una nueva transmisión en un dispositivo separado. Esto es lo que realmente sucede.

    Usar el plugin

    El complemento Chromecast para Brightcove Player se utiliza como todos los demás plugins de Brightcove Player, es decir, debe cargar el plugin en el reproductor.

    Uso de Studio

    Los siguientes pasos detallan el uso de Studio para cargar el plugin.

    Cuando se configura en Studio, la Player Management API seleccionará automáticamente la mejor combinación de versiones para su reproductor. Si obtienes v1.x del plugin de Chromecast, ¡necesitas actualizar tu reproductor!
    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. En el menú de navegación de la izquierda, seleccione Plugins.

    4. A continuación, seleccione el botón Agregar un complemento y, a continuación, seleccione Brightcove Plugin.

    5. Amplíe el menú desplegable de Brightcove Plugin y seleccione Chromecast Receiver.

    6. Opcional: Introduzca las opciones de configuración en el cuadro de texto Opciones (JSON) . Se muestra un ejemplo para el reproductor predeterminado en la cuenta 1752604059001 . Usted, por supuesto, necesita ingresar información para su jugador. 

      {
"playerUrl" : "https://players.brightcove.net/1507807800001/default_default/index.min.js"
}

      Consulte a continuación para obtener más detalles sobre las opciones.

    7. Seleccione el botón Guardar. Ahora verá el plugin de Chromecast Receiver añadido a la lista de complementos para su reproductor.

    8. Para publicar el reproductor, seleccione Publicar e incrustar > Publicar cambios.
    9. Para cerrar el cuadro de diálogo abierto, seleccione Cerrar.

    10. Vuelve a MEDIA y publica tu vídeo o lista de reproducción con el reproductor que acabas de actualizar para Chromecast.

      Una vez cargado el plugin, el vídeo ha comenzado a reproducirse y un dispositivo Chromecast disponible está dentro del alcance, el botón de lanzamiento aparecerá en el reproductor como se muestra aquí en esta captura de pantalla de un reproductor:

      El botón de lanzamiento del plugin de Chromecast aparecerá en la barra de control del reproductor cuando un dispositivo Chromecast disponible esté dentro del alcance.

    Configuración manual

    Siga estos pasos para agregar el complemento usando el editor JSON:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.

    3. En el menú de navegación izquierdo, seleccione Editor JSON.

    4. Agregar un objeto a la plugins matriz

      Su código debería verse similar a esto: 

      {
  "compatibility": true,
  "video_cloud": {
    "policy_key": "your policy key"
  },
  "player": {
    "template": {
      "name": "single-video-template",
      "version": "6.45.4"
    }
  },
  "studio_configuration": {
    "player": {
      "responsive": true,
      "height": 540,
      "width": 960,
      "units": "px"
    }
  },
  "plugins": [
    {
      "name": "chromecastReceiver",
      "options": {},
      "stylesheets": ["https://players.brightcove.net/videojs-chromecast-receiver/2/videojs-chromecast-receiver.css"],
      "scripts": ["https://players.brightcove.net/videojs-chromecast-receiver/2/videojs-chromecast-receiver.js"]
    }
  ]
}

      Consulte a continuación para obtener más detalles sobre las opciones.

    Opciones de configuración

    Independientemente del método que elija para configurar el complemento Chromecast para Brightcove Player, puede pasar opciones para modificar el comportamiento del complemento. Si bien no es necesario establecer estas opciones, es posible realizar una personalización adicional para los usuarios avanzados.

    Para pasar opciones en el código, usaría algo similar a lo siguiente: 

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  options = {};
  options.playerUrl = '//players.brightcove.net/1752604059001/default_default/index.min.js';
  myPlayer.chromecastReceiver(options);
});

    Consejos

    Si los reproductores web contienen un gran número de complementos o scripts personalizados, deberías considerar la posibilidad de crear un reproductor ligero específicamente para la reproducción de Chromecast.

    Dado que el reproductor web y su motor de reproducción requieren un poco de recursos, la ejecución en un entorno limitado de recursos (como un Chromecast) con scripts pesados adicionales puede afectar negativamente a la experiencia del usuario.

    Opciones

    El complemento Chromecast admite las siguientes opciones:

    • options.playerUrl
      Tipo: string
      Defecto : ''

      De forma predeterminada, el reproductor del lado del remitente se refleja en el receptor. Puede utilizar esta opción para proporcionar una URL personalizada de Brightcove Player del lado del receptor.

      Dado que se trata de un reproductor de Brightcove cargado en el receptor, puede personalizar, diseñar y depurar el reproductor fuera de Chromecast.

      Debe ser una dirección URL de un index.min.js archivo en la CDN de Brightcove
    • options.splashScreen
      Tipo: string
      Defecto : ''

      La pantalla de presentación que debe mostrarse antes de un vídeo y cuando los vídeos están cambiando.

    • options.authRequest
      Tipo : string o object o function
      Defecto : {}

      Esta es una opción poco común, ¡así que probablemente no la necesite!

      Puedes utilizar esta opción si necesitas Chromecast para realizar una solicitud de autorización para reproducir contenido antes de solicitar un vídeo.

      También puede usar esto para obtener cookies, pero su servidor tendrá que estar configurado para permitir players.brightcove.net que se establezcan cookies entre dominios si aloja su propio contenido.

      Uso de cadenas

      Cuando authRequest es una cadena, se realizará una solicitud GET vacía a esa URL.

      Uso de objetos

      Cuando authRequest es un objeto, se puede utilizar para especificar ciertas propiedades de carga útil de solicitud:

      Propiedad Tipo Obligatorio Predeterminado Descripción
      url string Sí '' La URL a solicitar
      method string No GET El método HTTP a utilizar
      body string No '' El órgano de solicitud
      headers object No {} Un objeto que describe encabezados HTTP donde las claves son el nombre del encabezado y las propiedades son los valores de encabezado
      withCredentials boolean No false Un objeto que describe encabezados HTTP donde las claves son el nombre del encabezado y las propiedades son los valores de encabezado
      timeout number No 15000 Cuántos milisegundos esperar antes de renunciar a la solicitud de autenticación

      Uso de funciones

      Cuando authRequest es una función, debe devolver una cadena o un objeto por esos usos.

    Limitaciones y problemas conocidos

    • El contenido HEVC/4K solo se admite en dispositivos Chromecast Ultra y Chromecast más reciente con dispositivos Google TV.
    • La publicidad del lado del cliente no es compatible actualmente, ¡pero SSAI es compatible!
    • Hemos visto problemas con la búsqueda en el remitente después de que se haya seleccionado una pista de subtítulos y se muestra en el receptor.
    • En Chromecast con dispositivos Google TV, hemos visto problemas con la interfaz de usuario, así como problemas de reproducción al usar subtítulos.
    • Google ha dejado de admitir orígenes no seguros (HTTP) con Chromecast, por lo tanto, el complemento no funcionará en contextos no seguros. En estos casos, el botón Chromecast del reproductor no aparecerá.
    • Dado que el complemento Chromecast se basa en el soporte del navegador para el casting, solo se admite en las siguientes combinaciones de OS/navegador:
      • Escritorio/Chrome
      • Android/Cromo
    • Google no admite varias instancias del botón remitente de Chromecast en una sola página. Las soluciones posibles serían incrustar el reproductor de Brightcove mediante la implementación del reproductor Estándar (iframe), o crear instancias y destruir dinámicamente a los jugadores según sea necesario. Consulte el documento Ejemplo del reproductor de Brightcove: Cargando el reproductor dinámicamente para obtener información sobre esta última posibilidad.
    • Este plugin no se emite a Google Nest Hub. El complemento solo admite dispositivos Chromecast reales (es decir, Chromecast y Chromecast Ultra).

    Cambios de v1.x a v2.x

    Además de los cambios internos, como usar la API CAF en lugar de la API de Chromecast v2, las siguientes interfaces han cambiado:

    • options.css y ya no options.js son compatibles.
    • customData.analyticsParams se añadió
    • customData.catalogData se renombró customData.catalogParams para mejorar la consistencia con otros usos
    • customData.policyKey fue reemplazado por customData.catalogParams.policyKey
    • customData.keySystems se añadió
