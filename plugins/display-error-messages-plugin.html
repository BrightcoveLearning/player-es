    Mostrar plugin de mensajes de error

    En este tema, aprenderá a modificar el comportamiento predeterminado de los mensajes de error, crear mensajes de error personalizados y errores de envío.

    Resumen

    El plugin de mensajes de error permite que el reproductor muestre mensajes fáciles de usar cuando encuentra un error. La hoja de estilos predeterminada muestra los mensajes como una superposición semitransparente encima del propio elemento de vídeo. Puede cambiar el texto del mensaje existente y agregar sus propios estilos. Incluso puede crear mensajes personalizados que se activan cuando lo desee.

    El mensaje de error que se muestra en la imagen anterior se creó actualizando el reproductor con un src valor no válido en la sources propiedad.

    El plugin de mensajes de error es un complemento predeterminado y se carga automáticamente con Brightcove Player. Sin embargo, puede elegir no cargarlo. Sin este plugin, verá un conjunto limitado de mensajes de error y algunos errores solo aparecerán en la consola del navegador. Para obtener más información sobre cómo no cargar un complemento predeterminado al crear un reproductor, consulte el documento Descripción general del complemento del reproductor .

    Para personalizar un mensaje de error puede utilizar uno de los dos métodos que se muestran en este documento:
    • Coloque el código en la página: Esto alterará sólo el jugador en la página.
    • Utilice un complemento personalizado: Esto alterará todas las instancias del jugador.
    NO se pueden personalizar los errores modificando la configuración del reproductor.

Si elige actualizar este plugin desde una página de código, sólo afectará a esa instancia de su reproductor.

    Errores estándar

    Este complemento tiene un conjunto de mensajes de error predeterminados para los errores de vídeo HTML5 estándar basados en el valor del código de error de tiempo de ejecución:

    • Código de error: 1
      • Tipo: MEDIA_ERR_ABORTED
      • Titulo: Se canceló la descarga de vídeo
      • Mensaje: Ha anulado la reproducción multimedia
    • Código de error: 2
      • Tipo: RED MEDIA_ERR_RED
      • Titulo: Se perdió la conexión de vídeo, confirme que está conectado a Internet
      • Mensaje: Un error de red causó un error parcial en la descarga de medios. Actualmente es más útil para MP4 y/o formatos de vídeo de descarga progresiva. Consulte la sección Problemas conocidos de este documento para obtener más información.
    • Código de error: 3
      • Tipo: MEDIA_ERR_DECODE
      • Titulo: El vídeo es malo o en un formato que no se puede reproducir en el navegador
      • Mensaje: La reproducción multimedia se anuló debido a un problema de corrupción o porque el medio utilizado características que su navegador no admitía.
    • Código de error: 4
      • Tipo: MEDIA_ERR_SRC_NOT_COMPATIBLE
      • Titulo: Este vídeo no está disponible o es incompatible con este navegador.
      • Mensaje: El contenido no se ha podido cargar debido a un fallo de red o del servidor o porque el formato es incompatible.
    • Código de error: 5
      • Tipo: MEDIA_ERR_CRYTED
      • Titulo: El vídeo que intentas ver está encriptado y no sabemos cómo descifrarlo
      • Mensaje: Los medios están cifrados y no tenemos las claves para descifrarlo.

    Si un error no tiene un código de error asociado, se muestra un mensaje genérico:

    • Código de error: desconocido
      • Message: MEDIA_ERR_UNKNOWN
      • Descripción: Se ha encontrado un problema no previsto, vuelva a comprobarlo pronto e inténtelo de nuevo

    Anula texto

    Hay tres partes del mensaje de error que puede cambiar:

    • headline : Este es el texto del mensaje en la parte superior.
    • type : Este es el código de error: texto.
    • message : Estos son los detalles técnicos: texto.

    El siguiente ejemplo muestra cómo anular el texto del mensaje para el error estándar con un valor de código de error de 4. Las propiedades se definen de la siguiente manera:

    • plugins: Esta propiedad contiene una matriz de propiedades y valores. Para este plugin, solo necesita proporcionar la name propiedad con el valor de errors.
    • options: Esta propiedad se utiliza para pasar datos al complemento.
    • errors: Esta propiedad define el código de error que desea actualizar. Aquí estamos anulando el texto del mensaje para el headline, type, y message.
      Nota: el código de error puede ser cualquier valor de cadena, como anular el texto del unknown mensaje.

    Usar en código de página

    Si incluye el script de errores en el código, puede anular el texto del mensaje de la siguiente manera:

    myPlayer.errors({
  "errors": {
    "4": {
      "headline": "This is a custom error message",
      "type": "custom type",
      "message": "these are details"
    }
  }
});

    Usar un complemento personalizado

    Si desea actualizar todas las instancias del reproductor, cree un complemento personalizado y agréguelo al reproductor en el módulo Players de Video Cloud Studio. Para obtener más información acerca de los complementos, consulte el documento Configuring Player Plugins .

    Para crear un complemento para anular el texto del mensaje estándar, siga estos pasos:

    1. Cree un archivo JavaScript almacenado en una ubicación accesible a Internet con el código del complemento de Brightcove Player. Debería verse similar a esto, pero con sus propios valores: 

      videojs.registerPlugin('errorText', function() {
  var myPlayer = this;

  myPlayer.errors({
    "errors": {
      "4": {
        "headline": "The Live Stream will begin soon",
        "type": "CUSTOM_TYPE",
        "message": "Please come back, once the live event has begun!"
      }
    }
  });
});

    2. En el módulo Reproductores , seleccione Plugins en la navegación izquierda.

    3. Expanda el botón Agregar un complemento y seleccione Complemento personalizado.

    4. En la página de detalles del complemento, agregue valores para:

      • Nombre del plugin - El nombre del plugin
      • URL de JavaScript - La ubicación de su plugin de reproductor desde el primer paso
    5. Seleccione el botón Guardar.

    6. Publica tu reproductor.

      Cuando realice cambios en el código del plugin, debe publicar el reproductor para que recoja esos cambios.

    Errores personalizados definidos por Brightcove

    También se pueden definir errores personalizados. Brightcove ha definido una serie de errores personalizados, explicados en esta sección, y también puede crear errores personalizados, detallados en la siguiente sección.

    • Brightcove recomienda que los valores de código de error personalizados sean una cadena. Verá que dos de los errores proporcionados usan números negativos por razones históricas, pero las cadenas alfanuméricas/descriptivas tienen menos probabilidades de causar problemas de colisión.
    • Los mensajes de error personalizados se pueden nombrar todo lo que desee. Brightcove recomienda que el tipo comience con PLAYER_ERR respecto al estandarizado MEDIA_ERR para evitar confusiones.
    • Como se detalla más adelante en esta sección, puede hacer que los errores personalizados sean desechables o no.

    Se han agregado cinco mensajes de error personalizados como referencia:

    • Código de error ፦1
      • Message: PLAYER_ERR_NO_SRC
      • Descripción: No hay ningún vídeo cargado.
    • Código de error ፦2
      • Message: PLAYER_ERR_TIMEOUT
      • Descripción: No se ha podido descargar el vídeo.
        Nota: En general, PLAYER_ERR_TIMEOUT ocurre cuando el reproductor espera reproducir vídeo pero no puede progresar durante 45 segundos. Este puede ser el resultado final de muchas cosas que suceden antes, incluida la suspensión del navegador cuando se coloca una pestaña en segundo plano, por ejemplo. Pero puede ser una especie de condición general para condiciones de red o plataforma que impiden que la reproducción continúe.
    • Código de error: No establecido
      • Message: PLAYER_ERR_DOMINIO_RESTRINGIDA
      • Descripción: Este vídeo está restringido para reproducirse en tu dominio actual
    • Código de error: No establecido
      • Message: PLAYER_ERR_IP_RESTRINGIDA
      • Descripción: Este vídeo está restringido en su dirección IP actual
    • Código de error: No establecido
      • Message: PLAYER_ERR_GEO_RESTRINGIDA
      • Descripción: Este vídeo está restringido para reproducirse en su región geográfica actual

    Errores personalizados definidos por el usuario

    Al definir sus propios errores personalizados, Brightcove recomienda no utilizar un código. (Puede ver en la sección anterior que esto es lo que Brightcove está haciendo ahora con los errores personalizados más recientes que están definiendo). También debe considerar usar el PLAYER_ERR_ prefijo para sus errores personalizados por razones de coherencia, pero por supuesto puede nombrarlos como desee.

    Si incluye el script de errores en el código, puede agregar mensajes personalizados de la siguiente manera:

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  //custom error
  myPlayer.errors({
    "errors": {
      "PLAYER_ERR_AGE_GATE": {
        "headline": "You must be at least 18 years of age.",
        "message": "Content may be considered inappropriate for some users."
    }
  }
});

    Visualización de errores personalizados

    Cuando defina errores personalizados, es necesario que Brightcove Player sepa cuándo desea mostrarlos. Para hacer esto, cree su propio código para determinar cuándo se debe mostrar el mensaje. A continuación, utilice la error() función para mostrar el mensaje de la siguiente manera:

    //display custom message
var age_appropriate = false;
myPlayer.on("loadstart", function () {
  if(!age_appropriate) {
    myPlayer.error({code:'PLAYER_ERR_AGE_GATE'});
  }
});

    Aquí la age_appropriate variable se establece en false , pero agregaría su propio código para determinar cuándo mostrar sus mensajes de error personalizados.

    Hacer que los errores personalizados no sean desechables

    De forma predeterminada, el visor de vídeo descarta un mensaje de error personalizado. Como muestra la siguiente captura de pantalla, hay un botón Aceptar para hacer clic para descartar la ventana, así como una X en la esquina superior derecha.

    error de catálogo de bc-

    Es posible que el manejo de errores en la ready() sección normal en el bloque de script pueda causar problemas. Por ejemplo, puede suceder que el evento bc-catalog-error se pueda enviar antes de que el reproductor esté listo, y si escucha el error en la ready() sección, no podrá manejar el error. Este problema puede producirse cuando se utiliza el filtrado geográfico, un vídeo está inactivo, un vídeo está fuera del rango de programación o en una cuenta diferente. Puede encontrar que no hay ningún problema en su código, pero el problema puede depender del navegador, así que tenga cuidado.

    Por ejemplo, aquí está el código del complemento que muestra un mensaje cuando un vídeo está inactivo:

    videojs.registerPlugin('inactiveErrorCheck', function() {
  var myPlayer = this;
  myPlayer.one('bc-catalog-error', function(){
    var specificError;
    myPlayer.errors({
      'errors': {
          'inactive-video-error': {
          'headline': 'The video you are trying to watch is inactive.',
          'dismiss': false
        }
      }
    });
    if (typeof(myPlayer.catalog.error) !== 'undefined') {
      specificError = myPlayer.catalog.error.data[0];
      if (specificError !== 'undefined' & specificError.error_code == "VIDEO_NOT_PLAYABLE") {
        myPlayer.error({code:'inactive-video-error'});
      };
    };
  });
});
    Para obtener más información sobre la gestión de errores de catalog, consulte la sección Manejar bc-catalog-error del documento catalog del Reproductor .

    Tasa de error exagerada

    Si está recibiendo lo que parece un número irrazonable de errores reportados, es importante saber que puede obtener eventos de error duplicados para la misma sesión, creando esta tasa de error exagerada. Brightcove Player envía errores a los análisis en el momento exacto y en la misma cantidad que se notifican al jugador. Por ejemplo, si no hay medios en un reproductor y el código llama de alguna manera play() mil veces seguidas, se notificarán a analítica mil errores PLAYER_ERR_NO_SRC .

    Si hay algunas sesiones con toneladas de errores sesgando el análisis, debe considerar usar la siguiente lógica para obtener una mejor idea de los problemas reales:

    number_of_sessions_with_errors / total_number_of_sessions
    en lugar de 
    count of errors/number of views

    Problemas conocidos

    • Al reproducir fuentes HLS, el comportamiento después de perder la conectividad de red probablemente no sea lo que esperaría. Los detalles son:
      • En la versión 6.x de Brightcove Player, se solicitan segmentos HLS sin fin y nunca aparece MEDIA_ERR_NETWORK .
      • En la versión 5.x de Brightcove Player después de un período de tiempo (más de 30 segundos) aparece un error PLAYER_ERR_TIMEOUT .
    • Al cargar vídeos en Edge en Windows 10 (tanto en Studio como en URL públicas), se muestra un error MEDIA_ERR_SRC_NOT_SUPPORTED y no se puede reproducir el vídeo.

    • En dispositivos Android y iPhones, los eventos de toque para los mensajes de error no se burbujean hasta el elemento de vídeo principal. Esto impide cerrar mensajes de error cuando aparecen. Este comportamiento puede ser un problema si el usuario está en modo de pantalla completa, ya que no tendrá forma de salir de este estado.

      Este problema se está trabajando actualmente y debería solucionarse en una versión futura del jugador. Por ahora, puede probar una solución alternativa como esta:

      player.on("touchstart", function(e) {
  if (player.error_) {
    player.error(null);
    e.preventDefault();
  }
})
