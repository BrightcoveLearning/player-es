    Configuración del Reproductor de Brightcove para DRM

    En este tema, aprenderá cómo Brightcove Player implementa la administración de derechos digitales (DRM). El documento primero muestra cómo implementar el plugin DRM, luego es seguido de detalles del complemento y cómo se implementa.

    Brightcove está adoptando las siguientes tecnologías para entregar contenido protegido DRM a la más amplia variedad posible de navegadores y dispositivos:

    • MPEG-DASH con DRM CENC compatible con Native/EME
    • HLS con FairPlay

    Para utilizar DRM con Brightcove Player, debe:

    • Producir contenido habilitado para DRM
    • Habilitar Brightcove Player para utilizar el complemento DRM
    • En algunos casos, configure el complemento DRM para usar su servidor de licencias
    Notas:
    1. DRM debe estar habilitado por su gestor de cuentas.
    2. Se recomienda utilizar la versión 6.18.1+de Brightcove Player.
    3. Para obtener más información sobre la compatibilidad con Brightcove con DRM, consulte Descripción general: Gestión de derechos digitales (DRM) en Video Cloud.
    Actualmente, Brightcove no admite la reproducción DRM de sólo vídeo o solo audio.

    Terminología

    Revise estos términos clave utilizados en este documento.

    Plazo Definición
    DRM Desde Wikipedia: Lagestión de derechos digitales (DRM) es una clase de tecnologías de protección de copia que son utilizadas por fabricantes de hardware y software, editores, titulares de derechos de autor e individuos con la intención de controlar el uso de contenido digital y dispositivos después de la venta.
    MPEG-DASH Desde Wikipedia: Latransmisión dinámica adaptable a través de HTTP (DASH), también conocida como MPEG-DASH, es una técnica de transmisión de velocidad de bits adaptable que permite la transmisión de contenido multimedia de alta calidad a través de Internet desde servidores web HTTP convencionales. Similar a la solución HTTP Live Streaming (HLS) de Apple, MPEG-DASH funciona dividiendo el contenido en una secuencia de pequeños segmentos de archivos basados en HTTP, cada segmento contiene un intervalo corto de tiempo de reproducción de un contenido que es potencialmente de muchas horas de duración, como una película o la transmisión en vivo de un evento.
    CENC A partir de la norma ISO: El esquema de cifrado común 'CENC' especifica métodos estándar de cifrado y asignación de claves que pueden ser utilizados por uno o más sistemas de gestión de claves y derechos digitales (sistemas DRM) para habilitar el descifrado del mismo archivo utilizando diferentes sistemas DRM.
    Extensiones de medios cifrados (EME) Desde Wikipedia: EME es una especificación del borrador del W3C para proporcionar un canal de comunicación entre navegadores web y software de agente de administración de derechos digitales (DRM). Esto permite el uso de vídeo HTML5 para reproducir contenido envuelto en DRM sin la necesidad de complementos multimedia de terceros, como Microsoft Silverlight.

    Implementar usando el módulo Players

    Para implementar el complemento DRM en Studio y habilitar DRM en función de la configuración de su cuenta, siga estos pasos:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir la funcionalidad DRM.
    2. Haga clic en el enlace del reproductor para abrir las propiedades del reproductor.
    3. Haga clic en Reproducir en el menú de navegación de la izquierda.
    4. A continuación, marque la casilla Habilitar DRM .
    5. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    6. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.

    Arquitectura del complemento DRM

    El plugin DRM (videojs-drm) es un contenedor alrededor de dos complementos:

    • videojs-plateado
    • videojs-contrib-eme

    El plugin videojs-drm versión 5 utiliza las capacidades de reproducción DASH integradas del reproductor. Esto utiliza VHS, que es la versión siguiente y renombrada del plugin incorporado videojs-contrib-hls .

    A partir del jugador 6.26.0 el jugador es compatible con DASH multi-período. Para versiones anteriores del jugador, el jugador Shaka es necesario para DASH multi-período.

    Si desea utilizar Shaka Player para la reproducción de DASH, como hizo en la versión 4, puede incluir el siguiente script junto con el script videojs-drm versión 5:

        https://players.brightcove.net/videojs-shaka/1/videojs-shaka.js
     

    El plugin videojs-silverlight permite la reproducción de contenido DASH en ciertos navegadores de Internet Explorer.

    El plugin videojs-contrib-eme permite la reproducción de contenido FairPlay HLS.

    Notas:

    • Al especificar las opciones de FairPlay, el nombre del plugin refleja el nombre del plugin de videojs-contrib-eme, eme.

    Tecnologías de reproducción utilizadas

    Brightcove Player utiliza diferentes tecnologías de reproducción DRM con diferentes navegadores. A continuación se detallan las tecnologías DRM utilizadas con Brightcove Player:

    • FairPlay: Sistema DRM de Apple
    • PlayReady: Sistema DRM de Microsoft
    • Widevine: Sistema DRM de Google

    En la tabla siguiente se detalla la relación entre el navegador (última versión), el formato y la tecnología de reproducción utilizada en Brightcove Player:

    Navegador Formato Tecnología de reproducción Tipo de copia utilizado
    para entregar contenido DRM
    Escritorio Chrome DASH con Widevine Nativo/EME MPEG-DASH
    Chrome Móvil1 DASH con Widevine Nativo/EME MPEG-DASH
    Internet Explorer2 DASH con PlayReady Silverlight MPEG-DASH
    Edge DASH con PlayReady Nativo/EME MPEG-DASH
    Safari HLS con FairPlay HLS nativo HLS
    Firefox DASH con Widevine Nativo/EME MPEG-DASH

    1No se admite la reproducción de DRM con Chrome Mobile en iOS.

    2IE11 en Windows 10 y 8.1, e IE11 en Win8 Metro, use Native/EME. Todas las demás versiones y configuraciones de IE utilizan el plugin Silverlight.

    Producir contenido DRM

    Hay dos pasos que debe realizar para crear contenido habilitado para DRM:

    1. Póngase en contacto con su administrador de cuenta para que su cuenta (s) DRM esté habilitada (s). A continuación, puede configurar su cuenta con las claves de licencia adecuadas y perfiles de ingesta para habilitar la creación de contenido protegido DRM.

    2. Produzca contenido protegido DRM. Puede optar por cargar contenido nuevo o volver a codificar contenido existente como DRM. Esto se hace seleccionando el perfil de ingesta que produce la tecnología de encriptación deseada.

      Para obtener más información, consulte el documento Ingerir contenido con DRM .

    Tendrá que producir contenido protegido DRM que utilice ya sea manifiestos MPEG-DASH con vídeos segmentados y cifrados, o contenido HLS FairPlay.

    Implementar la reproducción de FairPlay en código

    Si desea implementar la reproducción de FairPlay usando el código incrustado en la página, debe seguir estos pasos para usar el plugin DRM:

    1. En el encabezado de su página HTML, incluya la hoja de estilo del complemento: 
          <link href="https://players.brightcove.net/videojs-drm/5/videojs-drm.css" rel="stylesheet">
    2. Incluya el JavaScript del plugin: 
          <script src="https://players.brightcove.net/videojs-drm/5/videojs-drm.min.js"></script>

    3. Si su cuenta está configurada para Dynamic Delivery, simplemente llame al plugin EME para inicializarlo.

          <script type="text/javascript">
      videojs.getPlayer('myPlayerID').ready(function() {
        var myPlayer = this;
        myPlayer.eme();
    });
    </script>

       

      Si su cuenta NO está configurada para Dynamic Delivery, entonces debe llamar y configurar el plugin EME con información de credenciales de FairPlay.

          videojs.getPlayer('myPlayerID').ready(function() {
      var myPlayer = this;
      myPlayer.eme({
        "applicationId": "YOUR_APPLICATION_ID",
        "publisherId": "YOUR_PUBISHER_ID"
      });
    });
    </script>

      Brightcove Player envía la información de credenciales de FairPlay a una ruta de acceso de certificado FairPlay de Video Cloud.

    Siga estos pasos para usar el complemento DRM con contenido protegido de FairPlay:

    1. En el encabezado de su página HTML, incluya la hoja de estilo del complemento: 
          <link href="https://players.brightcove.net/videojs-drm/5/videojs-drm.css" rel="stylesheet">
    2. Incluya el JavaScript del plugin: 
          <script src="https://players.brightcove.net/videojs-drm/5/videojs-drm.min.js"></script>
    3. En un bloque JavaScript de la página, llame y configure el complemento EME con información de credenciales de FairPlay proporcionando las funciones GetCertificate, GetContentID y GetLicense al complemento EME. Estas funciones son específicas para su propia implementación del servidor de licencias de FairPlay, y se pueden pasar directamente al complemento EME antes de que se establezca un src : 
          <script type="text/javascript">
      videojs.getPlayer('myPlayerID').ready(function() {
        var myPlayer = this;
        myPlayer.eme({
          keySystems: {
            'com.apple.fps.1_0': {
            getCertificate: function (emeOptions, callback) {
            // request certificate
            // if err, callback(err)
            // if success, callback(null, certificate) where certificate
            // is a Uint8Array
          },
          getContentId: function (emeOptions, initData) {
            // return content ID as a string
            },
            getLicense: function (emeOptions, contentId, keyMessage, callback) {
              // request key
              // if err, callback(err)
              // if success, callback(null, key) as an arraybuffer
            }
          }
        }
      });
    </script>

    Tenga en cuenta que la información de FairPlay también se puede pasar como parte de cada objeto src . Las EmeOptions se proporcionan como un parámetro para todas las funciones. Son una referencia a las opciones de plugin fusionadas con (sobrescrito por) las opciones de origen de la fuente actual. Está disponible para facilitar el acceso a las opciones para que no tenga que mantenerlas usted mismo.

        player.src({
      type: 'application/vnd.apple.mpegurl',
      src: 'http://www.example.com/path/to/master.m3u8',
      keySystems: {
        "com.apple.fps.1_0": {
          getCertificate: function(emeOptions, callback) { ... },
          getContentId: function(emeOptions, initData) { ... },
          getLicense: function(emeOptions, contentId, keyMessage, callback) { ... }
        }
      }
    });

    Por ejemplo, si necesita usar un ApplicationID y PublisherId para la solicitud GetCertificate , puede pasar las opciones del complemento de esta manera:

        {
      keySystems: {
        "com.apple.fps.1_0": {
          getCertificate: function(emeOptions, callback) {
          var applicationId = emeOptions.applicationId; // 'application-id'
          var publisherId = emeOptions.publisherId; // 'publisher-id'
          // ...
        }
          // ...
        }
      },
      applicationId: 'application-id'
      publisherId: 'publisher-id'
    }

    O bien, si necesita un PublisherID específico de origen, puede sobrescribirlo a través de las opciones de origen:

        // plugin options
    {
      keySystems: {
        "com.apple.fps.1_0": {
        getCertificate: function(emeOptions, callback) {
        var applicationId = emeOptions.applicationId; // 'application-id'
        var publisherId = emeOptions.publisherId; // 'source-specific-publisher-id'
        // ...
      },
        // ...
      }
    },
    applicationId: 'application-id'
    publisherId: 'publisher-id'
    }
    // source options
    player.src({
      src: '<URL>',
      type: 'application/vnd.apple.mpegurl',
      publisherId: 'source-specific-publisher-id'
    });

    A continuación se muestra un ejemplo de implementación, pasando las opciones en opciones como src , recuperando el ID de contenido del nombre de host, URI de licencia y certificado fijos, y un URI de licencia que requiere un POST con un cuerpo del mensaje clave:

        var uint8ArrayToString = function(array) {
    return String.fromCharCode.apply(null, new Uint16Array(array.buffer));
    };
    var getHostnameFromUri = function(uri) {
      var link = document.createElement('a');
      link.href = uri;
      return link.hostname;
    };
    var getCertificate = function(emeOptions, callback) {
      videojs.xhr({
        uri: emeOptions.certificateUri,
        responseType: 'arraybuffer'
        }, function(err, response, responseBody) {
        if (err) {
          callback(err);
          return;
        }
        callback(null, new Uint8Array(responseBody));
      });
    };
    var getContentId = function(emeOptions, initData) {
      return getHostnameFromUri(uint8ArrayToString(initData));
    };
    var getLicense = function(emeOptions, contentId, keyMessage, callback) {
      videojs.xhr({
        uri: emeOptions.licenseUri,
        method: 'POST',
        responseType: 'arraybuffer',
        body: keyMessage,
        headers: {
          'Content-type': 'application/octet-stream'
        }
        }, function(err, response, responseBody) {
        if (err) {
          callback(err);
          return;
        }
        callback(null, responseBody);
      });
    };
    player.src({
      type: 'application/vnd.apple.mpegurl',
      src: 'http://www.example.com/path/to/master.m3u8',
      keySystems: {
        "com.apple.fps.1_0": {
        getCertificate: getCertificate,
        getContentId: getContentId,
        getLicense: getLicense
      }
      },
      certificateUri: 'http://example.com/fairplay/certificate/endpoint',
      licenseUri: 'http://example.com/fairplay/license/endpoint'
    });

    Implementación de la reproducción Widevine en el código

    Si desea implementar la reproducción Widevine usando el código incrustado en la página, debe seguir estos pasos para usar el complemento DRM:

    1. En el encabezado de su página HTML, incluya la hoja de estilo del complemento: 
          <link href="https://players.brightcove.net/videojs-drm/5/videojs-drm.css" rel="stylesheet">
    2. Incluya el JavaScript del plugin: 
          <script src="https://players.brightcove.net/videojs-drm/5/videojs-drm.min.js"></script>
    3. A continuación, simplemente llame al plugin EME para inicializarlo. 
          <script type="text/javascript">
      videojs.getPlayer('myPlayerID').ready(function() {
        var myPlayer = this;
        myPlayer.eme();
      });
    </script>

    Para el contenido Widevine Modular, debe configurar su reproductor para que use su servidor de licencias Widevine utilizando el controlador de origen del reproductor.

    Para actualizar una instancia de un reproductor en su página web, puede utilizar el controlador de origen del reproductor. Aquí hay un ejemplo de uso de la keySystemsOptions matriz con la player.src() función:

        player.src({
      src: 'http://example.com/my/manifest.mpd',
      type: 'application/dash+xml',
      keySystemOptions: [
      {
        name: 'com.widevine.alpha',
        options: {
        licenseUrl: 'http://m.widevine.com/proxy'
      }
      }]
    });

    También puede usar la updateSourceData función de la siguiente manera:

        videojs.Html5DashJS.updateSourceData = function(source) {
    source.keySystemOptions = [{
    name: 'com.widevine.alpha',
    options: {
    serverURL:'https://example.com/anotherlicense'
    }
    }];
    return source;
    };

    Para obtener más detalles, consulte la información de videojs-contrib-dash en GitHub.

    Apoyo a otros proveedores de DRM

    El plugin ha implementado una ruta que permite a los clientes implementar soporte para otros proveedores de DRM. Esto es sobre todo útil para Fairplay, ya que Fairplay requiere una lógica personalizada necesaria para obtener información de licencia. Agregar vendor.name a keySystems en una fuente determinada intentará utilizar esta lógica e incluir la información necesaria sobre el certificado y la licencia. A continuación se muestra un ejemplo para Azure:

        player.ready(function(){

    Problemas conocidos

    • los Predeterminado (visualización automática) ajuste de subtítulos en el Medios de comunicación módulo PISTA DE TEXTO No se admite la sección cuando se usa junto con DRM y subtítulos en el manifiesto. Los subtítulos en manifiesto se utilizan con los productos Brightcove Dynamic Delivery y SSAI, por ejemplo. Una solución alternativa para este problema es utilizar la <track> etiqueta con la implementación Advanced Brightcove Player. Esto se detalla en el documento Agregar subtítulos a vídeos mediante programación . Tenga en cuenta que debe usar el atributo con la etiqueta.default<track>
    • Activos DRM y Chrome: Cuando se utiliza la implementación estándar (iframe) reproductor con el plugin DRM, allow="encrypted-media" se requiere para poder reproducir recursos DRM en Chrome. 
          <iframe src="https://players.brightcove.net/123456789/BydO6uuuu_default/index.html?videoId=5783262319001"
    allowfullscreen
    webkitallowfullscreen
    mozallowfullscreen
    width="640" height="360"
    allow="encrypted-media"></iframe>
    • Evento especial para Silverlight/IE11: En la mayoría de los casos, si desea interactuar mediante programación con el reproductor, debe esperar a que ready o loadedmetadata  evento que se enviará. Sin embargo, si desea interactuar programáticamente con el reproductor al usar la tecnología de Silverlight en IE11, Y reproducir contenido DRM, debe esperar el canplay evento.
    • En iOS, solo Safari proporciona las API de explorador DRM (EME) necesarias, WebViews no tiene soporte para EME actualmente. Por lo tanto, Fairplay DRM solo funcionará en Safari.
