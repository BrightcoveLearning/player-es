    Compartir en redes sociales

    En este tema, aprenderá cómo usar el complemento de redes sociales para permitir a los usuarios compartir un vídeo a través de Facebook, Google+, Twitter, Tumblr, Pinterest y LinkedIn.

    Ejemplo

    Supongamos que tiene una página web con un Brightcove Player. Ahora, desea permitir que los usuarios compartan el vídeo en las plataformas de redes sociales. Con el plugin de redes sociales, verá un icono para compartir redes sociales en el muelle de título y descripción.

    Nota: Si el reproductor está configurado para no mostrar el muelle de título y descripción, el icono de compartir aparecerá en el área de controles del reproductor.

    Al hacer clic en este icono se muestra la pantalla para compartir redes sociales, como se muestra aquí.

    Tendrás que establecer un ancho y una altura para el jugador por dos razones:

    1. Si no se establece ancho o alto, no habrá valores para pasar a Twitter, lo que hará que la tarjeta de Twitter no se valida.
    2. El ancho debe ser lo suficientemente ancho para acomodar la interfaz del plugin social como se muestra en la captura de pantalla anterior. También puede hacer que la interfaz aparezca automáticamente en el extremo de vídeo utilizando la displayAfterVideo propiedad, documentada a continuación.

    Nota: En Studio, hay una opción para agregar el uso compartido en redes sociales a su reproductor y en muchos casos será suficiente para sus necesidades. Este documento cubre todos los aspectos del uso del plugin de redes sociales. La siguiente captura de pantalla imágenes UI de Studio.

    Haga clic aquí para ver una muestra de trabajo de una página web que utiliza el plugin e incluye las metaetiquetas de redes sociales.

    Resumen

    Este documento le guía a través de los pasos para habilitar el uso compartido en redes sociales para una página HTML existente con el Brightcove Player en ella. Esto supone que ha creado un Brightcove Player con la información de configuración adecuada y que ha agregado el reproductor a una página HTML.

    Para agregar el complemento de redes sociales, siga estos pasos:

    1. Configure el reproductor. Esto actualizará su Brightcove Player para utilizar el plugin de redes sociales. Hay tres formas de incluir el uso compartido en redes sociales con tu jugador:

      • Configurar las propiedades del reproductor
        En el módulo Players de Studio, puede configurar las propiedades en la sección Social. Esto proporciona una interfaz para seleccionar diferentes plataformas de redes sociales, pero no ofrece otras opciones de configuración.

      • Configurar el reproductor con opciones
        En el módulo Players de Studio usando la sección Plugins, hay una interfaz para proporcionar opciones de configuración.

      • Personalización de direcciones URL compartidas
        Esta implementación agrega el complemento social utilizando código.

    2. Añade metaetiquetas sociales a la página HTML de tu reproductor.
    3. Valida las metaetiquetas en tu página HTML para Facebook y Twitter.

    En un navegador, abra la página HTML con su Brightcove Player y utilice el icono de uso compartido en redes sociales del reproductor para compartir el vídeo.

    También puede implementar el plugin de redes sociales dentro del código de su página, pero esta es una solución más avanzada. Es posible que desee utilizar este enfoque para incluir direcciones URL de uso compartido personalizadas. Para obtener más información, consulte la sección Personalizar direcciones URL compartidas .

    Otras opciones para publicar en plataformas sociales incluyen las siguientes:

    • Publicar el Brightcove Player en artículos instantáneos de Facebook
    • Publicación de vídeos en plataformas de redes sociales

    Configurar reproductor

    Para agregar el complemento de uso compartido en redes sociales, siga estos pasos:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. Seleccione Social en el menú de navegación de la izquierda.
    4. A continuación, marque la casilla de verificación Habilitar uso compartido.
    5. Seleccione las plataformas de redes sociales a las que desea compartir vídeos.
    6. Para publicar el reproductor, seleccione Publicar e incrustar > Publicar cambios.
    7. Para cerrar el cuadro de diálogo abierto, seleccione Cerrar.

    Configurar reproductor con opciones

    Para una configuración más avanzada, puede usar propiedades como opciones para configurar el plugin. Para ello, siga estos pasos:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Seleccione el enlace para que el reproductor abra las propiedades del reproductor.
    3. Seleccione Social en el menú de navegación de la izquierda y asegúrese de que la casilla de verificación NO está marcada.
    4. Seleccione Plugins en el menú de navegación de la izquierda.
    5. A continuación, seleccione Plugins de las tres opciones.
    6. En el menú desplegable Añadir un complemento , seleccione Complemento personalizado.
    7. Para el nombre del complemento , escriba social.
    8. Para la URL de JavaScript , escriba: 
      		https://players.brightcove.net/videojs-social/3/videojs-social.min.js
    9. Para la URL CSS , escriba: 
      		https://players.brightcove.net/videojs-social/3/videojs-social.css
    10. Introduzca las opciones de configuración en el cuadro de texto Opciones (JSON) . En este ejemplo, las opciones hacen lo siguiente:
      • Líneas 2-3: Establezca el nombre y la descripción del vídeo.
      • Línea 4: Define la URL de su página web personalizada con el reproductor de vídeo y las metaetiquetas de uso compartido en redes sociales.
      • Líneas 5-6: Para Twitter, esto permite que un usuario comience a ver el vídeo después de un desplazamiento de 5 segundos.

      • Líneas 7-13: Permite compartir redes sociales para Facebook, Twitter, Tumblr, Pinterest y LinkedIn.

      	{
		"title": "Tiger",
		"description": "Video of two tigers",
		"url": "https://solutions.brightcove.com/bcls/brightcove-player/social-plugin/social-media.html",
		"deeplinking": true,
		"offset": "00:00:05",
		"services": {
			"facebook": true,
			"twitter": true,
			"tumblr": true,
			"pinterest": true,
			"linkedin": true
		}
	}
    11. Haga clic en Guardar.
    12. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    13. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.

    Propiedades

    Para los complementos en general, el options objeto se utiliza para pasar datos al plugin para personalizar la inicialización. En este caso, puede utilizar las siguientes propiedades en este objeto:

    título

    • Tipo: cadena.
    • Este es un título personalizado que aparecerá cuando se comparta el vídeo.

    descripción

    • Tipo: cadena.
    • Esta es una descripción personalizada que será utilizada por los servicios sociales que la apoyan.

    url

    • Tipo: cadena.
    • Esta es la URL que apunta a su página web personalizada que tiene su vídeo y las metaetiquetas para compartir. Consulte la sección de metaetiquetas para obtener detalles sobre cómo agregar los metadatos de redes sociales a la página del reproductor.

    Código EmbedCode

    • Tipo: cadena.
    • Este es el código de inserción iframe de Brightcove Player para compartir el vídeo. Esto le permite anular completamente el contenido del campo Código incrustado ubicado en el cuadro de diálogo para compartir.
    • Puede obtener el valor de esta propiedad mediante el método getEmbedCode () .

    QuitarEmbeed

    • Tipo: booleano
    • Predeterminado: false
    • Si se establece en true, desactiva (oculta) el código de inserción iframe

    DisplayAfterVideo

    • Tipo: booleano
    • Predeterminado:false
    • Hace que la pantalla social se muestre automáticamente en un ended evento

    profundización

    • Tipo: booleano
    • Predeterminado:false

    • Esto habilita o deshabilita la vinculación profunda para el vídeo compartido. Esta función permite al usuario comenzar a ver un vídeo desde un desplazamiento específico. Para obtener más detalles, consulte el documento Vinculación profunda .

      Nota: La función de enlace profundo actualmente solo funciona con Twitter.

    compensar

    • Tipo: cadena.
    • Formato: 00h00m00s
    • Esto se utiliza con la deeplinking propiedad y define cuándo empezar a reproducir el vídeo. Para compensar 1 minuto y 5 segundos, escribiría: 00h01m05s

    servicios

    Incluya todas las propiedades de servicio en este objeto. Para habilitar o deshabilitar la compatibilidad con un servicio, establezca el valor de la propiedad en true o false en lugar de eliminarlos.

    • facebook
      • Tipo: booleano
      • Predeterminado:true
      • Esto habilita el icono de compartir Facebook.
    • twitter
      • Tipo: booleano
      • Predeterminado:true
      • Esto habilita el icono de compartir Twitter.
    • tumblr
      • Tipo: booleano
      • Predeterminado:true
      • Esto habilita el icono de uso compartido de Tumblr.
    • pinterest
      • Tipo: booleano
      • Predeterminado:true
      • Esto habilita el icono de uso compartido de Pinterest.
    • linkedin
      • Tipo: booleano
      • Predeterminado:true
      • Esto habilita el icono de uso compartido de LinkedIn.

    ButtonParent

    • Tipo: texto
    • Predeterminado:shelf
    • Determina dónde debe ubicarse el icono de uso compartido. Si se establece en estante, el valor predeterminado, aparecerá el icono para compartir en el muelle. Si el valor se establece en ControlBar, el icono de compartir aparecerá en la barra de control.

    Ejemplo:

    	{
		"services": {
			"linkedin": true,
			"pinterest": true,
			"tumblr": true,
			"twitter": true,
			"facebook": true
		},
		"offset": "00:00:05",
		"deeplinking": true,
		"url": "https://solutions.brightcove.com/bcls/brightcove-player/social-plugin/social-media.html",
		"description": "Video of two tigers",
		"buttonParent": "controlBar",
		"title": "Tiger"
	}

    Métodos de configuración de Getter y setter

    Hay una serie de métodos disponibles para establecer y recuperar las opciones de configuración. Todos los métodos están asociados con el componente SociaLoverlay , que extiende la clase VideoJS.ModalDialog . (La información sobre esta clase se puede encontrar en el documento Implementación del ModalDialog .)

    Cuando se inicializa el complemento social, una instancia del componente SociaLoverlay está disponible en player.socialOverlay . El formato general para usar los métodos getter y setter es el siguiente:

    	player.socialOverlay.methodName()

    SetDirectLink (cadena)

    • Uso: player.socialOverlay.setDirectLink(string)
    • Propósito: Actualiza la URL utilizada por la superposición social. Establecer el valor en la cadena vacía («) hace que la URL se determine a petición.

    getDirectLink ()

    • Uso: player.socialOverlay.getDirectLink()
    • Propósito: Devuelve la URL utilizada por la superposición social.

    SetEbedCode (cadena)

    • Uso: player.socialOverlay.setEmbedCode(string)
    • Propósito: Actualiza el código de inserción utilizado por la superposición social.

    getEmbedCode ()

    • Uso: player.socialOverlay.getEmbedCode()
    • Propósito: Obtiene el código de inserción utilizado por la superposición social.

    SocialButton

    Cuando se inicializa el complemento social, una instancia del componente SocialButton está disponible en player.socialButton . Este componente amplía la clase Videojs.Button .

    Compartir en Facebook

    En este momento, solo el reproductor nativo de Facebook reproducirá vídeos en un muro de Facebook. Si quieres compartir tus vídeos y hacerlos reproducirlos en el muro de Facebook, puedes usar Brightcove Social. Para obtener más información, consulte el documento Introducción a Brightcove Social .

    Brightcove Player creará un enlace en la pared que abrirá el vídeo y el reproductor en una nueva pestaña o ventana del navegador. Para la mayoría de las personas, este comportamiento puede estar bien. Para ello, Facebook te permite definir el vídeo usando las metaetiquetas Open Graph (og) en la página HTML de tu página. Para obtener más información sobre cómo usar etiquetas og para Facebook, consulte el documento Compartir prácticas recomendadas .

    Para compartir un enlace al reproductor y al vídeo, siga estos pasos:

    1. En el módulo Jugadores, desplácese hacia abajo hasta la sección Compartir en redes sociales y seleccione Sí para permitir compartir.
    2. Guarde los cambios y publique su reproductor.

    Validar

    Una vez que haya seguido los pasos para incluir las metaetiquetas en su página, puede validar la página tanto para Facebook como para Twitter. Si se producen errores en cualquiera de los procesos de validación, actualice la configuración del reproductor según sea necesario.

    Facebook

    Para Facebook, vaya al Depurador de Facebook y pegue la URL en la página que contiene las metaetiquetas y haga clic en Depurar para validar.

    Twitter

    En Twitter, vaya al Validador de tarjetas, pegue la URL en la página que contiene las metaetiquetas y haga clic en Vista previa de la tarjeta para validar. Una ventaja añadida a la validación es que Twitter le pedirá que verifique su sitio web y usted entrará en la lista de espera para ser aprobado para las tarjetas de Twitter.

    Servicios personalizados

    El plugin ha incorporado en soporte para seis plataformas populares de medios sociales, pero también puede agregar otros usando servicios personalizados. Para ello se configura un options.customServices objeto. Este objeto es una matriz de objetos de servicio personalizados. Se muestra una configuración de ejemplo breve, luego los detalles del ejemplo proporcionado.

    El siguiente ejemplo agregaría tres botones. El primer y segundo botones para Xing y WhatsApp. El botón de WhatsApp solo aparecerá en el móvil. Estos dos ejemplos están usando Font Awesome para el icono, lo cual es conveniente si el reproductor es un incrustado en página en una página que ya utiliza la fuente, pero se podría usar cualquier otro estilo. El tercero es un botón de compartir Facebook usando la API de feed de Facebook que permite incluir un ID de aplicación de Facebook para la atribución. Desea deshabilitar el botón normal de Facebook cuando use esto.

    	[{
		className: 'fa fa-xing',
		title: 'Xing',
		hrefTemplate: 'https://www.xing.com/spi/shares/new?url=',
		bgColor: '#026466'
	},
	{
		className: 'fa fa-whatsapp',
		title: 'WhatsApp',
		hrefTemplate: 'https://api.whatsapp.com/send?phone=&text=title%20',
		bgColor: '#25d366',
		mobileOnly: true
	},
	{
		classSuffix: 'facebook',
		title: 'Facebook',
		hrefTemplate: 'https://www.facebook.com/dialog/feed?app_id=123456789&caption=&display=page&link='
	}]

    A continuación se definen las propiedades de un objeto de servicio personalizado:

    CustomService.Title

    • Tipo: cadena.
    • El título del destino compartido.

    CustomService.ClassName

    • Tipo: cadena.
    • Una clase o lista de clases separadas por espacios para agregar al botón.

    CustomService.ClassSuffix

    • Tipo: cadena.
    • El sufijo se agrega a vjs-icon- para generar la clase del botón. Sólo se utiliza si no className se especifica. Únicamente útil si se reemplaza uno de los destinos estándar, ya que la fuente Video.js no contiene iconos de destino social adicionales.

    CustomService.HREFTemplate

    • Tipo: cadena.
    • Una plantilla de picado para generar la URL de uso compartido, que puede contener las macros , , y . Se puede utilizar en lugar de customService.href donde no es posible especificar una función, por ejemplo, proporcionando las opciones en una cadena JSON a la Player Management API.

    CustomService.HREF

    • Tipo: cadena.
    • Una función para generar la url de uso compartido. Debe tomar un objeto y devolver una cadena. A menos que se especifique esto se genera desde hrefTemplate .

    CustomService.bgColor

    • Tipo: cadena.
    • Un valor de color CSS opcional que se aplicará como estilo de elemento de color de fondo. No es necesario si el botón está diseñado por CSS, pero se proporciona para mayor comodidad, especialmente cuando el icono será diseñado por una hoja de estilo existente.

    CustomService.MobileOnly

    • Tipo: booleano
    • Predeterminado: false
    • Si true , el botón solo se mostrará en Android e iOS.

    Problemas conocidos

    No hay descripción corta en la pantalla final

    La descripción corta del vídeo se muestra si hace clic en el botón para compartir mientras se reproduce el vídeo. En la pantalla final que contiene la información de uso compartido, no se muestra la descripción corta . Esto es lo que ha sido diseñado.

    Navegación de botones con teclado - Firefox para OS X

    Firefox para OSX tiene un comportamiento de tabulación predeterminado para tabular a través de cuadros de texto y enumera sólo elementos. Dado que los botones del Plugin Social son elementos de enlace, no será posible navegar a través de ellos con el teclado a menos que marque la casilla Todos los controles en Accesos directos de > > teclado de Preferencias del Sistema.
