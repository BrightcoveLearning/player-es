    Publicidad con el complemento IMA3

    Este tema cubre el uso del complemento IMA y cómo se puede implementar usando Studio y mediante codificación personalizada.

    El complemento IMA integra el reproductor de Brightcove con los anuncios interactivos de medios (IMA) de Google para HTML5 versión 3. Esto le permite solicitar y realizar un seguimiento de los anuncios VAST, VPAID y VMAP para su reproductor. Para obtener más información sobre Google IMA, consulte el documento Uso del SDK para IMA y HTML5, versión 3.

    Ejemplo del reproductor

    El siguiente vídeo muestra el uso del complemento IMA. Reproduzca el vídeo para ver un pre-roll, un midroll saltable a 5 segundos y, finalmente, un post-roll.

    Comprobación del servidor de anuncios

    Lo primero que debe hacer es verificar la validez de la etiqueta de anuncio que está planeando utilizar. Asegúrese de haber copiado su URL y vaya a la siguiente página: Video Suite Inspector (al hacer clic en este enlace, la página se abrirá en una nueva ventana o pestaña).

    Pegue la URL de la etiqueta de anuncio en el campo de entrada del Input type formulario. Haga clic Test Ad y verá su anuncio (s) reproducirse, con el vídeo proporcionado por Google intercalado. Si la etiqueta de anuncio no funciona en este entorno de prueba, no funcionará con Brightcove Player.

    Nota: El proveedor de anuncios puede alojar la etiqueta de anuncio VAST o puede copiar el XML y alojarlo en su propia URL.

    Implementar usando el módulo Players - Sección Publicidad

    En esta sección del documento utilizará Studio para implementar publicidad utilizando la sección Publicidad . En este caso, está limitado a las opciones que proporciona el formulario. Si desea personalizar la implementación usando algunas de las muchas opciones disponibles que no se proporcionan en esta sección, utilice la sección Implementar usando el módulo Players - Plugins , que ofrece la oportunidad de suministrar opciones a través de JSON.

     

    Para implementar el complemento IMA utilizando el módulo Players, siga estos pasos:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir funcionalidad publicitaria.
    2. Haga clic en el enlace del reproductor para abrir las propiedades del reproductor.
    3. Haga clic en Publicidad en el menú de navegación izquierdo.
    4. Marque la casilla de verificación Habilitar cliente (IMA) .
    5. Introduzca el Server URL para el servidor de anuncios.
    6. Seleccione la Request Ads configuración.
      • On load - Los anuncios se solicitan inmediatamente cuando se carga el reproductor (normalmente es la mejor experiencia para DFP/VPAID).
      • On play : la primera solicitud de anuncio se retrasa hasta que se inicie la reproducción.
      • On demand - Todas las solicitudes de anuncios se iniciarán mediante programación utilizando el player.ima3.adrequest() método. Este modo no admite anuncios de pre-oll o postroll.
      • On cue point : las solicitudes de anuncios se iniciarán en un punto de referencia de anuncio distribuido. Consulte el documento Visualización de anuncios mediante puntos de anuncio para obtener más detalles.
    7. Seleccione el anuncio VPAID Mode. El modo VPAID se utiliza para habilitar la compatibilidad con VPAID 2 en los anuncios de IMA.
      • Enabled - Reproducir anuncios VPAID en un iframe con un dominio diferente
      • Insecure - Reproducir anuncios VPAID en un iframe con el mismo dominio
      • Disabled - Los anuncios de VPAID generan un error
    8. Establezca el Timeout valor. Es la cantidad máxima de tiempo que debe esperar, en milisegundos, para que un anuncio se inicialice antes de la reproducción.
    9. Haga una elección para Tiempos de espera difíciles. Si se desmarca esta opción, el anuncio de carga lenta podría interrumpir la reproducción de vídeo.
    10. Establezca el número máximo de redirecciones. Esto especifica el número máximo de redirecciones antes de que se denieguen las redirecciones posteriores y se cancelará la carga del anuncio. El número de redirecciones afecta directamente a la latencia y, por lo tanto, a la experiencia del usuario.
    11. Para la versión del complemento, se recomienda encarecidamente que utilice la última versión.
    12. Ver un ejemplo, formulario completado:
    13. Haga clic en Guardar.
    14. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    15. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.

    Cuando se guarden los cambios en las propiedades publicitarias, el complemento IMA se configurará como parte de la configuración del complemento. El JavaScript y CSS estarán ocultos ya que los agregó a través de la Advertising sección.

    El complemento IMA admite propiedades adicionales que no están disponibles en esta sección de la interfaz de usuario. Consulte la siguiente sección de este documento para ver una forma de utilizar más opciones de configuración.

    Nota: Es posible que desee construir dinámicamente el serverUrl en el cliente (lo que significa que el complemento IMA4 ya se inicializaría), por ejemplo después de un inicio de sesión de usuario. Puede hacer esto estableciendo ese valor en una cadena vacía en el options { "serverUrl": "" } y luego asigne el valor mediante programación.

    Implementar usando el módulo Players - Sección Plugins

    Si desea configurar el plugin IMA4 más allá de las opciones proporcionadas en la sección Publicidad , puede utilizar la sección Plugins , que proporciona una forma de proporcionar opciones a través de JSON.

    Para implementar el complemento IMA4 agregará el nombre de la función del plugin y las URL a los archivos JavaScript y CSS del plugin:

    1. Abra el módulo PLAYERS y cree un nuevo reproductor o localice el reproductor al que desea añadir el plugin.
    2. Haga clic en el enlace del reproductor para abrir las propiedades del reproductor.
    3. Haga clic en Plugins en el menú de navegación izquierdo.
    4. A continuación, haga clic en Plugins de las tres opciones.
    5. En Agregar un complemento , seleccione Plugin personalizado.
    6. Para el nombre del complemento , escriba ima3.
    7. Para la URL de JavaScript , escriba: 
      https://players.brightcove.net/videojs-ima3/3/videojs.ima3.min.js
    8. Para la URL CSS , escriba: 
      https://players.brightcove.net/videojs-ima3/3/videojs.ima3.min.css
    9. Introduzca las opciones de configuración en el cuadro de texto Opciones (JSON) . 
      {
  "serverUrl": "//pubads.g.doubleclick.net/gampad/ads?sz=400x300&iu=%2F6062...",
  "timeout": 5000,
  "debug": true
}
    10. Haga clic en Guardar.
    11. Para publicar el reproductor, haga clic en Publicar e incrustar > Publicar cambios.
    12. Para cerrar el cuadro de diálogo abierto, haga clic en Cerrar.

    JSON vs notación JavaScript

    Si examina la Options sección anterior, verá que la información de configuración se suministra en formato JSON. Si está utilizando código para implementar su plugin IMA4, la notación para configurar opciones con JavaScript es ligeramente diferente.

    Aquí puede ver las opciones en formato JSON que podría usar en Studio:

    {
  "requestMode": "onload",
  "serverUrl": "//solutions.brightcove.com/bcls/brightcove-player/vmap/simple-vmap.xml?cust_params={mediainfo.ad_keys}",
  "timeout": 5000
}

    Si está utilizando JavaScript para configurar sus opciones, el formato JSON funcionaría, pero también podría usar la sintaxis de JavaScript como se muestra aquí:

    {
  requestMode: 'onload',
  serverUrl: '//solutions.brightcove.com/bcls/brightcove-player/vmap/simple-vmap',
  timeout: 5000
}

    Opciones

    El plugin video.js IMA4 se construye sobre el marco de anuncios video.js y acepta cualquier opción que proporcione el marco de anuncios. Eche un vistazo al archivo LÉAME del marco publicitario para obtener detalles sobre el conjunto actual de configuraciones anulables. Tanto las opciones específicas del complemento IMA como las opciones de marco publicitario se documentan aquí.

    Las opciones son:

    • ClickTrackingElement
    • depurar

    • DisableCustomPlaybackForios10Plus

    ClickTrackingElement

    Tipo: HTMLElement Valor predeterminado: no definido

    Si la tecnología de anuncios HTML se utiliza en el modo de reproducción de anuncios personalizado , se especifica un elemento HTML alternativo que se utilizará para realizar un seguimiento de los toques publicitarios en dispositivos que no admiten eventos de entrada sobre el elemento de vídeo. Hay más detalles disponibles en la documentación de parámetros para IMA ADDisplayContainer. Si proporciona un elemento de seguimiento de clics, es su responsabilidad mostrarlo y ocultarlo en los momentos apropiados en las plataformas apropiadas. En la mayoría de los casos, es mejor dejar esta configuración sin definir y permitir que el complemento y el IMA gestionen la interacción con los anuncios.

    depurar

    Tipo: boolean Valor predeterminado: false

    Si debug se establece en true, el marco de anuncios generará información adicional sobre su estado actual durante la reproducción. Esto puede ser útil para diagnosticar problemas o comportamientos inesperados en una integración de anuncios.

    Esta opción forma parte del marco publicitario y se configura de la siguiente manera:

    player.ima3({
  debug: true
});

    La siguiente captura de pantalla muestra la extensa información mostrada activando la depuración.

    DebugControbads

    Tipo: boolean Valor predeterminado: false

    Se utiliza para habilitar el registro de depuración para videojs-contrib-ads. Para que oncue las solicitudes ID3 funcionen, la trama ID3 TXXX recibida debe contener datos JSON con los siguientes campos:

    • Formato de etiqueta de aviso de anuncio:
      • name: Requerido - debe ser la cadena adCue>
      • id: Necesario: un valor único para identificar el anuncio dado que los tiempos son relativos en las transmisiones en directo
      • ServerURL: Opcional: invalidar el serverUrl en la configuración de IMA4 sólo para este anuncio
      • duración: Opcional: anexado como breaklength parámetro a la url del servidor
      Ejemplo:{"name": "adCue", "id": 123}
    • Formato de etiqueta de cancelación de anuncio:
      • name: Requerido - debe ser el valor de cadena adCancel
      • id: Necesario: un valor único para identificar la cancelación del anuncio dado que los tiempos son relativos en las transmisiones en directo.
      Ejemplo:{"name": "adCancel", "id": 234}

    DisableCustomPlaybackForios10Plus

    Tipo: boolean Valor predeterminado: false

    Esta propiedad forma parte del ima3SdkSettings objeto. Actúa como un getter y setter para controlar si deshabilitar la reproducción personalizada en los navegadores iOS 10+. El comportamiento es el siguiente:

    • Si es true, los anuncios se reproducirán en línea si el contenido de vídeo está en línea, lo que habilita los anuncios saltables. Sin embargo, el anuncio permanecerá en línea y no será compatible con la pantalla completa nativa de iOS.
    • Cuando es false, los anuncios se reproducirán en el mismo reproductor que el contenido.

    Consulte la sección Anuncios saltables de iOS de este documento para obtener un ejemplo del uso de esta configuración.

    HardTimeouts

    Tipo: boolean Valor predeterminado: true

    Abandona los anuncios que terminan de cargarse después de haber agotado el tiempo de espera. Esto evitará que los anuncios de pre-oll lentos se interrumpan después de que timeout haya transcurrido el tiempo y el vídeo de contenido haya comenzado a reproducirse.

    Esta opción forma parte del marco publicitario y se configura de la siguiente manera:

    player.ima3({
  hardTimeouts: true
});

    Si se establece esta opción, false se producirá un flash de contenido antes de los anuncios.

    iMA3SDKSettings

    Tipo: object Valor predeterminado: no definido

    Si se proporciona, las propiedades de este objeto se utilizan para establecer el IMA3SDKSettings de nivel de página cuando el SDK de IMA haya terminado de cargarse. Se espera que las propiedades de este objeto sean el equivalente en mayúsculas camel de los métodos setter en el objeto de configuración del SDK , menos el prefijo 'set'. Por ejemplo, si proporcionó este objeto al inicializar el complemento: 

    player.ima3({
  ima3SdkSettings: {
    'numRedirects': 3,
    'ppid': 'publisher-provided-id'
  }
}

    Luego, el complemento video.js IMA ejecutaría código que se ve algo como esto cuando IMA se había cargado:

    window.google.ima.ImaSdkSettings.setNumRedirects(3);
window.google.ima.ImaSdkSettings.setPpid('publisher-provided-id');

    RequestMode

    Tipo: string Valor predeterminado: onload

    Esta opción forma parte del marco de anuncios se configura de la siguiente manera:

    player.ima3({
  requestMode: 'onplay'
});

    Hay cuatro valores posibles para esta opción:

    • onload: Los anuncios se solicitan inmediatamente cuando se carga el reproductor. Normalmente, esta es la mejor experiencia para DFP/VPAID.
    • onplay: La primera solicitud de anuncio se retrasa hasta que se inicie la reproducción. Esto es importante para las redes publicitarias que consideran que las solicitudes de anuncios se reproducen, por lo que no querrá que una solicitud de anuncio antes que una solicitud de reproducción. Esto hará que los anuncios objeto de tráfico se caigan eventualmente, o el cliente recibirá menos por los anuncios mostrados.
    • ondemand: Los anuncios solo se reproducirán cuando se inicien utilizando el player.ima3.adrequest() método manualmente. Este modo no admite anuncios de pre-oll o postroll.
    • oncue:Esta opción se comporta de manera diferente dependiendo de si se utiliza con la useMediaCuePoints opción.

      Uso de puntos de referencia con Video Cloud

      Puede crear puntos de referencia de anuncio para un vídeo en Studio y, a continuación, hacer que se reproduzca un anuncio cuando se activa el punto de referencia. Para obtener más información, consulte el documento Display Ads Using Ad Cue Points .

      Usar puntos de referencia en una transmisión en directo

      Los anuncios se solicitan en función de los puntos de referencia ID3 en una transmisión en directo. Para que este tipo de solicitud funcione correctamente, la trama ID3 TXXX recibida debe contener datos JSON con los siguientes campos:

      • name: Requerido; debe ser el valor de cadena adCue
      • id: Obligatorio; valor único para identificar el anuncio dado que los tiempos son relativos en las transmisiones en directo
      • serverUrl: Opcional; Se anexa como parámetro de longitud de corte a la url del servidor
      • duration: Opcional; duración del anuncio

      Ejemplo:

      {"name": "adCue", "id": 123}

      También puede cancelar puntos de referencia ID3 en una transmisión en vivo creada adCue por un mediante adCancel. Se debe utilizar el siguiente formato para el objeto enviado: 

      {"name": "adCancel", "id": 123}

      Tanto el name y id son necesarios.

      • Las señales de anuncios ID3 en vivo no son compatibles con Android.
      • Las señales de cancelación de anuncios ID3 en vivo no son compatibles con iOS.

    ServerURL

    Tipo: string o function

    Valor predeterminado (un anuncio genérico de Google):

    //pubads.g.doubleclick.net/gampad/ads?sz=400x300&iu=%2F6062%2Fiab_vast_samples&ciu_szs=300x250%2C728x90&gdfp_req=1&env=vp&output=xml_vast2&unviewed_position_start=1&url=[referrer_url]&correlator=[timestamp]&cust_params=iab_vast_samples%3Dlinear

    A continuación, proporcionará la URL de su servidor de anuncios. Esta opción se puede configurar en Brightcove Studio como se muestra arriba. También puede establecer el valor en código. A continuación se muestran dos ejemplos. (Recuerde que debe probar la etiqueta de anuncio en su servidor antes de intentar usarla en el complemento).

    Si el valor es una cadena, representa la URL del servidor de anuncios desde donde se solicitan anuncios y se puede establecer en código de la siguiente manera:

    player.ima3({
  serverUrl: 'your ad server'
});

    Si el valor es una función, el parámetro de función es una callback función que se debe llamar con la URL del servidor como argumento. Esto proporciona soporte para procesos asincrónicos, como pujas por cabecera. En el siguiente ejemplo, verá información del mediainfo objeto utilizado para determinar qué URL de anuncios se debe utilizar en función de un ID de cuenta: 

    // Initialize IMA plugin
myPlayer.ima3({
  serverUrl: function(callback) {
    if (myPlayer.mediainfo.id === '4034552795001') {
      callback('https://pubads.g.doubleclick.net/.../url1');
    } else {
      callback('https://pubads.g.doubleclick.net/.../url2');
    }
  },
  requestMode: 'onload',
  debug: true,
  debugContribAds: true
});

    ShowVpaidControls

    Tipo: boolean Valor predeterminado: false

    Se true configura para mostrar controles personalizados de Brightcove sobre los anuncios VPAID. Es posible que funcionen o no dependiendo de la implementación de VPAID, por lo que Brightcove sugiere probar esta función con sus anuncios antes de activarla en producción.

    tiempo de espera

    Tipo: number Valor predeterminado: 4000

    La cantidad máxima de tiempo, en milisegundos, que se espera a que se reproduzca un anuncio antes de que se salte un salto de anuncio.

    Esta opción forma parte del marco de anuncios se configura de la siguiente manera:

    player.ima3({
  timeout: 5000
});

    En las pruebas internas de Brightcove se encontró que cuatro segundos parecían ser lo suficientemente largos como para acomodar la inicialización lenta en la mayoría de los casos, pero aún así lo suficientemente cortos como para que las fallas al inicializar no parecieran fallas del reproductor o del vídeo de contenido.

    Nota: Si establece el tiempo de espera tanto en Studio como en la configuración del plugin en la página HTML usando JavaScript, la configuración de JavaScript tendrá prioridad.

    Si utiliza Studio para configurar la publicidad, cuando seleccione En el punto de referencia la useMediaCuePoints y requestMode las opciones se establecerán correctamente para usted.

    VPAIDMode

    • DISABLED: Los anuncios de VPAID generan un error.

    player.ima3({

    Macros de anuncios y ServerURL

    Existen macros de anuncios disponibles para facilitar el trabajo al crear la URL del servidor de anuncios. Estas macros le permiten utilizar variables en la URL del servidor para las que el complemento IMA4 sustituirá los valores apropiados. Por ejemplo, la siguiente URL del servidor contiene algunas de las variables: 

    {"serverUrl": "//myadserver.com/ad?video={mediainfo.id}&duration={player.duration}"}

    Los complementos IMA4 sustituirían los valores apropiados, y la URL del servidor realmente utilizada aparecería de la siguiente manera:

    {"serverUrl": "//myadserver.com/ad?video=12340001&duration=60"}

    La siguiente es la lista completa de variables para las que se utilizarán valores sustituidos:

    Macro Descripción
    {document.referrer} URL de página de referencia
    {mediainfo.ad_keys} Cadena de texto de formulario libre que se puede agregar y editar en el módulo Media de Studio; debe usar el parámetro de consulta en el formulario 
    cust_params={mediainfo.ad_keys}
    {mediainfo.description} Descripción corta (250 chars max)
    {mediainfo.duration} Duración del vídeo según lo informado por Video Cloud
    {mediainfo.id} Id. del vídeo
    {mediainfo.name} Título del vídeo
    {mediainfo.reference_id} Identificador de referencia
    {mediainfo.tags} Etiquetas (metadatos) asociadas al vídeo
    {player.duration} Duración del vídeo medido por el reproductor (posiblemente ligeramente diferente de mediainfo.duration y probablemente más preciso). Tenga en cuenta que esto devuelve 0 si el vídeo no está cargado. Tenga cuidado de sincronización de su solicitud de anuncio con esta macro.
    {player.height} La altura actual del jugador
    {player.id} Identificador del reproductor
    {Player.PageURL} La URL de la página: Devuelve el referrer del documento si está en un iframe, de lo contrario, la ubicación de la ventana.
    {player.width} Ancho del reproductor actual
    {playlist.id} Extraído del objeto playlistinfo
    {playlist.name} Extraído del objeto playlistinfo
    {aleatorio} Un número aleatorio 0-1 billón (Se utiliza para crear una impresión única. Esto evita que el anuncio se almacene en caché en el navegador y evita discrepancias de impresión.)
    {marca de tiempo} Hora local actual en milisegundos desde 1/1/70
    {window.location.href} URL de página actual
    NOTA: Cuando recupere la URL principal de un reproductor iframe o una experiencia In-Page de la Gallery, window.location.href se recomienda utilizar. La razón de esto es que las experiencias de In-Page utilizando el código de inserción estándar construyen un iframe dinámicamente en tiempo de ejecución para evitar problemas de estilo y scripting. Dado que el iframe se genera a través de JavaScript, toma la misma window.location de la página principal y sigue la política del mismo origen. Desafortunadamente, los navegadores tratan document.referrer de manera diferente para estos iframes generados. Chrome, Safari y Opera establecen document.referrer en la window.location de la página principal, pero solo en http (no https). Firefox sigue las especificaciones correctamente y establece document.referrer al referrer de la página principal. Todos los navegadores configuran correctamente document.referrer al referrer de la página principal en https.

    Valores predeterminados y macros de anuncios

    Puede proporcionar valores predeterminados a las macros de anuncios. Se puede proporcionar un valor predeterminado dentro de una macro, en cuyo caso este valor se utilizará cuando una variable no esté definida. La sintaxis es: 

    {macro=default}

    Por ejemplo,

    http://example.com/ad/{pageVariable.adConf=1234}

    resolvería lo siguiente si no está definido:window.adConf

    http://example.com/ad/1234

    Macros dinámicos

    Las macros dinámicas proporcionan acceso a los valores siguientes:

    • El customFields objeto del vídeo a través de la mediainfo.customFields variable.
    • El window objeto del DOM a través de la pageVariable variable.

    Por ejemplo, puede utilizar lo siguiente en su solicitud de anuncio:

    //myadserver.com/ad?l={pageVariable.navigator.language}&category={mediainfo.customFields.type}

    Para los pageVariable valores, sólo se permiten ciertos tipos de valores, como se muestra en esta tabla:

    Tipo Qué Sucede

    Número Convertido en cadena automáticamente
    Booleano Convertido en cadena automáticamente
    Nulo Devuelve la cadena null
    Indefinido Registra la advertencia y devuelve una cadena vacía
    Otro Registra la advertencia y devuelve una cadena vacía

    Macros de anuncios personalizados

    Aunque la Dynamic macros técnica descrita anteriormente es el método preferido para acceder a información específica a través de macros, es posible que haya definido valores personalizados que utiliza al solicitar anuncios de su servidor de anuncios a los que no puede acceder mediante macros dinámicos. En este caso, puede utilizar sus valores anulando el adMacroReplacement() método. Al anular este método, le permite pasar sus valores especializados para la solicitud de anuncio.

    El siguiente es un ejemplo de anular el adMacroReplacement() método. En este ejemplo, los valores personalizados se definen como parte del DOM de la página, lo que permite personalizar las solicitudes de anuncios por página. En este ejemplo, mySite.category es la ubicación donde se almacena la información de solicitud de anuncio. 

    brightcovePlayer.ima3.adMacroReplacement = function (url) {
    var parameters = {
    '{category}': mySite.category
  };
  for (var i in parameters) {
    url = url.split(i).join(encodeURIComponent(parameters[i]));
  }
  return url;
}

    Usar valores específicos ayudará a aclarar exactamente lo que está sucediendo. Supongamos que la URL de solicitud al servidor de anuncios es la siguiente:

    //myadserver.com/myads?adWord={category}

    Y supongamos que el valor asignado dinámicamente mySite.category en la página es la cadena fishing-pole. Después de que se llame a la versión del adMacroReplacement() método, la URL de solicitud de anuncio aparecerá como: 

    //myadserver.com/myads?adWord=fishing-pole

    En resumen, al anular el adMacroReplacement() método, permite utilizar valores personalizados como macros de anuncios y asignar dinámicamente valores a la solicitud de anuncio URL.

    Métodos

    Cuando necesite interactuar con el SDK de IMA, debe esperar a que se distribuya el ima3-ready evento antes de que se pueda utilizar correctamente el SDK. Esto incluye los dos métodos siguientes.

    player.ima3.adrequest ()

    Al llamar a este método se creará una solicitud de anuncio bajo demanda inmediatamente después de recibir una respuesta al anuncio. Al invocar este método se crea un nuevo IMA ADManager, lo que significa que se perderá cualquier información de respuesta de anuncio anterior (por ejemplo, un anuncio de postroll devuelto en una respuesta VAST anterior). Brightcove recomienda que utilice este método únicamente en los casos en que no se conozca el conocimiento inicial de los tiempos de anuncio o se realicen todas las llamadas de anuncios bajo demanda. En todos los demás casos, tiene sentido poner todos los datos del anuncio en la llamada inicial de VAST en la inicialización del complemento.

    Los siguientes son dos puntos importantes a tener en cuenta al usar player.ima3.adrequest( ):

    • El método es para uso únicamente con el modo de ondemand solicitud.
    • El método no se recomienda para las versiones preliminares, ya que el contenido se reproducirá antes de que finalice la solicitud de anuncio, lo que dará como resultado un flash de contenido.
    Parámetros
    • adRequestUrl String Ruta a una etiqueta de anuncio VAST. Puede y debe pasar URL relativas. Este parámetro es opcional y el configurado serverUrl se utiliza si no se pasa ningún parámetro.
    Devoluciones:
    • Nada

    Ejemplo

    player.ima3.adrequest('//pubads.g.doubleclick.net/gampad/ads?sz=640x360&iu=/6062/iab_vast_samples/skippable&ciu_szs=300x250,728x90&impl=s&gdfp_req=1&env=vp&output=xml_vast2&unviewed_position_start=1&url=[referrer_url]&correlator=[timestamp]');

    Player.IMA3.setAdsRenderingSettings ()

    Este método le permite establecer la configuración de representación de anuncios para el SDK de IMA para HTML5.

    Si todavía no existe el Administrador de anuncios, este método guarda la configuración y, cuando se crea un Administrador de anuncios, utiliza la configuración que proporcionó. Si ya existe un Administrador de anuncios, se actualiza para usar la configuración. En cualquier caso, cualquier nuevo gestor de anuncios creado en el futuro también utilizará la configuración más reciente que haya proporcionado. Puede crear un AdsRenderingSettings objeto accediendo directamente al SDK de IMA. Hay una variedad de configuraciones disponibles.

    Parámetros
    • google.ima.AdsRenderingSettings objeto
    Devoluciones:
    • Nada

    Ejemplo:

    var adsRenderingSettings = new google.ima.AdsRenderingSettings();
adsRenderingSettings.bitrate = 2500;
adsRenderingSettings.enablePreloading = true;
player.ima3.setAdsRenderingSettings(adsRenderingSettings);

    Aplicación híbrida

    Hasta ahora en este documento ha visto el complemento IMA implementado en Studio, luego en código. A algunos clientes les gusta realizar una implementación híbrida donde el complemento básico se agrega en Studio, pero luego la configuración se realiza en JavaScript de la página. Esta implementación híbrida se discute en esta sección.

    Como acabas de ver en la sección anterior, al implementar el complemento IMA4 puramente en código manipula el plugin en formato de función. Cuando instala el complemento en Studio, luego configura en la página, debe tratar el complemento como un objeto. Por ejemplo, supongamos que ha instalado el complemento IMA4 usando Studio como se muestra aquí, teniendo en cuenta que no se proporciona ninguna etiqueta de anuncio :

    Ahora, en el JavaScript en la página puede asignar valores de propiedad (recuerde, trate el plugin como un objeto ahora) como se muestra aquí:

    videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  myPlayer.ima3.settings.serverUrl = 'http://solutions.brightcove.com/bcls/brightcove-player/vmap/simple-vmap.xml';
});

    Por supuesto, se pueden asignar valores a otras propiedades de esta manera.

    Asistencia de depuración

    Existen dos opciones de ayuda para depurar problemas de reproducción de anuncios. Si no hace nada, en la consola solo verá información sobre cuándo se iniciaron y terminaron los anuncios:

    La primera opción se menciona anteriormente en este documento en la sección Opciones . Esto activa la depuración en el nivel del complemento. Verá información adicional de depuración:

    La segunda opción utiliza una herramienta suministrada por Google. Especifique una sdkurl opción con un valor que apunte a un archivo JavaScript proporcionado por Google. Aquí se muestra un ejemplo de configuración: 

    var myPlayer = bc('myPlayerID');
myPlayer.ima3({
  "serverUrl": "//solutions.brightcove.com/bcls/brightcove-player/vmap/simple-vmap.xml",
  "debug": true,
  "sdkurl": "//imasdk.googleapis.com/js/sdkloader/ima3_debug.js"
});
myPlayer.ready(function() {
  myPlayer = this;
});

    La información de depuración ampliada parecería similar a la siguiente. (La información resaltada en amarillo proviene de la depuración del complemento y la información resaltada en azul es generada por la herramienta Google).

    Nota: No hay dependencias entre la información de depuración del complemento y la información de Google. Se pueden usar por separado o juntos.

    Para obtener más información sobre los errores de IMA de Google, consulte la sección Class Google.ima.adError del Google IMA HTML5 SDK APIs documento de Google.

    Problemas conocidos

    Imagen en imagen desactivada

    El control de imagen en imagen está deshabilitado cuando se implementa el complemento IMA4. Esta es una decisión de diseño intencional.

    Configuración de la reproducción automática en Studio

    NO configure la reproducción automática en Studio si está utilizando el complemento IMA4 para mostrar anuncios. Si se establece la reproducción automática en Studio, podría producirse un error en la reproducción de anuncios. Consulte el documento Consideraciones de reproducción automática para obtener más información.

    Más información y el temporizador de cuenta atrás de anuncios ya no se muestra de forma predeterminada

    Existe una solución alternativa utilizando la useStyledLinearAds propiedad del SDK de IMA de Google. Establezca este valor en true, como se muestra aquí:

    adsRenderingSettings.useStyledLinearAds=true;

    Para obtener más información, consulta la documentación de Google.ima.adsRenderingSettings de Google.

    Chrome en Android: Dessilenciar el pre-roll, el vídeo no se reproducirá automáticamente

    Al utilizar Chrome (versión más reciente) en Android y dessilenciar el reproductor mientras se está reproduciendo el pre-roll, el reproductor no comienza a reproducir un vídeo automáticamente después de que finalice el pre-roll. Si anula el silencio del anuncio, la función de persistencia del volumen de Brightcove Player también anulará el silencio del contenido, ya que la intención del espectador es escuchar el audio. Sin embargo, en las versiones más recientes de Chrome Android, el contenido sin silenciar no se puede reproducir automáticamente y se ha añadido el requisito de gesto del usuario para comenzar la reproducción. Esta es una configuración de nivel OS/dispositivo y no es algo que Brightcove pueda anular.

    Es posible que la visualización de la hora actual de la preroll no sea precisa

    Este problema tiene que ver con una limitación en la forma en que el SDK de IMA informa la hora actual. No hay ninguna solución documentada en este momento.

    Es posible que los midrolls no silenciados no se reproduzcan en Safari 11 Desktop

    Es posible que los midroll no silenciados no se reproduzcan en Safari 11 Desktop debido a un cambio en el SDK de IMA de Google. Se introdujo un nuevo comportamiento para desencadenar un error y cancelar un anuncio cuando Safari 11 rechaza la reproducción automática (Desktop y posiblemente iOS). Los Midrolls afectados de esta manera activarán un error de anuncio 400, con el código de error 1205 indicando que se ha evitado la reproducción automática.

    Entornos compatibles

    Para comprobar si hay combinaciones compatibles de plataformas, estándares publicitarios y medios de vídeo, consulte el documento de compatibilidad con la función de vídeo y SDK de Google. Ese documento y el Inspector de Video Suite le ayudarán a diagnosticar las configuraciones de anuncios IMA4 que simplemente no funcionarán.

    Nota: Para utilizar el Inspector de Video Suite, necesitará su etiqueta de anuncio VAST o su respuesta de anuncio VAST. A continuación, puede pegarlo en el campo de texto dado y luego hacer clic en el Test Ad botón.

    Superposiciones y transiciones de pantalla completa

    video.js utiliza la API de pantalla completa cuando esté disponible. Diferentes navegadores implementan la transición a pantalla completa de manera diferente y que pueden producir discrepancias en la apariencia al pasar dentro y fuera del modo de pantalla completa. En la mayoría de las implementaciones, el elemento que se está tomando en pantalla completa se escala geométricamente (es decir, zoom) desde su tamaño original hasta el objetivo. Sin embargo, la mayoría de los anuncios de superposición están diseñados para mostrarse a un tamaño fijo, por lo que pueden aparecer distorsionados hasta que finalice la animación.

    Anuncios saltables en dispositivos iOS

    Con el complemento IMA4, se pueden reproducir anuncios saltables en iPhones cuando se cumplan las siguientes condiciones:

    • El playsinline atributo está presente en el video elemento
    • La disableCustomPlaybackForIOS10Plus configuración se pasa al complemento y se establece en true

    Para el playsinline atributo, simplemente incluirlo como un atributo en la video etiqueta:

    <video-js id="player"
  width="640"
  height="360"
  data-video-id="4524585416001"
  data-account="4360108595001"
  data-player="r12ukws9l"
  data-embed="default"
  class="video-js"
  playsinline
  controls></video-js>
    Nota: El playsinline atributo solo funciona con iOS 10+.

    Para la disableCustomPlaybackForIOS10Plus configuración, asígnelo como una propiedad de ima3SdkSettings: 

    player.ima3({
  ima3SdkSettings: {
    "disableCustomPlaybackForIOS10Plus": true
  }
})

    Si está utilizando Studio y desea usar la configuración allí, agregue esto a la configuración del plugin IMA4:

    {
  "ima3SdkSettings": {
    "disableCustomPlaybackForIOS10Plus": true
  }
}

    Para obtener detalles completos de esta configuración, consulte la entrada DisableCustomPlaybackForios10plus en la ima3SdkSettings sección de este documento.

    Skippable ad limitations:

    • El Skip Ad botón puede estar parcialmente cubierto por la barra de control de anuncios en algunos dispositivos móviles. Esto dificulta que los usuarios finales salten el anuncio. El usuario puede pellizcar para hacer zoom para hacer el Skip Ad botón más grande en dispositivos móviles para poder hacer clic en él.
    • iPhone sin playsinline y disableCustomPlaybackForIOS10Plus - Los anuncios saltables no se reproducen.
    • iPhone con playsinline y disableCustomPlaybackForIOS10Plus - Los anuncios se reproducen en línea, pero si se utiliza a pantalla completa, los anuncios no se muestran, pero el audio del anuncio se reproducirá. En otras palabras, los anuncios saltables en pantalla completa no funcionan correctamente.
    • iPad - Los anuncios se reproducen en línea, pero saltar no está disponible en modo de pantalla completa

    iOS 10 iPad y iPhone: Los anuncios previos a la publicación no funcionan cuando se usan playsinline y se van a pantalla completa

    Si se utiliza playsinline en iOS 10 iPad y iPhone para permitir ver un vídeo que no sea a pantalla completa, se inicia un anuncio previo a la carga y se hace clic en el botón de pantalla completa, el anuncio NO continuará reproduciéndose. Esto es una limitación de la implementación de IMA de Google, y Google no tiene planes para corregir el problema.

    Conflicto con gpt_proxy.js

    Si utiliza el script proxy GPT con IMA4, y adTechOder es el HTML5 primero, pueden producirse problemas de reproducción. El plugin IMA4 se verá afectado al usar cualquier script que use window.google o window.google.ima. Se sugiere que compruebe si está utilizando Brightcove Player y, si es así, no cargue el proxy en esas ocasiones.

    Skip Ad Botón Cambiar tamaño

    No es posible cambiar el tamaño del Skip Ad botón. La API de Brightcove Player no tiene métodos ni propiedades para especificar el tamaño del Skip Ad botón. En el nivel de desarrollador, si un editor utiliza anuncios de VPAID, los anuncios pueden implementar su propio Skip Ad botón y lógica para encajar en la interfaz de usuario y la distribución de elementos de Brightcove Player.

    Los anuncios no se reproducirán automáticamente en iOS

    Aunque no es específico del complemento IMA4, sepa que autoplay está restringido iOS, por lo que los anuncios no se iniciarán hasta que se realice un gesto de usuario.
