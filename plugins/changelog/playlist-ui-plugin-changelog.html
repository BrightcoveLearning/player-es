Registro de cambios del complemento de IU de lista de reproducción

A continuación se realizan un seguimiento de los cambios realizados en el complemento:

Registro de cambios

7 mayo 2020

v3.8.0

  • Opción habilitada para mostrar descripciones de videos en miniaturas de listas de reproducción

10 feb 2020

v3.7.0

  • Traducciones al árabe añadidas
  • Se corrigió un error que se lanzaba durante la inicialización al usar la siguiente superposición

9 oct 2019

v3.6.3

  • Mostrar la pestaña de la lista de reproducción cuando esté enfocada

8 de oct de 2019

v3.6.2

  • Se resolvió un problema de interfaz de usuario de seguimiento con listas de reproducción horizontales en Firefox e IE11

9 sep. 2019

v3.6.1

  • Se solucionó un problema con las listas de reproducción horizontales que no se mostraban correctamente en IE11 y Firefox

27 ago. 2019

v3.6.0

  • Traducciones añadidas para chino, alemán, español, francés, japonés y coreano

v3.5.0

  • Agrega soporte para la IU de lista de reproducción similar a iframe para reproductores en la página a través de un elemento contenedor

20 mar 2019

v3.4.4

  • Actualizar videojs-playlist a 4.3.1; esto resuelve las regresiones introducidas en videojs-playlist 4.3.0

14 de enero de 2019

v3.4.3

  • Actualizado videojs-playlist a 4.3.0; esto resuelve un problema poco común con listas de reproducción que contienen varios elementos con la misma URL de origen

29 de oct de 2018

v3.4.2

  • Adicional playlistPicker opción, por defecto a true , para permitir la eliminación del selector de lista de reproducción (cuando se pasa como false)
  • Corrección de errores: Haga que la lista de reproducción horizontal en los reproductores iframe tenga una altura estática del 20%, en lugar de una altura variable
  • Corrección de errores: Soluciona un problema por el que aparecía una barra de desplazamiento vertical en las listas de reproducción horizontales.
  • Corrección de errores: Eliminar la interfaz de usuario de la lista de reproducción cuando se elimine el reproductor

3 de octubre de 2018

v3.4.1

  • Se solucionó un problema por el cual aparecía una barra de desplazamiento vertical en las listas de reproducción horizontales
  • Eliminar la interfaz de usuario de la lista de reproducción cuando se elimine el reproductor

25 sep. 2018

v3.4.0

  • Se agregó la opción playlistPicker, predeterminada en true, para permitir la eliminación del selector de lista de reproducción.

17 sep. 2018

v3.3.2

  • Se solucionó un problema por el cual un reproductor de lista de reproducción cargado en el momento incorrecto podía hacer que la interfaz de usuario no se procesara
  • Se ha eliminado el script postinstalación para evitar problemas de instalación
  • Silenciado un play prometer excepción en listas de reproducción
  • Actualizado mediante el generador de complementos v7.2.0

13 jun. 2018

v3.3.1

  • Se solucionó un problema por el cual el botón del siguiente elemento no tenía un cursor de puntero

23 mayo 2018

v3.3.0

  • Admite la asociación explícita de un elemento de la lista de reproducción con un jugador mediante atributos

9 de mayo de 2018

v3.2.1

  • Se solucionó un problema por el cual un código de inserción que coloca un script antes de un contenedor de lista de reproducción en una página con varios jugadores con listas de reproducción podía provocar que esas listas de reproducción se desordenen.

9 de abril de 2018

v3.2.0

  • Se agregó la opción de reproducción aleatoria para aleatorizar la lista de reproducción cada vez que se cambia

29 de marzo de 2018

v3.1.1

  • Truncar títulos de video más largos con puntos suspensivos cuando se desborden

25 de enero de 2018

v3.1.0

  • Actualización para videojs-playlist 4.2.0, agregando duringplaylistchange evento y rest opción para el shuffle() método

17 ene. 2018

v3.0.1

  • Se solucionó el caso en el que la lista de reproducción estaba vacía, pero la barra lateral de la lista de reproducción en un iframe aún se mostraba
  • Se solucionó un problema por el cual el avance automático ocurría incluso si el usuario reiniciaba la reproducción después de un evento finalizado.
  • Se corrigió hacer clic / tocar los íconos en la siguiente superposición y pantalla final en el dispositivo móvil
