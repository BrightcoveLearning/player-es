    Plugin UI de lista de reproducción

    En este tema, aprenderá cómo usar el plugin de la interfaz de usuario de la lista de reproducción para mejorar el aspecto y el aspecto de la funcionalidad predeterminada de la lista de reproducción.

    Resumen

    El complemento de interfaz de usuario de la lista de reproducción contiene las opciones que puede usar para personalizar el comportamiento de la lista de reproducción. Las numerosas opciones proporcionan cambios en las listas de reproducción, incluyendo su diseño, comportamiento y estrategias de implementación. En los ejemplos siguientes se muestran los dos diseños básicos para listas de reproducción, vertical y horizontal. Para cada ejemplo se enumeran un pequeño conjunto de comportamientos para una introducción de lo que se puede personalizar.

    Los siguientes documentos son información esencial para completar la imagen del uso de listas de reproducción:
    • Implementación de listas de reproducción: Este documento muestra las decisiones que debe tomar sobre el uso de listas de reproducción y cómo obtener el código de reproductor que necesita para obtener realmente en su página HTML.
    • Estilo de listas de reproducción: Este documento proporciona información sobre cómo personalizar el aspecto de las listas de reproducción.

    Disposición vertical

    Este ejemplo de lista de reproducción a continuación demuestra:

    • La lista de reproducción vertical situada en el lado derecho del reproductor tiene un botón mostrar/ocultar y la lista de reproducción se dimensiona y posiciona automáticamente. Este es el comportamiento predeterminado cuando se utiliza la implementación del reproductor Estándar (iframe) .
    • La lista de reproducción se muestra inicialmente. Esto se puede alterar usando la hideOnStart opción.
    • Cerca del final del vídeo aparece una superposición que muestra la hora hasta el siguiente vídeo y la miniatura del siguiente vídeo. Este es el comportamiento predeterminado y se controla a través de la nextOverlay opción.

    Diseño horizontal

    Este ejemplo de lista de reproducción a continuación demuestra:

    • Una lista de reproducción horizontal ubicada debajo del vídeo.
    • Además de la superposición que se muestra cerca del final del vídeo, como es true en el ejemplo anterior, una pantalla final muestra la hora antes de que comience a reproducirse el siguiente vídeo. Esto se debe a que la autoadvance opción es distinta de cero y la pantalla final se muestra para la pausa entre vídeos. La pantalla final se puede modificar mediante la nextEndscreen opción.

    Asociación de reproductores/lista de reproducción

    De forma predeterminada, el complemento de interfaz de usuario de la lista de reproducción se encargará de encontrar el elemento contenedor de lista de reproducción correcto para un jugador determinado. Esto significa que encontrará el primer .vjs-playlist elemento vacío en el DOM y usará eso. Sin embargo, el plugin ofrece asociaciones más explícitas que se pueden hacer entre un jugador y un contenedor de lista de reproducción cuando se está construyendo flujos de trabajo complejos con varios jugadores.

    data-for atributo

    El data-for atributo se puede aplicar al contenedor de la lista de reproducción para asociarlo con el de un jugador id , por ejemplo: 

    <video-js id="myPlayerID"
  ...></video-js>

<div class="vjs-playlist" data-for="myPlayerID"></div>

    Este es el método más específico de asociación explícita disponible. Tiene prioridad sobre otros métodos.

    data-player y data-embed atributos

    los data-player y data-embed Los atributos se pueden aplicar al contenedor de la lista de reproducción para asociarlo con un Brightcove Player. Tenga en cuenta que ambos atributos deben usarse para que el asociado funcione correctamente. En el siguiente ejemplo, sólo la segunda <div> etiqueta mantendrá la lista de reproducción del jugador a medida que se realiza una asociación específica. La primera <div> estará vacía. 

    <video-js data-playlist-id="5455901760001"
	data-account="1507807800001"
	data-player="SJLNAJye7"
	data-embed="default"...></video-js>
<script src="https://players.brightcove.net/1507807800001/SJLNAJye7_default/index.min.js"></script>

<div class="vjs-playlist"></div>

<div class="vjs-playlist" data-player="SJLNAJye7" data-embed="default"></div>

    Opciones

    Puede pasar un objeto de opciones al plugin tras la inicialización. Este objeto puede contener cualquiera de las siguientes opciones:

    En la siguiente sección se detalla cómo utilizar las opciones usando código y Studio.

    autoavance

    • autoadvance:
      • Tipo:number
      • Predeterminado:undefined
      • Determina si, y durante cuánto tiempo, se produce una pausa entre los vídeos de una lista de reproducción. Para obtener más información, consulte la sección de autoavance en la Guía de API de listas de reproducción.

    HideOnStart

    • hideOnStart:
      • Tipo:boolean
      • Predeterminado:false
      • Determina si la lista de reproducción está inicialmente oculta de la vista. Esto solo es funcional cuando se utiliza la implementación del reproductor iframe , lo cual es lógico ya que la funcionalidad mostrar/ocultar lista de reproducción solo está disponible en el iframe. Esta opción NO funciona con una lista de reproducción horizontal.
      • Requisitos y dependencias:
        • playlistPicker: true
        • incrustar iframe
        • horizontal: false

    horizontal

    • horizontal:
      • Tipo:boolean
      • Predeterminado:false
      • Muestra la lista de reproducción horizontalmente debajo del reproductor en lugar de verticalmente.
      • Requisitos y dependencias:
        • playlistPicker: true

    NextButton

    • nextButton:
      • Tipo:boolean
      • Predeterminado:true
      • Si true, se añade un botón para pasar al siguiente elemento de la lista de reproducción. El botón se puede desactivar configurando la opción en false. El botón se añade a la barra de control situada a la derecha del botón de reproducción.

    NExtendScreen

    • nextEndscreen:
    • Tipo:boolean
    • Predeterminado:true
    • Si true, el reproductor mostrará una pantalla final modal sobre el reproductor después de la reproducción. La pantalla final se puede desactivar estableciendo la opción en false. Esta pantalla final mostrará una vista previa del vídeo siguiente en la lista de reproducción. Los detalles del modo en la opción son:
      • La autoadvance opción debe establecerse en un valor mayor que cero (de lo contrario, no hay tiempo para mostrar la pantalla final y se omite).
      • La cuenta atrás mostrada representa el tiempo restante hasta el autoadvance desencadenador.
      • El reproductor no debe incluir una implementación del plugin de pantalla final personalizado o del plugin social configurado para mostrar después de un vídeo. Si se detecta cualquiera de las condiciones, no se mostrará la pantalla final.

    NexToVerlay

    • nextOverlay:
      • Tipo:boolean
      • Predeterminado:true
      • Si true, el reproductor mostrará una pequeña superposición en la esquina inferior derecha del reproductor. La superposición se puede desactivar estableciendo la opción en false. Esta superposición mostrará una vista previa del próximo vídeo en la lista de reproducción. Los detalles del modo en la opción son:
        • Debe haber un elemento de lista de reproducción posterior.
        • La autoadvance opción debe ser mayor o igual que cero.
        • La cuenta atrás representa el tiempo restante en el vídeo más cualquier tiempo de autoadvance espera.
        • La superposición aparece 10 segundos desde el final del vídeo. Sin embargo, si el vídeo tiene menos de 30 segundos de duración, aparece en el punto 2/3 de la duración.
        • Si la superposición es rechazada por el usuario, permanecerá descartada hasta que se cambie el origen. Volverá a aparecer para nuevas fuentes.
        • Si la pantalla final hacia arriba está activada, la superposición se ocultará cuando finalice el vídeo. De lo contrario, permanecerá abierto hasta el próximo video.

    lista de reproducción

    • playlist:
      • Tipo:array
      • Predeterminado:undefined
      • Se utiliza para pasar los datos iniciales de la lista de reproducción. Para obtener más información, consulte la sección de listas de reproducción en la Guía de API de listas de reproducción.

    PlayListPicker

    • playlistPicker:
      • Tipo:boolean
      • Predeterminado:true
      • Determina si se debe incluir o no una lista visual de vídeos en la lista de reproducción para que el usuario pueda hacer clic en ellos. los hideOnStart y horizontal Las opciones modifican su apariencia y comportamiento. Cuando playlistPicker es false , otros elementos de la interfaz de usuario todavía se pueden mostrar usando el nextButton , nextEndscreen y nextOverlay opciones.

    PlayOnSelect

    • playOnSelect:
      • Tipo:boolean
      • Predeterminado:false
      • La playOnSelect opción se utiliza para controlar si el vídeo debe comenzar a reproducirse cuando el usuario hace clic en un vídeo de la lista de reproducción. Cuando la opción está configurada en true, la selección de un nuevo vídeo de la lista de reproducción comenzará a reproducirse ese vídeo, incluso si el reproductor estaba en pausa previamente. De forma predeterminada, al hacer clic en un nuevo vídeo de la lista de reproducción, se cargará el nuevo vídeo, pero mantendrá el reproductor en pausa si se ha pausado de antemano.
      • Requisitos y dependencias:
        • playlistPicker: true

    repetir

    • repeat:
      • Tipo:boolean
      • Predeterminado:false
      • Repite una lista de reproducción después de haber terminado el último vídeo de la lista de reproducción. Esta funcionalidad reproduce el primer vídeo de la lista de reproducción una vez finalizado el último vídeo de la lista de reproducción.

    MostrarDescripción

    • showDescription:
      • Tipo:boolean
      • Predeterminado:false
      • Mostrar descripciones de vídeo para cada elemento de la lista de reproducción.

    barajar

    • shuffle:
      • Tipo:boolean
      • Predeterminado:false
      • Mezcla los elementos de la lista de reproducción cada vez que se carguen nuevos datos.

    Uso de las opciones

    Tiene dos formas de utilizar la opción:

    1. En la sección > PLUGINS PLAYERS de Studio.
    2. Usar JavaScript con el reproductor.
