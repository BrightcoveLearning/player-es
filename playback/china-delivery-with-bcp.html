Introducción

Brightcove proporciona una solución para enviar video a China desde fuera del país. La solución consta de dos partes:

  • Un servidor proxy alojado en China
  • Una CDN conectada específica de China

El proxy enruta el tráfico que no es de video (solicitudes de API de reproducción, balizas de Analytics y el propio reproductor web) hacia y desde los espectadores en China a la infraestructura de Brightcove ubicada en el exterior.

El CDN está conectado al proxy y maneja el contenido de video e imagen.

Los siguientes requisitos son necesarios para admitir Brightcove China Delivery:

Configuración

Jugador Brightcove

  • Se requiere Brightcove Player versión 6.44+; última versión recomendada

Configuración de Brightcove Player

Para entregar video dentro de China, debe configurar el reproductor para usar el proxy de entrega de Brightcove China.

Para crear un reproductor compatible con China, siga estos pasos:

  1. En Video Cloud Studio, abra el JUGADORES módulo. Utiliza un reproductor existente o crea uno nuevo.

    Recuerde utilizar Brightcove Player versión 6.44 o posterior (se recomienda la última versión).
  2. Seleccione el enlace del nombre del jugador para abrir las propiedades del jugador.

  3. En la navegación izquierda, selecciona Editor JSON.

  4. Para configurar el proxy, agregue lo siguiente:

    "china_proxy_domain": "china.example.com"
    Debe ser solo un nombre de host. Sin protocolo (por ejemplo, https: //) y sin caracteres especiales, incluidas barras diagonales.

  5. El código debería tener un aspecto similar a este:

  6. Hacer clic Ahorrar y publique el reproductor.
  7. Cuando inserte o comparta su reproductor, observe que el código de inserción utiliza el CDN de Brightcove Player. 
    <script
  src="//players.brightcove.net/1752604059001/jnOD8yLME_default/index.min.js">
</script>
  8. Cambie el CDN de Brightcove Player para usar su dominio de entrega de China. 
    <script
  src="//china.example.com/1752604059001/jnOD8yLME_default/index.min.js">
</script>
    Utilizando el players.brightcove.net El dominio resultará en una incapacidad para reproducir contenido y tiempos de carga del reproductor degradados en China.

Nota

Con china_proxy_domain establecido, todas las solicitudes iniciadas por el jugador se enviarán a las URL dentro de ese dominio. Usando el valor de muestra anterior (china.example.com ) los usuarios pueden ver solicitudes para:

Sistema Camino
Reproductores CDN //china.example.com/jugadores / ...
API de reproducción //china.example.com/playback/v1 / ...
Analítica //china.example.com/v2/tracker ...

Localización

Con china_proxy_domain establecido, Brightcove Player automáticamente configurarse con dos idiomas:

  • zh-Hans: Chino simplificado (predeterminado)
  • zh-Hant: Chino tradicional

Establecer un china_proxy_domain tiene el mismo efecto que establecer las siguientes configuraciones (aunque no son necesarias):

"language": "zh-Hans",
"languages": ["zh-Hans", "zh-Hant"],

Este efecto predeterminado de la china_proxy_domain no ocurrirá si se proporciona algún valor para el language o languages ¡configuración! Para anular el comportamiento predeterminado, puede mezclar y combinar estas configuraciones.

Agregar más idiomas

Si hay más idiomas que zh-Hans y zh-Hant son necesarios, puede agregar una configuración de idiomas al reproductor. Esta configuración podría tener un aspecto similar a lo siguiente:

"china_proxy_domain": "china.example.com",
"languages": ["zh-Hans", "zh-Hant", "en", "jp"],

El reproductor en este ejemplo todavía estaría predeterminado en zh-Hans , pero también agregaría los idiomas inglés y japonés.

Cambiar el idioma predeterminado

Para configurar un idioma predeterminado diferente al zh-Hans , Agrega un language configuración. La configuración debería tener un aspecto similar a esto:

"china_proxy_domain": "china.example.com",
"language": "zh-Hant",
