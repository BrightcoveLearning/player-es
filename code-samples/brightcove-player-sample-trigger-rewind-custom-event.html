    Ejemplo de reproductor de Brightcove: Desencadenar rebobinado con un evento personalizado

    En este tema, aprenderá cómo implementar un botón de rebobinado distribuyendo y escuchando un evento personalizado.

    Resumen

    Este ejemplo y el código asociado se proporcionan como guía para su desarrollo de producción. El código no está certificado para uso en producción ni es compatible oficialmente.

    Aunque Brightcove Player proporciona una serie de eventos listos para usar, es posible que desee utilizar sus propios eventos personalizados para desacoplar y eliminar las dependencias entre los objetos DOM y los detectores de eventos. Este modelo controlado por eventos es una opción arquitectónica que quizás desee considerar.

    En este ejemplo se agrega un botón Atrás a la barra de controles. Cuando el usuario hace clic en el botón Atrás, se activa un evento personalizado junto con la cantidad de tiempo para rebobinar. El detector de eventos personalizado encuentra la posición actual del vídeo y rebobina el vídeo.

    La secuencia de desarrollo

    Aquí está la secuencia de desarrollo recomendada:

    1. Utilice la implementación del reproductor incrustado en página para probar la funcionalidad de su reproductor, plugin y CSS (si es necesario CSS)
    2. Coloque el JavaScript y CSS del complemento en archivos separados para pruebas locales
    3. Implemente el código del complemento y CSS en su servidor una vez que haya resuelto cualquier error
    4. Usa Studio para añadir el plugin y CSS a tu reproductor
    5. Reemplace la implementación del reproductor incrustado en página si determina que la implementación del iframe es una mejor opción (detallada en la siguiente sección)

    Para obtener más información acerca de estos pasos, revise el paso a paso: Guía de desarrollo de plugins.

    iframe o incrustar en página

    Al desarrollar mejoras para Brightcove Player, tendrá que decidir si el código es el más adecuado para la implementación de iframe o In-Page In-Page In-Page. La recomendación de la mejor práctica es construir un complemento para su uso con una implementación iframe. Las ventajas de usar el reproductor iframe son:

    • No hay colisiones con JavaScript y/o CSS existentes
    • Responsabilidad automática
    • El iframe facilita el uso en aplicaciones de redes sociales (o siempre que el vídeo tenga que «viajar» a otras aplicaciones)

    Aunque la integración del reproductor de incrustaciones en página puede ser más compleja, hay ocasiones en las que planificará su código en torno a esa implementación. Para generalizar, este enfoque es mejor cuando la página que contiene necesita comunicarse con el jugador. Específicamente, aquí hay algunos ejemplos:

    • El código de la página que contiene necesita escuchar y actuar sobre los eventos del jugador
    • El reproductor utiliza estilos de la página que contiene
    • El iframe hará que la lógica de la aplicación falle, como una redirección desde la página que contiene

    Incluso si su implementación final no usa el código iframe, aún puede usar el código incrustado en página con un complemento para su JavaScript y un archivo separado para su CSS. Esto encapsula tu lógica para que puedas usarla fácilmente en varios jugadores.

    Flujo de aplicación

    La lógica básica detrás de esta aplicación es:

    • Cree un nuevo elemento en el DOM para el botón de rebobinado.
    • Añada el botón de rebobinado a la barra de control del reproductor.
    • Escucha cuando un usuario hace clic en el botón de rebobinado. Cuando se hace clic en él, active su evento personalizado con datos.
    • Escuche el evento personalizado con datos y rebobine la reproducción de vídeo en función de los datos definidos.

    Agregar elementos al DOM

    Encuentre el código que está etiquetado:

    // +++ Add elements to the DOM +++

    Cree un nuevo <div> elemento en el DOM y asigne un id atributo con un valor de BackButton. Cree un nuevo <img> elemento en el DOM y asigne el src atributo con la URL a su botón de rebobinado.

    Añadir botón de rebobinado a la barra de controles

    Encuentre el código que está etiquetado:

    // +++ Add rewind button to the control bar +++

    Obtenga el spacer elemento en la barra de control y agregue los elementos del botón de rebobinado.

    Descripción de los eventos personalizados

    Los eventos personalizados le permiten desacoplar y eliminar dependencias entre los objetos DOM y los detectores de eventos, creando un modelo arquitectónico controlado por eventos.

    Envío

    Para distribuir un evento personalizado, utilice el trigger() método de la siguiente manera:

    // trigger a custom event
myPlayer.trigger('eventName');
or
// trigger a custom event with data
myPlayer.trigger('eventName', {data: 'some data'});

    Para obtener más información sobre el envío de eventos personalizados, consulte el índice de Player API.

    En este ejemplo, el código hace lo siguiente:

    • distribuye un evento personalizado, denominado rewind
    • Envía un objeto de datos con un nombre de amount y un valor de 5 - Verá en el código JavaScript donde se establece este valor
    // trigger a custom event with data
myPlayer.trigger('rewind', {'amount': rewindAmount});

    Escuchar

    Para escuchar un evento personalizado, puede utilizar el on() método de la siguiente manera:

    // listen for a custom event
player.on('eventName', function(evt){});
or
// listen for a custom event with data
player.on('eventName', function(evt,data){});

    También puede usar el one() método si desea que el detector de eventos se active solo una vez y luego se elimine. Para obtener más información sobre cómo escuchar eventos, consulte el índice de Player API.

    En este ejemplo, el código hace lo siguiente:

    • Escucha un evento personalizado, denominado rewind
    • Pasa el evento y los objetos de datos a una función
    // listen for a custom event with data
 myPlayer.on('rewind',function(evt,data){
});
