    Conceptos clave

    Algunos conceptos deben entenderse para utilizar eficazmente los puntos de referencia en Brightcove Player. Estos conceptos se explican en esta sección del documento.El siguiente concepto debe entenderse para utilizar eficazmente los puntos de referencia en Brightcove Player.

    Puntos de referencia del catalog de Video Cloud

    El primer concepto a entender tiene que ver con la terminología. En Brightcove Player, los puntos de referencia se almacenan como elementos de pista de texto, según el estándar HTML. Esto significa que cuando se utiliza un vídeo de Video Cloud, los puntos de referencia del «estilo de Video Cloud» se convierten en pistas de texto.

    Estos puntos de referencia del «estilo de Video Cloud» también se pueden denominar puntos de referencia «catalog», ya que se leen en el catalog de Video Cloud (consulte el documento catalog del Reproductor para obtener más información sobre el catálogo). Cuando se realiza esta conversión, parte de la información del punto de referencia del catalog, como el tipo y la hora de punto de referencia, se convierten en la pista de texto.

    Estructura de puntos de referencia del catálogo

    El siguiente concepto a entender es que hay una diferencia significativa entre los puntos de referencia del catalog y el estándar HTML en la estructura. La diferencia es que un punto de referencia HTML puede tener una duración. Esto significa que para cada punto de referencia, se distribuirán dos eventos de cambio de punto de referencia: uno al inicio del punto de referencia y otro al final del punto de referencia.

    En el proceso de conversión, cada punto de referencia del catalog se convierte en un punto de referencia con tiempos de inicio y parada idénticos. Esto significa que para cada punto de referencia del catalog, se distribuirán dos eventos de punto de referencia y deben contabilizarse en el código.

    activecues matriz

    OtroUn punto conceptual importante a entender cuando se trata de puntos de referencia HTML es la activeCues matriz. Todos los puntos de referencia se definen en una matriz. Hay otra matriz, activeCues, que contiene los puntos de referencia que están «activos», lo que significa que el tiempo de reproducción está entre los tiempos de inicio y parada del punto de referencia.

    Cuando se utilizan puntos de referencia del catalog, las horas de inicio y de parada son idénticas, por lo que solo estarán activas para el segundo definido. Además, con los puntos de referencia del catalog, es muy poco probable que dos puntos de referencia estén activos al mismo tiempo, ya que no se superponen.

    Nota: Puede haber varios conjuntos de pistas de texto, por ejemplo un conjunto para puntos de referencia y otro para subtítulos de vídeo. En el código de ejemplo de este documento, las pistas de texto de puntos de referencia se encuentran en el primer y segundo elementos de la matriz TextTracks. Verá esto reflejado en el código.

    Tipos de puntos de referencia

    En este documento verá puntos de referencia con una type propiedad asignada un valor. Estos valores de tipo se asignan al crear puntos de referencia en la interfaz de usuario de Studio. Hay dos tipos de puntos de referencia, pero ambos simplemente proporcionan un valor de cadena a la type propiedad. El valor apropiado sólo es útil si se utiliza en JavaScript personalizado mientras se procesa la información del punto de referencia. Los tipos se detallan aquí:

    • Ad - Asigna el valor de cadena de AD a la type propiedad
    • Code - Asigna el valor de cadena de CODE a la type propiedad

    Recuperar toda la información de puntos de referencia de Video Cloud

    Como puede haber notado que no toda la información de puntos de referencia de Video Cloud está directamente disponible para usted desde la activecues matriz. Esto se corrige fácilmente recuperando la información deseada de la mediainfo propiedad.

    El enfoque básico de esta solución es:

    1. Espere a que el loadstart evento se complete la mediainfo propiedad.
    2. Asigne la cue_points matriz de la mediainfo propiedad a una variable. Esta variable contiene información completa de puntos de referencia de Video Cloud.
    3. En un evento de punto de referencia, recupere los datos de punto de referencia correspondientes en función del valor de la time propiedad. Esto se hará utilizando una función auxiliar que toma un objeto de una matriz en función de un valor de propiedad en el objeto.
    4. Utilice los datos del punto de referencia.

    La siguiente imagen muestra toda la matriz de puntos de referencia (arriba a la izquierda), la colección de datos de punto de referencia único (arriba a la derecha) y una propiedad de esa colección de datos de punto de referencia único (abajo a la derecha).

    En el fragmento de abajo solo se explicará el código nuevo/modificado de los ejemplos anteriores.

    • Líneas 452-463: Incluya la función que se utilizará para extraer una única colección de datos de punto de referencia de la matriz de todos los puntos de referencia. Tenga en cuenta que le pasa la matriz de todos los puntos de referencia, la propiedad en la que está buscando un valor específico y, por último, el valor que está buscando.
    • Línea 432: Escucha el evento.loadstart Cuando el vídeo comienza a cargarse, la mediainfo propiedad se rellena.
    • Línea 434: Asigne la matriz de todos los puntos de referencia de Video Cloud a una variable.

      • cuePointAra: Toda la colección de puntos de referencia de Video Cloud.
      • 'time': Propiedad en la que se busca un valor.
      • tt.activeCues[0].startTime: Hora de inicio del punto de referencia que se está manejando actualmente en el controlador de eventos de despacho de puntos de referencia.

    <script>
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      cuePointAra = [],
      allCuePointData;
    myPlayer.on('loadstart', function () {
      //console.log('mediainfo', myPlayer.mediainfo);
      cuePointAra = myPlayer.mediainfo.cue_points;
      var tt = myPlayer.textTracks()[0];
      tt.oncuechange = function () {
        if (tt.activeCues[0] !== undefined) {
          var dynamicHTML = "id: " + tt.activeCues[0].id + ", ";
          dynamicHTML += "text: " + tt.activeCues[0].text + ", ";
          dynamicHTML += "startTime: " + tt.activeCues[0].startTime + ",  ";
          dynamicHTML += "endTime: " + tt.activeCues[0].endTime;
          document.getElementById("insertionPoint").innerHTML += dynamicHTML + "<br/>";
          allCuePointData = getSubArray(cuePointAra, 'time', tt.activeCues[0].startTime);
          console.log('cue point data:', allCuePointData);
          console.log('cue point metadata:', allCuePointData[0].metadata);
        }
      }
      myPlayer.play();
      myPlayer.muted(true);
    });

    function getSubArray(targetArray, objProperty, value) {
      var i, totalItems = targetArray.length,
        objFound = false,
        idxArr = [];
      for (i = 0; i < totalItems; i++) {
        if (targetArray[i][objProperty] === value) {
          objFound = true;
          idxArr.push(targetArray[i]);
        }
      }
      return idxArr;
    };
  });
</script>

    Puntos de referencia estándar HTML5

    Los puntos de referencia estándar HTML5 se almacenan como elementos de pista en un formato prescrito. Un buen contenido introductorio se puede encontrar en el siguiente tutorial HTML5 Rocks: Empezar con el elemento Track.     Formato de archivo WebVTT para puntos de referencia

    El formato de archivo WebVTT está estrictamente definido. Para los puntos de referencia, el archivo se compone de la siguiente manera:

    • La cadena WebVTT como la primera línea del archivo
    • Una sola línea en blanco
    • El identificador de un punto de referencia específico
    • La duración en el formato 00:00:00 .000 — > 00:00:00 .000; esto es el formato horas:minutos:seconds.milisegundos y se analiza estrictamente; los números deben estar rellenos de cero si es necesario
    • Los caracteres que siguen a la duración de la siguiente línea en blanco se colocan en un valor de texto ; si desea almacenar varios valores dispares aquí JSON funciona bien, ya que se puede analizar fácilmente
    • Una sola línea en blanco
    • Se pueden agregar varios puntos de referencia utilizando el formato identificador/duración/texto/línea en blanco
    Nota: Puede validar su formato WebVTT con este validador.

    Introducción a los puntos de referencia del proceso

    Los pasos básicos para usar los puntos de referencia estándar HTML5 son:

    1. Lea en el archivo WebVTT que contiene los puntos de referencia utilizando una <track> etiqueta anidada como hijo de la <video-js> etiqueta.
    2. En un controlador de loadedmetadata eventos, recupere el elemento de seguimiento de texto adecuado.
    3. En el mismo controlador de eventos, configure un segundo controlador de eventos para el evento de punto de oncuechange referencia.
    4. En el controlador de oncuechange eventos, compruebe si el objeto activecues array tiene un elemento cero definido, en caso afirmativo, actuar en el inicio del punto de referencia.
      Nota: Tendrá oncuechange eventos distribuido cuando el punto de referencia comience y se detenga. La activecues matriz tendrá un valor sólo para la duración del punto de referencia. Por lo tanto, si se produce un oncuechange evento y no hay objetos definidos en la activecues matriz, sabe que el evento distribuido es para una parada de punto de referencia.
    5. En el controlador de oncuechange eventos, compruebe si el objeto activecues array tiene el elemento cero definido, si no, actuar sobre la parada del punto de referencia.
    Nota: Si sus puntos de referencia tienen duraciones superpuestas, la lógica de los pasos anteriores tendrá que ser ligeramente alterada.

    Buscar pista correcta

    En varios lugares de este documento podría producirse un posible problema si hay varias pistas de texto asociadas a un reproductor. Se asumió que sólo una pista de texto está asociada con el reproductor, por lo que se utilizó este código: var tt = myPlayer.textTracks()[0];. Al seleccionar el elemento de matriz cero se asume una sola pista de texto asociada al reproductor.

    Las pistas de texto no solo se utilizan para puntos de referencia, sino también para otros tipos de datos. El atributo kind puede contener los siguientes valores:

    • subtítulos
    • subtítulos
    • descripciones
    • capítulos
    • metadatos

    Esto significa que es muy posible que se puedan asociar varias pistas de texto con el reproductor, y se necesita una manera de encontrar la correcta para su lógica de aplicación. El siguiente código recorre las pistas de texto disponibles hasta que se encuentre el valor de metadatos (punto de referencia):

    <script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      allTextTacks,
      attLength,
      tt;
    myPlayer.one("loadedmetadata", function () {
      allTextTacks = myPlayer.textTracks();
      attLength = allTextTacks.length;
      for (var i = 0; i < attLength; i++) {
        if (allTextTacks[i].kind === 'metadata') {
          tt = allTextTacks[i];
          break;
        };
      };
    });
  });
</script>

    La lógica de ejemplos anteriores se podría usar para procesar los puntos de referencia utilizando la tt variable, que contiene la pista de texto deseada.

    ID3 y puntos de referencia de medios

    Si tiene puntos de referencia ID3 o puntos de referencia de medios asociados con sus medios, puede tratarlos utilizando el id3CuePointsTrack() y mediaCuePointsTrack() métodos. Por ejemplo, para escuchar los cambios de punto de referencia, usaría lo siguiente:

    videojs.getPlayer('myPlayerID').ready(function () {
  var myPlayer = this;
  myPlayer.one("canplay", function () {
    myPlayer.id3CuePointsTrack().on('cuechange', function () {
      // process cue point here
    });
  });
});

    Detalles de ID3

    A continuación se proporciona más información sobre el uso de puntos de referencia ID3:

    • Las etiquetas ID3 se pueden utilizar para insertar metadatos cronometrados en la secuencia.
    • Puede tener varios fotogramas ID3 por segmento.
    • Brightcove Player analiza los puntos de referencia ID3 y los expone como una pista de texto, utilizando el id3CuePointsTrack() método antes mencionado.
    • Es una práctica recomendada esperar el canplay evento o existe la posibilidad de que la pista no sea recuperable al intentar acceder a él.
    • Se admiten varios fotogramas por etiqueta.

    Problemas conocidos

    • En Safari, los puntos de referencia no se activarán si la hora de finalización es igual a la hora de inicio. Por lo tanto, si la compatibilidad con Safari es un problema, los puntos de referencia necesitan una duración mayor que cero.
